1프로젝트 개요

배경

RTBIO는 현재 발주 접수, 재고 관리, 정산, 영업 데이터 관리 등 핵심 운영 업무를 엑셀·수기 중심으로 처리하고 있다. 이로 인해 중복 입력, 오기입, 실시간 현황 파악 불가 등의 문제가 발생하고 있으며, 향후 의료기기 2등급 제품 판매 확대 시 납품 이력 및 추적 관리 요구도 증가할 것으로 예상된다.

목표

  • 경영지원팀 수기 업무(하루 약 30건 발주 처리)를 시스템 기반으로 자동화
  • 부서별 데이터를 하나의 통합 시스템으로 관리
  • 향후 앱 확장, 의료기기 추적, 생산 연동까지 단계적으로 확장 가능한 구조 구축

사업적 포지셔닝

본 시스템은 RTBIO의 업무 흐름에 100% 맞춤 설계된 커스텀 ERP다. 기존 범용 ERP 대비 아래와 같은 차별점을 갖는다.

항목기존 ERP (더존, SAP 등)본 시스템
커스터마이징어렵고 비쌈RTBIO 업무에 100% 맞춤
UI/UX복잡한 구조실무자 중심 단순 설계
확장모듈 구매 필요필요 기능 직접 추가
비용 구조라이선스 비용 지속 발생유지보수 계약으로 전환

2기존 ERP 관련 방향 선택

본 시스템 구축 방식에 대해 두 가지 옵션을 제시한다.

옵션 A — 독립 시스템 구축 (권장)

기존 ERP와 별도로 완전히 새로운 시스템을 구축한다. 기존 ERP의 데이터는 엑셀 내보내기를 통해 초기 마이그레이션 방식으로 이전한다.

장점

  • RTBIO 업무 흐름에 100% 맞춤 설계 가능
  • 기존 ERP 구조에 종속되지 않음
  • 장기적으로 유지보수·확장이 용이
  • 운영 시스템 단일화로 담당자 혼선 제거

단점

  • 초기 데이터 마이그레이션 작업 필요
  • 기존 ERP와 병행 운영 기간 발생 가능

옵션 B — 기존 ERP 연동형

기존 ERP를 유지하면서 본 시스템을 프론트엔드 + 자동화 레이어로 연결한다.

장점

  • 기존 ERP 데이터 즉시 활용 가능
  • 전환 리스크 낮음

단점

  • 기존 ERP 구조에 맞춰야 해 커스터마이징 제한
  • 두 시스템 간 데이터 정합성 관리 필요
  • 기존 ERP 벤더 의존도 유지
권장안: 옵션 A — 기존 ERP는 엑셀로 데이터를 내보낼 수 있어 초기 마이그레이션이 가능하다. 장기적으로 RTBIO 업무 확장성과 시스템 단일화를 고려할 때 독립 구축이 더 유리하다.

4웹 vs 앱 판단

결론: 1단계는 반응형 웹으로 구축. 앱은 2단계 이후 필요성 검토 후 결정.

판단 근거

기준웹 (반응형)네이티브 앱
경영지원팀 내부 업무✅ PC 중심, 웹으로 충분불필요
대리점/병원 발주✅ 링크 접속, 설치 불필요설치 거부감 있을 수 있음
영업사원 외근 중 조회⚠️ 가능하나 경험 다소 낮음✅ 더 편함
푸시 알림⚠️ 웹푸시 가능하나 제한적✅ 네이티브 푸시
개발 비용낮음높음 (iOS + Android)
배포 속도즉시앱스토어 심사 필요

앱 전환 검토 시점 (Phase 2 이후)

아래 조건 중 2개 이상 해당 시 앱 개발 착수를 권장한다.

  • 영업사원이 외근 중 실시간 조회를 자주 하는가?
  • 거래처가 앱 설치를 원하는가?
  • 푸시 알림이 업무에 실질적으로 필요한가?

앱 전환 시 비용 최소화 전략

1단계 웹 서비스를 API-first 구조로 설계하기 때문에, 앱 전환 시 NestJS 백엔드를 그대로 재사용할 수 있다. React Native로 앱 UI만 추가하면 iOS/Android 동시 출시가 가능하여 추가 개발 비용을 최소화한다.

5사용자 역할 및 유저 플로우

5-1. 권한별 사용자 역할 및 데이터 소유권

역할대상접근 범위
슈퍼어드민RTBIO 관리자전체 시스템
경영지원팀RTBIO 내부 담당자발주·거래명세서·정산·마감
품질관리팀RTBIO 내부 담당자재고 입출고·현황 대시보드
영업부RTBIO 영업 직원거래처 매출·미수금 조회 (읽기 전용)
외부거래처대리점 / 병원 담당자발주 입력 + 본인 거래 내역 조회
데이터 소유권: 시스템 내 모든 데이터의 소유권은 RTBIO에 귀속된다. 시스템 인수인계 완료 후 슈퍼어드민 계정은 RTBIO로 이전되며, 하마다랩스는 유지보수 계약 기간 중 인프라 모니터링 및 장애 대응 목적에 한정된 별도 관리 계정을 보유한다. 해당 계정은 RTBIO의 요청 시 언제든 접근 로그를 제공하며, 계약 종료 시 즉시 폐기한다.

5-2. 핵심 플로우 — 발주 → 출고 → 정산

[외부거래처: 대리점/병원] │ 발주 옵션 A: 웹폼 직접 입력 │ 발주 옵션 B: 카카오톡/엑셀 전송 → AI 파싱 ▼ [발주 접수] ── 자동 발주번호 생성 ▼ [경영지원팀 확인] - 품목·수량·배송비 자동 계산 - 거래처별 예외 정책 자동 적용 - 이상 있으면 수동 수정 후 확정 ▼ [출고 확인] ── (Phase 2부터 재고 자동 차감 연동) - Phase 1: 경영지원팀이 수동으로 출고 확인 - Phase 2 이후: 품질관리팀 재고 시스템과 자동 연동 ▼ [거래명세서 자동 생성] → PDF + 메일 발송 ▼ [정산 관리] → 거래처별 정산 주기 자동 적용 ▼ [월 마감 보고서 자동 생성]

5-3. 발주 옵션 비교

옵션 A — 웹폼 직접 입력 (권장)

거래처 담당자가 URL로 접속하여 직접 발주하는 방식이다.

어드민이 거래처 계정 등록 → 초대 메일 자동 발송 → 거래처 담당자 비밀번호 설정 → URL 접속 → 로그인 → 품목 선택 → 수량 입력 → 배송지 확인 → 제출 → 접수 확인 문자/메일 자동 발송
항목내용
장점데이터 정형화, 자동화 처리 용이, 오류 최소화
단점거래처 담당자 초기 적응 기간 필요
추가 비용없음

옵션 B — 기존 방식 유지 + AI 파싱

거래처는 기존처럼 카카오톡·엑셀로 발주하고, 내부에서 AI가 자동 파싱하는 방식이다.

항목내용
장점거래처 변화 없음, 기존 방식 유지
단점AI 파싱 정확도 한계, 수동 수정 필요 케이스 발생
추가 비용AI API 토큰 비용 발생 (발주 건수 비례)
권장안: 옵션 A를 기본으로 제공한다. 옵션 B는 특정 거래처의 강한 요청이 있을 경우 별도 추가 계약으로 적용한다. (옵션 A 단독 / 옵션 A + 옵션 B 병행 중 선택)

5-4. 배송비 자동 계산

  • 기본 기준: 품목군별 고정 배송비 적용
  • 거래처별 면제 조건: 특정 거래처에 배송비 면제 또는 별도 기준 설정 가능
  • 금액 기준 면제: 주문 금액 일정 기준 이상 시 배송비 자동 면제 설정 가능
  • 설정 주체: 경영지원팀 어드민 화면에서 직접 등록·수정

5-5. 거래처별 예외 정책

partner_policies 테이블에서 거래처별로 아래 유형의 예외를 설정·관리한다.

  • 할인율 (품목별 또는 전체 적용)
  • 배송비 면제 여부 및 기준
  • 외상 한도 (미수금 허용 한도)
  • 정산 주기 (월말 마감, 60일, 입금 확인 후 등 자유 설정)
  • 특정 품목군 접근 제한

5-6. 정산 주기 관리

정산 유형예시 적용 거래처
월말 일괄 마감대부분의 대리점
N일 후 정산 (예: 60일)일부 병원
입금 확인 후 정산소규모 거래처
개별 발주 건별 정산특수 계약 거래처

정산 주기는 어드민에서 거래처별로 자유롭게 설정하며, 마감 시점이 되면 시스템이 자동으로 해당 기간 발주를 집계하여 정산 내역을 생성한다.

5-7. 경영지원팀 일별 업무 플로우

출근 ├─ 1. 대시보드에서 신규 발주 확인 ├─ 2. 발주 내용 검토 및 확정 (클릭 한 번) ├─ 3. 출고 확인 후 거래명세서 자동 발송 ├─ 4. 미수금·정산 현황 확인 └─ (월말) 마감 보고서 자동 생성 → 발송

현재: 하루 약 30건 수기 처리 → 목표: 확정 클릭만으로 처리

6단계별 기능 로드맵

📌 현장 프로세스 파악 후 기능 확정 안내

아래 기능 목록은 현재까지 파악된 정보를 기반으로 작성한 초안(예비 프레임워크)이다. 계약 확정 후 하마다랩스 PM이 RTBIO 현장에 상주하며 각 부서 실무자와 함께 실제 업무 프로세스를 직접 파악한다. 이 과정에서 기존 엑셀·수기 업무의 세부 흐름, 예외 케이스, 현장 요구사항 등을 반영하여 기능을 재정의·추가·조정할 예정이다.

따라서 Phase별 세부 기능은 현장 프로세스 파악 완료 후 최종 확정되며, 현재 명시된 항목은 변경될 수 있다.

9서비스 운영 및 지원

인프라 운영

시스템 인프라(서버, DB, 스토리지, 모니터링, 배포)는 하마다랩스가 전담 운영한다. RTBIO는 인프라를 직접 관리할 필요 없이, 월 구독료에 아래 항목이 모두 포함된다.

포함 항목내용
AWS 인프라서버(컨테이너), DB, S3 스토리지, SES 메일 발송
모니터링·배포서버 상태 모니터링, 장애 감지, 배포 관리
백업일별 자동 백업, 30일 보관
AI LLM 토큰 비용발주 옵션 B(AI 파싱) 사용 시 발생하는 AI API 토큰 비용 포함
RTBIO는 별도 인프라 비용 없이 월 구독료 하나로 시스템 전체를 이용할 수 있다.

장애 대응 기준

등급기준대응 목표 시간

  • 지원 채널: 업무 시간 내 담당자 직통 연락
  • 백업 복구: RPO(복구 시점 목표) 24시간, RTO(복구 시간 목표) 8시간
  • 계약 종료 시 데이터 처리: 계약 종료 시 백업 데이터를 포함한 모든 데이터를 RTBIO에 이관하거나 RTBIO의 요청에 따라 삭제한다.
