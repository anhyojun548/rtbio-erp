04. 포털 × 기능 · RBAC 매트릭스

5개 포털이 R01~R24 중 어떤 기능을 다루는지, Role 별로 어떤 액션에 접근 가능한지 한눈에 매칭.

1. 포털 개요

📊 /admin (경영지원)

거래처·제품·재고·주문·명세서·수금·원장·보고서 — 전체 관리자 기능.

TENANT_OWNER ADMIN

💼 /exec (영업)

내 거래처, 내 주문, 담당자 매출, 학회 방명록 — 담당자 관점 데이터.

EXEC + ADMIN/OWNER 대리

🏭 /qc (QC)

출고 칸반, 재고 현황·알럼, 유통기한, 샘플 출고, 설정 — 창고/출고 작업.

QC + ADMIN/OWNER

👔 /ceo (임원)

12-col grid 위젯 대시보드 — KPI/리스트 프리셋 10종 커스터마이징.

TENANT_OWNER ADMIN

🤝 /client (거래처)

내 발주 · 명세서 · 수금 · 계약서 · 프로필 — 읽기 전용.

CLIENT

🌐 공용 라우트

/login · /403 · /api/auth/* — 인증 흐름 전용.

3. 포털별 메뉴 구조

📊 /admin — 경영지원 (20+ 메뉴)

  1. 대시보드 (재고 알럼 + 만료 계약)

  3. 제품 · 제품 상세 (로트 패널)
  4. 재고 현황 · 재고 이력
  5. 재고 알럼 (OUT/LOW)
  6. 유통기한 대시보드
  7. 주문 목록 · 신규 · 상세 (상태머신)
  8. 출고 칸반 · 단계 관리
  9. 출고 내역 + CSV
  10. 거래명세서 목록 · 상세 · PDF
  11. 수금 (Payment + BankTxn)
  12. 원장 (월별 마감)
  13. 월간 보고서
  14. 담당자별 매출
  15. 영업 이력서
  16. 판매 계약서
  17. 데이터 사용량

  1. 대시보드 (내 거래처/주문/매출)
  2. 내 거래처 (정렬·필터)
  3. 내 주문

5. RBAC 구현 (코드)

모든 서버 액션은 src/lib/session.ts 의 두 헬퍼를 통해 권한을 강제합니다.

requireRole(...roles)

세션에서 role 을 꺼내 허용 목록과 매칭. 불일치 시 /403 redirect.

await requireRole("ADMIN", "TENANT_OWNER");
await requireRole("QC", "ADMIN", "TENANT_OWNER");

requireClient()

role=CLIENT && clientId 존재 검증. 거래처 포털 전용.

const user = await requireClient();
// user.clientId 보장됨
const orders = await prisma.order.findMany({
  where: { clientId: user.clientId }
});
