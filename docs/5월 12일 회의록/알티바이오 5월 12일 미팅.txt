1. 처음 물건 생산 발주가 들어갔을때 프로젝트? 카드? 형태로 만들어져서 해당 건 입출고를 관리할 수 있게  <- 이부분은 품질관리팀이랑 얘기를 나눠야됨
2. 이 기능은 영업팀, 대표 관리자 대시보드에서 한눈에 트레킹이 되어야됨

발주는 원단, 부자재 ,제품으로 세가지로 카테고리를 나눠서 관리 해야됨

전체적으로 공지사항 기능 필요 -> 거래처에 발송할 수 있는 기능 그리고 전체업체, 업체별로 선택할 수 있게 기능을 나누는 작업 필요


경영지원팀
매입매출이나 거래처 원장 등등 각 월별로 월만 선택하면 한번에 볼 수 있는 기능이 있으면 좋을꺼같다(기존 시작, 종료일 날짜 보는것도 필요)
모든 조회 페이지에서는 시작날짜 종료날짜 월별 조회 기능 필요
거래처 원장에서 발주 내역 클릭했을때 창이 윈도우로 뜨게(플로팅)
거래명세서 출력또는 발행할때 몇번 했는지도 확인할 수 있는 기능 필요
20개의 업체에서 각각 발주를 했으면 경영지원팀은 기간을 정해놓고 
보고서 페이지는 사이즈 색상 표시 필요, 날짜 선택했을때 업체별로, 제품별로 필터링 할 수 있는 기능 필요
매입매출장 발주내역 수정 기능 필요 왜냐하면 이미 발주가 완료된 제품을 업체에서 반품할때도 있음 그래서 수정기능과 반입 관리 기능 필요(환불 및 반품 했을때 관리) 그리고 발주건마다 히스토리 파악도 필요함 이건 품질관리팀에서도 히스토리를 조회할 수 있어야됨
각업체마다 무료 배송 설정 기능 필요


udi기능은
병원에 납품하면 공급내역 보고를 해야됨
1. 조회를 하면 병원에 납품한게 쭈르륵 뜨고, 공급 구분 하고, 공급형태는 의료기관에 공급 해서 udi 전송을 하면 의료기기통합정보시스템으로 보고자료가 전송됨
2. 결국 하고싶은건 해당월에 병원에 납품한 제품들을 공급내역을 보고 udi를 전송

품질관리팀 
보고서 작성은 모든 양식이 들어가야되고, 이미지, 파일첨부 기능이 필요하다
메뉴얼, 절차서의 용도는 어디서든 확인을 할 수 있도록 조회 기능이 필요하고 챗봇이 해당 내용을 기반으로 답변해주고
보고서가 작성됐을때 저장되고 조회, 삭제 관리 가능하게
현재 발주방식 절차와 표, 양식이 필요함
발주, 입고 확인은 일단 수기로하는게 편하긴함 일단 현재 상황에서는 표로 보는게 효과적이다
발주확정 후 출고관리로 넘어갔을때 포장 전 단계까지는 발주취소 기능이 필요(업체에서 갑자기 취소할 수 있기 때문에)
출고건이 수정이되면(경영팀에서 수정함) 출고관리 칸반보드에 카드에 알림이 뜨고, 그 알림을 확인하기 전까지 칸반보드에서 다음단계로 안넘어감

택배사를 로젠택배를 쓰고 있는데 웹에서 송장을 출력해서 붙히고 있다. 발주건에 대해서 송장번호를 일일이 기입하는게 힘들다 <- 로젠택배 api 연동이 안돼서 확인이 안됨
포장이 완료됐을때 몇박스로 어디에 포장을 했다. 표시를 하면 거래처에서 확인할 수 있는 기능이 필요하다
