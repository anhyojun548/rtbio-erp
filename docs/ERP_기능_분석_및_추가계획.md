## 2. GAP 분석: 빠진 기능

아래를 **3가지 레벨**로 구분한다.
- **Tier 1** — 런칭 전 반드시 필요 (없으면 얼마에요 대체 불가)
- **Tier 2** — 운영 3개월 내 추가 (없으면 불편)
- **Tier 3** — 추후 필요시 (있으면 좋음)

---

### 2.1 데이터 입력 화면 (엑셀형 그리드)

얼마에요를 쓰던 사용자들이 가장 먼저 찾는 기능. 엑셀처럼 행을 추가하면서 빠르게 데이터를 입력하는 화면.

| # | 화면 | 포탈 | 설명 | Tier |
|---|------|------|------|------|
| 1 | **매입 입력** | 경영지원 | 공급처 선택, 제품/수량/매입단가 행 추가, 합계 자동 계산 | 🔴 1 |
| 2 | **수금/입금 등록** | 경영지원 | 거래처 선택, 입금액/결제방법/일자 입력, 일괄 등록 | 🔴 1 |
| 3 | **입고 등록** | 품질관리 | 제품 선택, 입고수량/일자, 매입 연동 | 🔴 1 |
| 4 | **재고 조정** | 품질관리 | 실사 후 차이 조정, 사유 코드 (파손/분실/실사) | 🔴 1 |
| 5 | **단가 일괄 변경** | 경영지원 | 제품군 단가 변경 시 전 거래처 일괄 반영 | 🟡 2 |
| 6 | **엑셀 임포트/내보내기** | 경영지원 | 모든 목록 화면에서 엑셀 다운로드, 데이터 일괄 업로드 | 🔴 1 |

## 4. 엑셀형 그리드 UI 패턴

매입/수금/입고/재고조정 입력 화면은 모두 동일한 패턴을 사용한다:

```
┌──────────────────────────────────────────────────────────┐
│  📅 2026-04-14     공급처: [드롭다운 ▼]     [저장] [취소] │
├──────────────────────────────────────────────────────────┤
│  #  │ 제품명        │ 규격  │ 수량  │ 단가     │ 금액      │
├─────┼───────────────┼───────┼───────┼──────────┼──────────┤
│  1  │ [검색/선택 ▼] │ M     │ [  ] │ [     ] │ 자동계산  │
│  2  │ [검색/선택 ▼] │ L     │ [  ] │ [     ] │ 자동계산  │
│  3  │ [검색/선택 ▼] │       │ [  ] │ [     ] │ 자동계산  │
│     │              │       │       │          │          │
│  [+ 행 추가]                              합계 │ ₩000,000 │
└──────────────────────────────────────────────────────────┘

[← 목록으로]                    [임시저장]  [확정 저장]
```

**핵심 UX:**
- Tab 키로 셀 이동 (엑셀 느낌)
- 제품 검색 시 코드/이름 자동완성
- 행 추가/삭제 자유
- 수량 × 단가 = 금액 실시간 계산
- 하단 합계 자동
- 빈 행 자동 무시

## 5. 개발계획서 반영 사항

위 분석 결과를 기존 개발계획서(개발계획서_RTBIO_ERP.md)에 반영해야 할 것:

### DB 스키마 추가
```
[suppliers]          — 공급업체 마스터 (매입 정규화)
[tax_invoices]       — 세금계산서 발행 이력
[fiscal_periods]     — 회계 기간 (월별 열기/닫기)
[notifications]      — 인앱 알림
[system_settings]    — 시스템 설정 (key-value)
[business_calendar]  — 휴일/영업일
```

### 기존 테이블 수정
```
모든 테이블에 deleted_at (nullable timestamp) 추가  — Soft Delete
orders, invoices 등에 version 필드 추가             — Optimistic Lock
products에 udi_code 필드 추가                       — UDI 바코드
purchases.supplier_name → purchases.supplier_id     — 정규화
```

### API 추가
```
GET/POST  /api/purchases          — 매입 등록/조회
GET/POST  /api/receiving          — 입고 등록/조회
POST      /api/inventory/adjust   — 재고 조정
GET       /api/ledger/:clientId/transactions  — 거래처 원장 상세
GET       /api/reports/aging      — 미수금 Aging
GET       /api/reports/journal    — 매입매출장
GET/PUT   /api/settings           — 시스템 설정
GET/POST  /api/tax-invoices       — 세금계산서
GET       /api/notifications      — 알림 조회
POST      /api/fiscal-periods     — 회계 기간 관리
```

### STEP 수정
```
STEP 0 (기존)  + 시스템 설정, Soft Delete, 문서 채번, 공급업체 마스터
STEP 1 (기존)  + 입고 등록 화면
STEP 2 (기존)  + 매입 입력, 수금 등록, 거래처 원장, 미수금 Aging, 매입매출장
               + 세금계산서 발행 화면, 재고 조정
               + 이메일 큐, 스케줄러
STEP 3 (기존)  + 회계 기간 관리, 원장 확정 잠금
STEP 4 (기존)  + 알림 센터, 인쇄 최적화, 비밀번호 변경
```
