1. 현재 업무 프로세스

RTBIO의 현재 업무 흐름을 팀별로 정리했습니다.

발주 접수 흐름

거래처업무폰 카톡 발주
→
업무 단톡방전 직원 공유
→
경영지원팀얼마에요 입력
→
품질관리팀포장 · 출고
일 평균 20~30건 (월/화/수 많을 때 30~40건), 모두 수기 처리 중

경영지원팀

  1. 업무폰 발주 확인 → 얼마에요에 매입/매출 수기 입력 (품목 코드 하나하나)
  2. 등록 후 발주서와 더블체크 (퇴근 전 또는 다음 날 출근 시)
  3. 거래명세서 발행 — 하나하나 PDF로 발행 (많게는 30~40건)
  4. 다음 메일로 거래처에 거래명세서 개별 발송
  5. 제품군 · 납품처마다 할인율이 다름 — 거래처 전용 품목에 금액 설정해놓고 사용 중
  6. 매달 마감 업무:
    • 거래처 원장 접속 → 업체별 매출액 확인 → 마감 원장 작성 (엑셀 수기 입력)
    • 잔액이 안 맞는 곳들이 있어서 엑셀로 잔액 맞춰서 계산
    • 최종 입금 요청 원장을 대리점/병원에 메일 발송
  7. 병원마다 결제 방식이 다름:
    • 당월 말 카드결제
    • 사용량만큼 카드결제 (영업팀이 현장에서 확인)
    • 계좌이체
    • 3개월 후 결제 (1월 매출 → 4월에 결제)
  8. 계좌 입금 업체 → 계산서 발행, 카드 결제 → 계산서 발행 안 함
  9. 일일 보고서 작성 (엑셀): 업체별 리스트 + 그날 발주 수량/금액 + 총 매출
  10. 기간별 매출 보고서 제출
    • 대리점: 1일~25일 (또는 말일)
    • 병원: 전달 26일~이번 달 25일

품질관리팀

  1. 업체 발주서 수신 → 고객과 카톡 소통 (수량 확인, 재고 부족 안내) → 확정 후 단톡방 공유
  2. 담당자 배정 — 카톡 이모지로 작업 상태 표시 중
    • 입 벌리는 모양 = 바코드 출력 완료
    • 스마일 마크 = 포장 완료 (누가 했는지 추적용)
  3. 바코드 출력 → 제품 세팅 → 택배 포장 → 송장 출력 → 출고
  4. 거래처마다 요청 사항이 다름
    • 거래처 명세표 동봉 요청하는 업체
    • RTBIO 명세표를 넣어야 하는 업체
  5. 택배 마감: 오후 3시 30분 (늦으면 4~5시)
    • 3시 30분 이후 발주 → 익일 출고 안내
    • 오전 발주 후 오후에 수량 변경 요청 가능 → 마감 전 수정 필요
  6. 퇴근 시 일일 재고 보고 (엑셀): 제품군별 발주량, 출고량, 잔여 재고
  7. 샘플/미팅용 출고 — 매출 미반영, 재고에서만 차감 처리 필요

발주 폼 특이사항

주의: 발주 업체명 ≠ 실제 배송 주소지인 경우 있음 (대리점이 병원으로 직송). 배송 방법(택배/방문/퀵) 구분 필요. 경영지원팀 거래명세서 비고란에 실제 배송 병원명 기입해야 함.

2. 미팅에서 확정된 방향

주요 의사결정 사항입니다.

발주 방식

웹사이트 발주폼

기존 엑셀 양식 그대로 웹에 띄워서 수량만 입력하는 방식. 쿠팡/네이버 스토어 방식은 제품군이 많아 번거로움.

확정
고객 인증

ID/PW 방식 (RTBIO가 생성)

ID: 담당자 이름 또는 병원명. PW: 관리번호 (예: 0010). 로그인 시 거래처별 할인율, 제품 자동 적용.

확정
ERP 전환

얼마에요 → 하마다랩스 ERP

완전 전환. 기존 데이터는 엑셀 내보내기로 이관 (약 3년치). RTBIO에서 사용 기능 목록 정리 예정.

확정
플랫폼

웹 기반 (앱 X)

앱은 비용 2배 + 애플 심사 이슈. 모바일 반응형 웹으로 충분. 알림은 카톡 비즈니스 검토 중.

비용 확인 필요

3. 요구 기능 정리

미팅에서 도출된 기능 요구사항을 팀별로 정리했습니다.

외부 (거래처용 웹)

# 기능 상세
1 발주폼 기존 엑셀 양식 기반, 전 제품 노출 + 수량 입력 방식
2 실시간 재고 반영 품절 제품 발주 차단 (단, 대표님 의견 확인 필요 — 일단 받고 후처리 가능성)
3 발주 수정 택배 마감 전까지 시간 제한 두고 수량 변경 가능
4 배송지 관리 기본 업체 주소 + 다른 주소 직접 입력 (병원 직송 대응)
5 배송 방법 선택 택배 / 방문 수령 / 퀵
6 발주 내역 조회 본인이 발주한 수량, 금액, 해당 월 내역 조회
7 출고 상태 확인 품질관리팀 출고 처리 시 웹에 실시간 반영
8 거래명세서 자동 발송 출고 완료 시 자동 발행/발송
9 무료배송 안내 수량 입력 시 무료배송 여부 + "N개 더 추가하면 무료배송" 안내
10 거래처별 할인율 로그인 시 해당 업체 전용 단가 자동 노출

내부 — 경영지원팀

# 기능 상세
1발주 자동 입력웹 발주 → DB 자동 반영 (수기 입력 제거)
2거래명세서 일괄 발행PDF 개별 저장 + 메일 발송 자동화
3거래처별 할인율 관리제품군 × 거래처별 할인율 설정
4결제 방식 관리당월 말 카드, 사용량 카드, 계좌이체, N개월 후 결제 등
5마감 원장 자동 생성거래처별 마감 기간 설정 → 자동 계산 + 잔액 정리
6원장 메일 발송마감 원장 완성 → 거래처 메일 자동 발송
7일일 보고서업체별 발주 수량/금액 + 당일 총 매출 자동 생성
8기간별 매출 보고서업체별 기간 설정 가능 (대리점 vs 병원 마감 기간 상이)
9계산서 발행 연동계좌 입금 업체만 계산서 발행
