2. 발주 → 출고 상세 (QC 칸반 6단계)

품질관리 포털의 칸반 보드가 물류 진행의 중심이 됩니다. 각 카드는 6단계 중 한 열에 위치합니다.

거래처 포털 발주 상세

client-portal에서 거래처가 직접 발주. 택배 마감시간(기본 15:30) 이전까지 수량 변경 가능. 실시간 availableStock 조회로 재고 부족 시 경고.

(실시간 재고 표시)"]

  CP3 --> CP4{"availableStock
충분?"}
  CP4 -->|"부족"| CP3
  CP4 -->|"충분"| CP5["발주 제출

3. 결제/정산 흐름 (월 1~25일 발주 → 월말 입금)

대리점 정기 결제 규칙: 매월 1~25일 발주분은 월말 일괄 입금이 원칙. 일부입금/연체는 미수금으로 관리.
flowchart LR
  O1["1~25일 발주"] --> O2["월말 일괄 입금 대상"]
  O2 --> B["🏦 은행 입금 수기 등록
(admin-portal)"]
  B --> CK{"입금액 확인"}
  CK -->|"= 예상금액"| ST1["✅ 입금완료"]
  CK -->|"< 예상금액"| ST2["⚠️ 일부입금
차액 = 미수금"]
  CK -->|"입금 없음"| ST3["🔴 미입금
연체일 경과"]
  ST2 --> AR["📊 미수금 관리
(Aging 0/30/60/90+)"]
  ST3 --> AR
  AR --> N["📧 자동 리마인드
(Nodemailer)"]

4. 반품 처리 (InventoryAdjustment 재사용)

별도 반품 테이블 없이 재고 조정 테이블에 reason='반품'으로 기록. 입·출고와 동일한 감사 추적.
flowchart TD
  R1["거래처 반품 요청"] --> R2["경영지원 반품 승인"]
  R2 --> R3["InventoryAdjustment
type='IN' reason='반품'"]
  R3 --> R4["physicalStock += qty
availableStock += qty"]
  R4 --> R5["📊 매출 차감 반영
(음수 거래명세서)"]
  R5 --> R6["✅ 미수금 차감 or
환불 처리"]

5. 포털별 워크플로우 요약

포털사용자핵심 워크플로우
👥 exec-portal영업팀 / 경영진 대시보드 조회 → 담당자별 매출 분석 → 거래처 상세 관리 → 사용량 입력 → 학회 방명록 → 발주서/명세표 출력
🏢 admin-portal경영지원 발주 접수 → 거래명세서 발행 → 세금계산서 발행 → 수금 관리 → 거래처원장/매입매출장
🏭 qc-portal품질관리 칸반 보드 진행 → 재고 관리 → 샘플 출고 → 운영 설정(업무시간/마감)
🏥 client-portal거래처 제품 조회 → 발주 → 주문 현황 → 명세서 다운로드 → 미수금 확인
