## 5. 스타트업 크레딧 비교

### AWS Activate
- **기본 티어**: $1,000 (이메일 신청, 대부분 승인)
- **Activate Portfolio**: $25,000~$100,000 (승인된 엑셀러레이터/VC 연결 필요)
- **유효기간**: 12~24개월
- **조건**: 까다로움, 심사 불투명

### Microsoft for Startups
- **기본 티어**: $1,000 즉시 + 검증 후 **$5,000 추가 = 총 $6,000**
- **Investor Offer**: 최대 **$150,000 / 2년** (승인된 VC 추천 필요)
- **유효기간**: 180일~2년
- **조건**: 법인 등록된 스타트업이면 대부분 통과 ✅

**크레딧 측면에서 Azure가 압도적으로 유리** (신청 장벽 낮음, 금액 비슷)

## 6. 장단점 정리

### AWS 선택 시

**장점**
- ✅ 과업내용서 명시된 스택 (별도 협의 불필요)
- ✅ 국내 개발자 풀 가장 넓음 (나중에 인력 충원 쉬움)
- ✅ 서비스 종류가 가장 많음 (세부 튜닝 가능)
- ✅ 국내 레퍼런스 풍부 (문제 해결 커뮤니티 큼)
- ✅ Next.js 생태계 최적 (SST, AWS Amplify 등)

**단점**
- ❌ 콘솔/IAM 복잡, 학습 곡선 가파름
- ❌ 크레딧 확보가 어려움 (Activate 조건 복잡)
- ❌ 데이터 전송 비용이 Azure보다 비쌈
- ❌ 초기 설정 보안 실수 위험 (기본값이 관대함)

### Azure 선택 시

**장점**
- ✅ **스타트업 크레딧 $6,000 확보 용이** → 초기 1년 거의 무료
- ✅ Microsoft 생태계 연동 (M365, GitHub, Copilot)
- ✅ 콘솔 직관적, IAM 간단
- ✅ 엔터프라이즈 고객 선호 (특히 대기업/공공 업체)
- ✅ 기본 보안 설정이 보수적 (실수 방지)

**단점**
- ❌ 과업내용서 변경 필요
- ❌ 국내 AWS 대비 개발자 풀 좁음
- ❌ 일부 서비스가 AWS보다 설익음
- ❌ Next.js 전용 도구가 AWS만큼 풍부하지 않음

## 7. 특수 고려 사항

### 7.1 의료/규제 대응

| 항목 | AWS | Azure |
|------|-----|-------|
| HIPAA 적합 서비스 | 모두 지원 | 모두 지원 |
| 국내 개인정보보호법 대응 자료 | 풍부 | 풍부 |
| CSAP (국내 클라우드 보안 인증) | ✅ | ✅ |
| ISMS-P 획득 시 | 공공/금융 진출 용이 | 동일 |

**→ 규제 측면에서 둘 다 동등**

### 7.2 개발 생산성

| 항목 | AWS | Azure |
|------|-----|-------|
| Next.js 컨테이너 배포 자동화 | 중간 (ECS + CodePipeline) | 쉬움 (Container Apps + GitHub Actions) |
| PostgreSQL 초기 세팅 | 중간 | 쉬움 |
| 로컬 개발 환경 | LocalStack 등 | Azure Functions Core Tools |
| CLI/SDK 편의성 | AWS CLI 강력 | Azure CLI 깔끔 |

**→ 초기 개발 속도는 Azure가 약간 빠름 (설정 간단)**

### 7.3 확장 시 제약

| 항목 | AWS | Azure |
|------|-----|-------|
| 3단계 Kubernetes | EKS (성숙) | AKS (성숙) |
| PostgreSQL 수평 확장 | Aurora, Partitioning | Hyperscale (Citus, 공식 지원) |
| 멀티리전 복제 | Aurora Global DB | Flexible Server Geo-Redundant |
| 전용 인스턴스 격리 | 풍부한 옵션 | 풍부한 옵션 |

**→ 확장성 측면에서 둘 다 충분. Azure의 Citus가 멀티테넌트 SaaS에 유리한 측면 있음**

## 10. 추천 시나리오

### A안: 안정 운영 우선 → **AWS**
- 과업내용서 변경 불필요
- 국내 레퍼런스 풍부 → 문제 해결 빠름
- 인력 교체/충원 시 부담 적음

### B안: 비용 효율 우선 → **Azure**
- 스타트업 크레딧으로 1~2년 거의 무료 운영
- 초기 자금 부담 최소화
- Microsoft 생태계 연동 이점

### C안: 하이브리드 (비추천)
- 운영 복잡도만 증가

---

## 11. 최종 권고

- **장기 안정성 최우선이면 AWS**
- **초기 비용 절감 + 빠른 개발 최우선이면 Azure**
- **과업내용서 수정이 어렵다면 AWS 고정**

**어느 쪽이든 1단계 구성과 확장 로드맵은 동일하게 적용 가능.** 회의에서 결정된 쪽으로 2단계, 3단계 세부 문서를 확정하면 됨.
