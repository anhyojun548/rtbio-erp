# RTBIO ERP 프로토타입 — 미팅 메모 누락 항목 보완 계획

**작성일**: 2026-04-17 | **기준**: 4/16 미팅 메모 재검토

**Context**: 4/16 미팅 메모 전체를 프로토타입 현황과 1:1 대조한 결과, 17개 항목이 미완료(🔴 13개) 또는 부분완료(🟡 9개) 상태. 이를 3단계로 나눠 완성도를 높인다. 이전 계획에서 프로토타입 1차 적용은 완료됐고, 이 계획은 **2차 보완**에 해당한다.

## 검증된 현황 (📋 참고)

- ✅ **이미 완성**: 보드 설정(칸반 열 편집), page-confirm 보류 버튼, 매입관리 비활성화, 신규거래처 유형 추가
- 🟡 **부분완성**: 세금계산서(플레이스홀더), 수금관리(일부입금 없음), 거래처원장(담당자 필터 없음), 모바일 카드 접기만
- 🔴 **미완성**: 재고 삭제, 대리점 검색, 업무시간 설정, 명세서 다운로드 버튼, 발주서/거래명세표 출력, 방명록 개별 입력, 대시보드 기간 전역 선택기, "기타" 거래처 유형

## Step A (최우선 — 5시간) — 4/21 데모 필수

### A-1. QC 칸반 카드 접기/펼치기 (데스크탑도 기본 접힘)
- **파일**: `prototype/qc-portal.html` CSS + JS 영역
- **작업**:
  - `.kanban-card` 내부를 "요약 영역(발주번호, 거래처, 상태)" + "상세 영역(담당자, 배송, 배송지, 합계)"로 분리
  - 상세 영역 기본 `display:none`
  - 카드 하단에 ▼/▲ 토글 버튼 추가
  - 기존 모바일 아코디언 CSS는 유지 (동일 메커니즘 재사용)
- **재사용**: `toggleCardDetail()` 함수 이미 존재 → 확장

### A-2. QC 재고현황 제품 삭제 기능
- **파일**: `qc-portal.html` `renderInventory()` 근처
- **작업**: 재고 행에 "삭제" 버튼 추가. 확인 모달 후 PRODUCTS 배열에서 제거
- **재사용**: `showModal()` 패턴

### A-3. QC 샘플출고 대리점 검색
- **파일**: `qc-portal.html` page-sample 섹션
- **작업**: `<select>` → `<input type="text">` + 자동완성 드롭다운(ac-wrapper/ac-dropdown 클래스 이미 존재)
- **재사용**: admin-portal의 거래처 자동완성 패턴 차용

### A-4. QC 칸반 카드 "거래명세서 다운로드" 버튼
- **파일**: `qc-portal.html` `renderKanban()` 카드 템플릿
- **작업**: 출고완료 단계 카드에만 📄 버튼 추가 → `showToast('거래명세서를 다운로드합니다')` (프로토타입이므로 토스트만)
- 경영지원이 명세서 발급한 건만 활성화 (order.invoiceIssued 플래그 mock)

### A-5. QC 업무시간/택배 마감시간 설정
- **파일**: `qc-portal.html` 새 페이지 `page-settings`
- **작업**:
  - 사이드바 하단에 ⚙️ "설정" 메뉴 추가
  - 업무 시작/종료 시각, 택배 마감시간 input
  - localStorage 저장
- **재사용**: 기존 form-group CSS

### A-6. 거래처 유형 "기타" 추가 (공통)
- **파일**: `admin-portal.html`, `exec-portal.html` 필터/등록 select
- **작업**: `<option value="기타">기타</option>` 추가

### C-4. 메인 대시보드 "거래처+담당자" 위젯 프리셋 추가

## 검증 방법

각 Step 완료 후 브라우저로 확인:

**Step A 완료 기준:**
1. qc-portal 열기 → 칸반 카드 기본 접힌 상태 → ▼ 클릭 시 상세 펼침
2. 재고현황 → 제품 행에 삭제 버튼 → 클릭 시 확인 모달 → 삭제
3. 샘플출고 → 대리점 입력칸에 글자 입력 → 자동완성 드롭다운
4. 칸반 카드(출고완료) → 📄 버튼 클릭 → 토스트
5. 사이드바 ⚙️ 설정 → 업무시간 설정 후 저장 → 새로고침해도 유지
6. 거래처 유형 select에 "기타" 옵션 존재

**Step B 완료 기준:**
1. 세금계산서 페이지 → 목록 보임 → 신규 발행 모달 → 저장
2. 수금관리 → 은행 입금 수기 추가 → 입금 상태 "일부입금" 표시
3. 거래처원장 → 담당자 필터 동작
4. 거래명세서 → 담당자 검색 동작
5. 영업 거래처 상세 → 발주서/명세표 출력 버튼
6. 학회 카드 클릭 → 방명록 개별 입력 모달

**Step C 완료 기준:**
1. admin 대시보드 → 상단 기간 선택 → 전 위젯 반영
2. 위젯 수정 → 개별 기간 override 설정 → 적용
3. 전역 기간 재변경 → 위젯 override 리셋 확인
4. 거래처 등록 → "할인율 설정" 버튼 → 팝업 오픈
5. 영업 현황 → 담당자 행 클릭 → 품목 드릴다운 필터링
6. 대시보드 위젯 추가 → "거래처별 담당자" 프리셋 존재
