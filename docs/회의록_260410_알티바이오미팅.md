## 1. 현재 업무 프로세스 (AS-IS)

### 발주 접수 흐름
1. **거래처** → 업무폰(카톡)으로 발주 전달 (하루 20~30건, 월/화/수 많을 때 30~40건)
2. **경영지원팀** → 업무 단톡방에 정리해서 공유
3. 이 내용을 보고 **품질관리팀**은 제품 포장, **경영지원팀**은 얼마에요에 발주 입력

### 경영지원팀 업무
1. 업무폰 발주 확인 → **얼마에요**에 매입/매출 수기 입력 (품목 코드 하나하나 입력)
2. 등록 후 발주서와 **더블체크** (퇴근 전 또는 다음 날 출근 시)
3. **거래명세서 발행** — 하나하나 PDF로 발행 (많게는 30~40건)
4. **다음 메일**로 거래처에 거래명세서 개별 발송
5. **제품군·납품처마다 할인율이 다름** — 거래처 전용 품목에 금액 설정해놓고 사용 중
6. **매달 마감 업무:**
   - 거래처 원장 접속 → 업체별 매출액 확인 → 마감 원장 작성 (엑셀 수기 입력)
   - 잔액이 안 맞는 곳들이 있어서 엑셀로 잔액 맞춰서 계산
   - 최종 입금 요청 원장을 대리점/병원에 메일 발송
7. **병원마다 결제 방식이 다름:**
   - 당월 말 카드결제
   - 사용량만큼 카드결제 (영업팀이 현장에서 확인)
   - 계좌이체
   - 3개월 후 결제 (1월 매출 → 4월에 결제)
8. 계좌 입금 업체 → **계산서 발행**, 카드 결제 → 계산서 발행 안 함
9. **일일 보고서 작성** (엑셀): 업체별 리스트 + 그날 발주 수량/금액 + 총 매출
10. **기간별 매출 보고서** 제출 — 업체별로 마감 기간이 다름
    - 대리점: 1일~25일 (또는 말일)
    - 병원: 전달 26일~이번 달 25일

### 품질관리팀 업무
1. 업체 발주서 수신 → **고객과 카톡으로 소통** (수량 확인, 재고 부족 안내 등) → 확정 후 단톡방 공유
2. **담당자 배정** — 카톡 이모지로 작업 상태 표시 중:
   - 입 벌리는 모양 = 바코드 출력 완료
   - 스마일 마크 = 포장 완료 (누가 했는지 추적용)
3. **바코드 출력 → 제품 세팅 → 택배 포장 → 송장 출력 → 출고**
4. 거래처마다 요청 사항이 다름:
   - 거래처 명세표 동봉 요청하는 업체
   - RTBIO 명세표를 넣어야 하는 업체
5. **택배 마감: 오후 3시 30분** (늦으면 4~5시)
   - 3시 30분 이후 발주 → 익일 출고 안내
   - 오전 발주 후 오후에 수량 변경 요청 들어올 수 있음 → 마감 전 수정 가능해야 함
6. **퇴근 시 일일 재고 보고** — 엑셀로 정리:
   - 그날 주문 제품, 발주 수량, 출고 수량, 남은 재고 수량
   - 제품군별로 캡처해서 단톡방에 공유
7. **샘플/미팅용 출고** — 매출에는 안 잡히고 재고에서만 차감 처리 필요

### 발주 폼 특이사항
- 발주 업체명 ≠ 실제 배송 주소지인 경우 있음 (대리점이 병원으로 직송)
- 배송 방법 구분 필요: 택배 / 방문 수령 / 퀵
- 경영지원팀 거래명세서 비고란에 실제 배송 병원명 기입해야 함

## 2. 미팅에서 확정된 방향

### 발주 방식
- **웹사이트 발주폼 방식 채택** (앱 X, AI 파싱 X)
- 이유: 거래처 연령대가 40대 이상 다수, 쿠팡/네이버 스토어 방식은 제품군이 많아 번거로움
- **기존 발주 폼(엑셀 양식) 그대로 웹에 띄워서** 수량만 입력하는 방식
- 품목 코드 조회 방식이 아닌, 전 제품이 미리 보이고 숫자만 입력하는 형태

### 고객 로그인
- **아이디/비밀번호 방식 채택**
- ID: 담당자 이름 또는 병원명
- PW: RTBIO에서 부여하는 관리번호 (예: 0010, 0011)
- RTBIO가 생성해서 거래처에 전달하는 방식
- 로그인하면 해당 거래처의 할인율, 제품이 자동 적용

### ERP 전환
- **얼마에요 → 하마다랩스 ERP로 완전 전환**
- 얼마에요 데이터는 엑셀로 내보내기 가능 (약 3년치 데이터)
- RTBIO에서 "얼마에요에서 어떤 기능을 많이 쓰는지" 정리해서 공유 예정

### 플랫폼
- **웹 기반** (앱 설치 없이, 모바일 반응형)
- 앱은 비용 2배 + 애플 심사 이슈로 비추천
- 알림은 카카오톡 비즈니스 또는 대안 검토 예정

## 3. 요구 기능 정리

### 외부 (거래처용 웹)
| # | 기능 | 상세 |
|---|------|------|
| 1 | 발주폼 | 기존 엑셀 양식 기반, 전 제품 노출 + 수량 입력 방식 |
| 2 | 실시간 재고 반영 | 품절 제품 발주 차단 (단, 대표님 의견 확인 필요 — 일단 받고 후처리 가능성) |

| 10 | 거래처별 할인율 자동 적용 | 로그인 시 해당 업체 전용 단가 노출 |

## 4. RTBIO 측 전달 요청 사항 (Action Items)

| 담당 | 전달 내용 | 비고 |
|------|----------|------|
| 경영지원팀 | 얼마에요에서 사용 중인 기능 목록 정리 | 어떤 기능을 언제 쓰는지 |
| 경영지원팀 | 기존 발주폼 엑셀 파일 공유 | 내용 없이 양식만 |
| 경영지원팀 | 거래처별 할인율·결제 방식·마감 기간 정리 엑셀 | 이미 정리해둔 것 있음 |
| 경영지원팀 | 일일 보고서 / 기간별 매출 보고서 샘플 | 엑셀 파일 |
| 경영지원팀 | 마감 원장 샘플 (대리점용, 병원용 각각) | 엑셀 파일 |
| 품질관리팀 | 추가 기능 요청 사항 정리 | 생각나는 대로 |
| 품질관리팀 | 제품별 무료배송 기준 수량 정리 | 제품군별 조건 |
| 하마다랩스 | 카카오톡 비즈니스 알림 비용 확인 | 건당 비용 파악 |
| 하마다랩스 | 프로토타입 제작 후 2차 미팅 | 화면 보면서 디테일 확정 |

## 5. 확정된 개발 방향 요약

```
얼마에요 → 하마다랩스 ERP로 완전 전환
거래처 발주 → 웹사이트 (기존 폼 기반, 수량 입력 방식)
고객 인증 → RTBIO가 ID/PW 생성해서 전달
내부 관리 → 웹 기반 (앱 없이), 부서별 권한 분리
알림 → 카카오톡 비즈니스 검토 중 (비용 확인 필요)
데이터 이관 → 얼마에요 엑셀 내보내기 → 신규 DB 이관
```
