PROBLEM STATEMENT

1. 문제 정의

RTBIO는 의료기기 유통 기업으로, 하루 20~40건의 발주를 엑셀과 수기로 처리하고 있습니다.

반복 입력

같은 데이터를 3~4번 입력

발주서 → 얼마에요 → 거래명세서 PDF → 다음 메일 발송. 같은 내용을 각각 수기로 처리.

오류 발생

수기 입력으로 인한 금액/수량 실수

30~40건을 하나하나 입력하면서 더블체크에도 잔액이 안 맞는 거래처 존재. 입사 전 누적 오류도 있음.

비효율 소통

카톡 이모지로 출고 상태 관리

품질관리팀 6명이 카톡 이모지로 담당자 배정/작업 상태 관리. 누락 추적 어려움.

마감 부담

월말 마감에 3~5일 소요

거래처별 마감 기간(대리점 vs 병원), 결제 방식(4가지 이상), 할인율이 모두 달라서 엑셀로 하나하나 계산.

해결하지 않을 경우의 리스크: 발주량 증가 시 인력 확충 필요, 잔액 오류로 거래처 신뢰 하락, 거래명세서 발행 지연으로 고객 불만 증가, 월말 마감 업무로 인한 경영지원팀 야근 상시화.
GOALS & NON-GOALS

2. 목표

Goals

G1 — 발주 자동화

수기 입력 제로화

거래처 웹 발주 → DB 자동 반영. 경영지원팀의 발주 입력 업무 완전 제거.

G2 — 명세서 자동화

거래명세서 자동 발행/발송

출고 완료 시 PDF 자동 생성 + 메일 자동 발송. 하루 30~40건 수기 발행 제거.

G3 — 출고 프로세스

품질관리팀 작업 상태 디지털화

카톡 이모지 → 웹 기반 상태 관리. 담당자 배정, 작업 단계, 출고 완료까지 실시간 트래킹.

G4 — 마감 자동화

원장/보고서 자동 생성

거래처별 마감 기간 설정 → 마감 원장 자동 계산 + 메일 발송. 월말 마감 3~5일 → 1일 이내.

Non-Goals (이번 범위 밖)

모바일 앱

비용 2배 + 애플 심사 이슈. 모바일 반응형 웹으로 대체. 추후 고도화 시 검토.

AI 발주 파싱

거래처가 보내는 사진/메시지를 AI로 인식하는 방식. 웹 폼 방식 확정으로 보류.

회계 시스템 연동

세금계산서/부가세 신고 자동화. 현재는 계산서 발행 알림 수준. 추후 별도 프로젝트.

해외 재고/생산 연동

베트남 생산 및 해외 재고 연동. 현재 범위에서 완전 분리. 별도 프로젝트로 진행.

USER STORIES

5. 유저 스토리

거래처 (외부)

거래처 담당자로서, 웹에서 기존 발주폼 형태로 수량만 입력해서 발주하고 싶다. 기존 엑셀처럼 전 제품이 보여서 하나하나 검색할 필요 없이 빠르게 발주할 수 있다.
거래처 담당자로서, 발주 후 택배 마감 전까지 수량을 수정하고 싶다. 급하게 추가/변경해야 할 때 전화 없이 직접 처리할 수 있다.
거래처 담당자로서, 발주 시 무료배송 기준까지 남은 수량을 확인하고 싶다. 매번 RTBIO에 전화로 물어볼 필요 없이 스스로 무료배송 조건을 맞출 수 있다.
거래처 담당자로서, 출고 상태와 거래명세서를 웹에서 바로 확인하고 싶다. 언제 출고됐는지, 거래명세서가 맞는지 전화 없이 확인할 수 있다.

경영지원팀

경영지원팀 담당자로서, 웹 발주가 들어오면 자동으로 DB에 반영되는 것을 원한다. 하루 30~40건을 하나하나 얼마에요에 수기 입력하는 작업이 사라진다.
경영지원팀 담당자로서, 거래명세서를 한 번에 일괄 발행하고 메일 자동 발송하고 싶다. 30~40건을 하나하나 PDF 저장하고 메일 보내는 작업이 클릭 한 번으로 끝난다.
경영지원팀 담당자로서, 마감 원장이 자동으로 생성되길 원한다. 거래처별 마감 기간/결제 방식이 달라서 엑셀로 맞추던 작업이 자동 계산된다.

품질관리팀

품질관리팀 담당자로서, 발주를 확인하고 당일출고/익일출고를 선택하면 거래처에 알림이 가게 하고 싶다. 배민 주문확인처럼 빠르게 처리하고, 거래처가 직접 상태를 확인할 수 있다.
품질관리팀 담당자로서, 웹에서 담당자 배정과 작업 상태(바코드/포장/출고)를 관리하고 싶다. 카톡 이모지 대신 정확한 상태 관리가 되고, 누가 어떤 작업을 했는지 추적 가능하다.
품질관리팀 담당자로서, 샘플/미팅용 출고를 별도로 처리해서 재고에서만 차감하고 싶다. 매출에는 안 잡히면서 재고 정확도를 유지할 수 있다.

영업팀

영업팀 담당자로서, 병원 방문 후 사용량을 웹에 입력하면 마감 원장에 자동 반영되길 원한다. 경영지원팀에 따로 알려줄 필요 없이 입력하면 바로 정산에 반영된다.

8. 비즈니스 규칙

발주 규칙

BR규칙명상세
01택배 마감 시간오후 15:30 이후 접수 발주는 익일 출고. 마감 시간은 관리자 설정 가능.
02발주 수정 제한출고 상태가 "대기" 또는 "바코드 출력" 단계일 때만 수정 가능. "포장" 이후 수정 불가.
03할인율 적용 우선순위거래처별 고정단가 > 거래처별 제품군 할인율 > 기본 단가. 고정단가가 있으면 할인율 무시.
04무료배송 기준영업팀 거래처는 전건 무료배송. 개인 거래처는 제품군별 최소 수량 기준. 기준 미달 시 배송비 부과.
05배송지 ≠ 업체 주소대리점이 병원으로 직송 시, 거래명세서 비고란에 실제 배송 병원명 자동 기입.

정산 규칙

BR규칙명상세
06거래처별 마감 기간대리점: 1일~25일 (또는 말일). 병원: 전달 26일~당월 25일. 거래처별 개별 설정 가능.
07결제 방식별 처리계좌이체 → 계산서 발행. 카드 결제 → 계산서 미발행. 사용량 결제 → 영업팀 사용량 입력 후 정산.
08사용량 결제 병원영업팀이 사용량 입력 전까지 해당 병원 마감 원장에 금액 미표시. 입력 후 자동 반영.
09거래명세서 양식거래처 설정에 따라 "거래처 양식" 또는 "RTBIO 양식" 중 선택. 동봉 여부도 거래처별 설정.

재고 규칙

BR규칙명상세

11샘플 출고출고 유형 "샘플"은 매출 미반영 + 재고만 차감 + 원가 기록 남김.
