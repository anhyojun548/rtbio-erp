## 4. 거래처 포털 (CLIENT)

거래처(대리점·병원) 담당자가 직접 사용하는 발주 전용 포털입니다. 제품을 엑셀 형태의 표에서 수량으로 발주하고, 발주 내역과 출고 상태를 확인하며, 알티바이오가 보낸 공지사항을 받아봅니다. 이 가이드는 개발 지식이 없는 실제 업무 사용자가 그대로 따라 하며 합격/불합격을 판정할 수 있도록 작성되었습니다.

### 접속 및 로그인 정보

| 항목 | 값 |
| --- | --- |
| 접속 주소(URL) | `http://localhost:3000/client` |
| 권한(역할) | CLIENT (거래처 담당자) |
| 로그인 계정 | `{거래처코드}@sample.local` (예: 거래처코드가 `medipharm`이면 `medipharm@sample.local`) |
| 비밀번호 | 관리자가 안내한 비밀번호 |

> 참고: 이 포털은 로그인 후 단일 화면(SPA) 안에서 여러 페이지가 전환되는 구조입니다. 페이지 이동은 화면 하단의 5개 탭(홈·발주·내역·공지·계정) 또는 화면 안의 버튼으로 이루어집니다. PC 화면에서는 상단에 가로 메뉴(홈·발주하기·발주내역·공지사항)가 함께 보일 수 있습니다.

### 발견된 메뉴 / 페이지 전체 목록

| 화면 ID | 이름 | 접근 방법 |
| --- | --- | --- |
| `page-loading` | 로딩 화면 | 접속 직후 자동 표시("거래처 정보를 불러오는 중...") |
| `page-login` | 로그인 화면 | 세션 없을 때 표시(정상 접근 시 미들웨어가 대신 처리) |
| `page-dashboard` | 홈(대시보드) | 하단 탭 "홈" / 상단 "홈" |
| `page-order` | 발주하기(발주폼) | 하단 탭 "발주" / 상단 "발주하기" / 홈의 "발주하기" 버튼 |
| `page-confirm` | 발주 확인 | 발주폼에서 "발주하기" 버튼 클릭 시 진입 |
| `page-history` | 발주 내역 | 하단 탭 "내역" / 상단 "발주내역" / 홈의 "발주내역" 버튼 |
| `page-status` | 출고 상태(주문 상세) | 발주 내역 또는 홈 최근발주의 주문 카드 클릭 |
| `page-notice` | 공지사항 | 하단 탭 "공지" / 상단 "공지사항" / 홈 공지 미니영역 |
| `page-account` | 내 계정 | 하단 탭 "계정" / 우측 상단 사용자 버튼(이름 첫 글자) |

주요 사용자 액션: 로그인 / 로그아웃, 카테고리 빠른 이동, 수량 입력(엑셀 표), 무료배송 안내 확인, 발주 확인 진입, 배송지 선택·신규 배송지 추가·임시 배송지 입력, 배송 방법 선택, 비고 입력, 발주 제출, 발주 내역 기간 필터(이번달·1개월·3개월·전체·직접설정), 주문 상세 보기, 발주 취소 요청, 공지사항 열람(읽음 처리·미읽음 배지·팝업·다시 보지 않기).

---

### 4.1 로그인

- **목적**: 거래처 계정으로 포털에 정상 진입할 수 있는지 확인합니다.
- **사전조건**: 관리자가 발급한 거래처 계정 1개(`{거래처코드}@sample.local` + 비밀번호)가 준비되어 있어야 합니다.
- **테스트 단계**:
  1. 브라우저에서 `http://localhost:3000/client` 에 접속합니다.
  2. 접속 직후 "거래처 정보를 불러오는 중..." 로딩 화면이 잠깐 보이는지 확인합니다.
  3. (로그인 화면이 표시되면) "아이디" 칸에 거래처 아이디, "비밀번호" 칸에 관리자가 안내한 비밀번호를 입력합니다.
  4. "로그인" 버튼을 클릭합니다. (또는 비밀번호 칸에서 Enter 키)
- **기대 결과**: 정상 계정이면 로딩 후 자동으로 "홈(대시보드)" 화면이 표시됩니다. 상단에 거래처명/담당자 인사말("안녕하세요, OOO님")과 회사명이 보이고, 하단에 5개 탭(홈·발주·내역·공지·계정)이 나타납니다.
- **Pass 기준**: 올바른 계정으로 홈 화면 진입에 성공하고 거래처명·담당자명이 본인 정보와 일치하면 합격. 아이디/비밀번호가 틀린 경우 "아이디 또는 비밀번호가 올바르지 않습니다." 안내가 나타나면 합격(잘못된 로그인 차단 확인).

### 4.2 로그아웃

- **목적**: 로그아웃이 정상 동작하고 세션이 종료되는지 확인합니다.
- **사전조건**: 4.1 로그인 완료 상태.
- **테스트 단계**:
  1. 하단 탭 "계정"을 클릭해 "내 계정" 화면으로 이동합니다. (또는 우측 상단의 이름 첫 글자 버튼을 눌러 "계정 정보" 팝업 → "로그아웃")
  2. 화면 하단의 빨간색 테두리 "로그아웃" 버튼을 클릭합니다.
- **기대 결과**: 세션이 종료되고 로그인 화면(`/login`)으로 이동합니다.
- **Pass 기준**: 로그아웃 후 로그인 페이지로 이동하고, 뒤로가기 등으로 다시 포털에 들어가도 자동 로그인되지 않으면 합격.

### 4.3 홈(대시보드) — 발주 요약 / 바로가기

- **목적**: 홈 화면에 이번 달 발주 건수·금액, 최근 발주, 공지 요약이 올바르게 표시되는지 확인합니다.
- **사전조건**: 로그인 완료. 가능하면 해당 거래처로 발주 이력이 1건 이상 있는 상태.
- **테스트 단계**:
  1. 하단 탭 "홈"을 클릭합니다.
  2. 상단 인사말("안녕하세요, OOO님")과 회사명을 확인합니다.
  3. "OOOO년 O월 발주" / "OOOO년 O월 금액" 두 개의 통계 카드를 확인합니다.
  4. "공지사항" 미니 영역과 "전체보기 ›" 링크를 확인합니다.
  5. "최근 발주" 영역에서 최근 주문 카드(최대 3건)를 확인합니다.
  6. 상단 박스 안의 "발주하기" / "발주내역" 바로가기 버튼을 각각 클릭해 해당 화면으로 이동하는지 확인합니다.
- **기대 결과**: 발주 건수/금액 카드에 실제 데이터가 표시됩니다(발주 이력이 없으면 "0건", "₩0"). 최근 발주 영역에는 주문번호·상태 뱃지·품목 요약·금액·일시가 표시되며, 발주 이력이 없으면 "발주 내역이 없습니다" 안내가 나옵니다. 바로가기 버튼은 각각 발주폼/발주내역으로 이동합니다.
- **Pass 기준**: 통계 카드 숫자가 실제 발주 데이터와 일치하고, 바로가기 버튼이 의도한 화면으로 이동하면 합격.

### 4.4 발주폼 — 마감 시간 안내 바

- **목적**: 발주폼 상단의 택배 마감 시간 안내가 표시되는지 확인합니다.
- **사전조건**: 로그인 완료.
- **테스트 단계**:
  1. 하단 탭 "발주"를 클릭합니다.
  2. 화면 상단의 시계 아이콘이 있는 시간 바를 확인합니다.
- **기대 결과**: "현재 OO:OO | 택배 마감 OO:OO (O시간 OO분 남음)" 형식으로 현재 시각·마감 시각·남은 시간이 표시됩니다. 마감이 임박/지났는지에 따라 바 색상이 달라질 수 있습니다.
- **Pass 기준**: 현재 시각과 마감 시각, 남은 시간이 표시되면 합격.

### 4.5 발주폼 — 카테고리 빠른 이동

- **목적**: 카테고리 칩(무릎·상지·하지·스프린트)을 눌러 해당 제품 표로 스크롤 이동하는지 확인합니다.
- **사전조건**: 발주폼("발주") 화면 진입.
- **테스트 단계**:
  1. 발주폼 상단의 카테고리 칩 줄에서 "상지" 칩을 클릭합니다.
  2. 화면이 상지 제품 섹션으로 부드럽게 스크롤되는지 확인합니다.
  3. 다른 카테고리(예: "스프린트")도 동일하게 확인합니다.
- **기대 결과**: 클릭한 카테고리의 제품 표가 화면 상단으로 스크롤되어 나타납니다.
- **Pass 기준**: 각 칩 클릭 시 해당 카테고리 섹션으로 이동하면 합격.

### 4.6 발주폼 — 수량 입력(엑셀 표)과 가격/소계 자동 계산

- **목적**: 사이즈별 수량 입력 시 행 소계와 주문 요약(소계·배송비·합계)이 즉시 갱신되는지 확인합니다.
- **사전조건**: 발주폼("발주") 화면 진입.
- **테스트 단계**:
  1. 원하는 제품 행에서 사이즈 칸(S/M/L/XL, 또는 Free 제품은 단일 칸)에 수량(예: 2)을 입력합니다.
  2. 입력한 칸의 색상이 강조되고, 해당 행의 "소계" 칸에 금액이 표시되는지 확인합니다.
  3. 단가 칸에 할인율(예: `-10%`)이 함께 표시되는 제품이 있는지 확인합니다(거래처별 고정가/카테고리 할인 적용 제품).
  4. PC 화면이면 우측 "주문 요약" 사이드바, 모바일 화면이면 표 아래 "주문 요약"과 하단 바에서 선택 품목 수·소계·배송비·합계가 갱신되는지 확인합니다.
  5. 수량을 0으로 지우면 해당 행 소계와 요약에서 빠지는지 확인합니다.
- **기대 결과**: 수량 입력 즉시 행 소계 = 단가 × 수량으로 계산되고, 주문 요약의 소계/합계가 모든 입력 품목을 더한 값으로 갱신됩니다. 단가는 거래처에 맞는 가격(고정가 > 카테고리 할인 > 기본가 순)으로 표시됩니다.
- **Pass 기준**: 행 소계와 요약 합계가 손계산과 일치하고, 수량 변경에 실시간 반영되면 합격. (도메인 규칙: 단가는 주문 시점 스냅샷으로, 제출 시 서버에서 다시 계산됩니다.)

### 4.7 발주폼 — 무료배송 안내 배너

- **목적**: 카테고리별 기준 수량에 따라 무료배송 달성/미달성 안내가 바뀌는지 확인합니다.
- **사전조건**: 발주폼 진입.
- **테스트 단계**:
  1. 수량을 입력하기 전 배너 문구(카테고리별 무료배송 기준 수량 안내)를 확인합니다.
  2. 특정 카테고리 제품 수량을 조금씩 늘립니다.
  3. 기준 수량 미만일 때 "OOO 제품 O개 더 추가하면 무료배송!" 문구가 나오는지 확인합니다.
  4. 기준 수량 이상으로 늘려 "OOO 무료배송 달성! (현재/기준 개)" 문구로 바뀌는지 확인합니다.
  5. 무료배송 달성 시 주문 요약의 "배송비"가 "무료"로 바뀌는지 확인합니다.
- **기대 결과**: 수량에 따라 배너 문구와 색상이 미달성↔달성으로 전환되고, 달성 시 배송비가 0(무료)으로 계산됩니다.
- **Pass 기준**: 기준 수량 경계에서 배너 문구·배송비가 올바르게 바뀌면 합격.

### 4.8 발주 확인 화면 진입

- **목적**: 발주폼에서 "발주하기" 클릭 시 발주 확인 화면으로 넘어가는지, 빈 주문은 막히는지 확인합니다.
- **사전조건**: 발주폼 진입.
- **테스트 단계**:
  1. (먼저) 아무 수량도 입력하지 않은 상태에서 "발주하기" 버튼을 클릭합니다.
  2. 다음으로 제품 1개 이상 수량을 입력한 뒤 "발주하기" 버튼을 클릭합니다.
- **기대 결과**: 수량이 없으면 "주문할 제품을 선택해주세요." 안내가 나오고 이동하지 않습니다. 수량이 있으면 "발주 확인" 화면으로 이동하며 주문 제품 표·합계가 표시됩니다.
- **Pass 기준**: 빈 주문은 차단되고, 정상 주문은 확인 화면으로 넘어가면 합격.

### 4.9 발주 확인 — 주문 제품 / 합계 검증

- **목적**: 발주 확인 화면의 제품 목록·단가·수량·금액·합계가 발주폼 입력과 일치하는지 확인합니다.
- **사전조건**: 4.8을 통해 발주 확인 화면 진입.
- **테스트 단계**:
  1. "주문 제품" 표에서 제품명·사이즈·단가·수량·금액 행을 발주폼 입력과 비교합니다.
  2. 표 하단 "합계 금액"을 확인합니다.
  3. 화면 하단 "최종 결제 금액"과 배송비 포함/무료 안내를 확인합니다.
- **기대 결과**: 표의 각 행 금액 = 단가 × 수량이고, 합계 금액 = 모든 행 금액의 합, 최종 결제 금액 = 합계 + 배송비(무료면 합계와 동일)입니다.
- **Pass 기준**: 표시된 모든 금액이 손계산과 일치하면 합격.

### 4.10 발주 확인 — 배송지 선택

- **목적**: 등록된 배송지 중 하나를 선택할 수 있고 기본 배송지가 기본 선택되는지 확인합니다.
- **사전조건**: 발주 확인 화면 진입. 거래처에 배송지가 1개 이상 등록되어 있으면 더 좋습니다.
- **테스트 단계**:
  1. "배송지 선택" 영역에서 등록된 배송지 목록(별칭·주소·수령인·연락처·메모)을 확인합니다.
  2. "기본" 뱃지가 붙은 배송지가 처음에 선택(라디오 체크)되어 있는지 확인합니다.
  3. 다른 배송지를 클릭해 선택을 변경합니다.
- **기대 결과**: 배송지 목록이 표시되고, 기본 배송지가 우선 선택되어 있으며, 클릭으로 선택을 바꿀 수 있습니다. 등록된 배송지가 없으면 "등록된 배송지가 없습니다..." 안내가 나오고 임시 주소 입력이 활성화됩니다.
- **Pass 기준**: 기본 배송지 우선 선택 및 선택 변경이 정상 동작하면 합격.

### 4.11 발주 확인 — 신규 배송지 추가

- **목적**: "+ 신규 배송지 추가"로 배송지를 추가하고 목록에 반영·자동 선택되는지 확인합니다.
- **사전조건**: 발주 확인 화면 진입.
- **테스트 단계**:
  1. "배송지 선택" 영역 우측 상단 "+ 신규 배송지 추가" 버튼을 클릭합니다.
  2. 팝업에서 "배송지 별칭 *"(예: 강남 지점), "배송 주소 *"는 필수로 입력합니다. 수령인·연락처·메모는 선택 입력합니다.
  3. (선택) "기본 배송지로 설정" 체크박스를 켭니다.
  4. 팝업의 확인 버튼을 클릭합니다.
  5. 일부러 별칭이나 주소를 비운 채 저장을 시도해 봅니다.
- **기대 결과**: 별칭·주소가 모두 입력되면 새 배송지가 목록에 추가되고 자동 선택되며 "배송지 'OOO'을(를) 추가했습니다." 안내가 나옵니다. "기본 배송지로 설정"을 켜면 다른 배송지의 기본 표시가 해제됩니다. 별칭/주소가 비면 "별칭과 주소를 입력해주세요." 안내가 나오고 저장되지 않습니다.
- **Pass 기준**: 정상 입력 시 추가·자동선택되고, 필수값 누락은 차단되면 합격. (참고: 이 화면에서 추가한 배송지는 이번 주문 화면 기준으로 즉시 반영됩니다.)

### 4.12 발주 확인 — 이번 주문만 다른 주소(임시 배송지)

- **목적**: 저장하지 않고 이번 주문에만 사용할 임시 주소를 입력할 수 있는지 확인합니다.
- **사전조건**: 발주 확인 화면 진입.
- **테스트 단계**:
  1. "이번 주문만 다른 주소로 배송 (저장 안 함)" 라디오를 선택합니다.
  2. 아래에 나타나는 입력란(받는 분·연락처·배송 주소·배송 메모)에 값을 입력합니다.
- **기대 결과**: 라디오 선택 시 임시 주소 입력란이 펼쳐지고, 다른 배송지를 선택하면 다시 접힙니다. 입력한 주소는 이 주문에만 사용됩니다.
- **Pass 기준**: 임시 주소 입력란이 토글되고 입력이 가능하면 합격. (도메인 규칙: 임시 주소는 배송지로 저장되지 않고 주문 스냅샷으로만 보관됩니다.)

### 4.13 발주 확인 — 배송 방법 / 비고 입력

- **목적**: 배송 방법 선택과 비고 입력이 가능한지 확인합니다.
- **사전조건**: 발주 확인 화면 진입.
- **테스트 단계**:
  1. "배송 방법"에서 택배(기본)·방문수령·퀵(추가비용 별도) 중 하나를 선택합니다.
  2. "비고 / 메모" 칸에 배송 요청사항을 입력합니다.
- **기대 결과**: 라디오로 배송 방법을 변경할 수 있고, 비고 칸에 자유롭게 텍스트를 입력할 수 있습니다.
- **Pass 기준**: 배송 방법 선택과 비고 입력이 정상 동작하면 합격.

### 4.14 발주 제출

- **목적**: 발주가 서버에 정상 등록되고 발주번호가 발급되는지 확인합니다.
- **사전조건**: 4.9~4.13 완료(제품·배송지·배송 방법 선택). 배송지는 반드시 주소가 채워져 있어야 합니다.
- **테스트 단계**:
  1. (먼저) 배송지를 선택하지 않거나 임시 주소를 비운 채 "발주 제출"을 눌러 봅니다.
  2. 정상적으로 배송지를 지정한 뒤 화면 하단 "발주 제출" 버튼을 클릭합니다.
  3. 버튼이 "발주 처리 중..."으로 바뀌며 중복 클릭이 막히는지 확인합니다.
  4. 완료 팝업의 내용을 확인합니다.
  5. 팝업 확인을 누른 뒤 이동한 화면을 확인합니다.
- **기대 결과**: 배송지가 비면 "배송지를 선택하거나 입력해주세요." 안내가 나오고 제출되지 않습니다. 정상 제출 시 "발주가 성공적으로 접수되었습니다" 완료 팝업이 뜨고 품목 수·배송방법·합계, 실제 "주문번호"(예: `ORD-...`), 배송지 정보, "택배 마감 15:30 이전 접수분은 당일 출고됩니다." 안내가 표시됩니다. 확인 후 발주 내역 화면으로 이동하며 방금 발주가 목록 최상단에 나타납니다.
- **Pass 기준**: 실제 주문번호가 발급되고 완료 팝업·발주 내역 반영이 확인되면 합격. 제출 실패 시 오류 메시지가 표시되고 버튼이 다시 활성화되면 합격(오류 처리 확인). (도메인 규칙: 제출 시 라인 단가는 서버에서 재계산되며 주문은 SUBMITTED 상태로 접수됩니다.)

### 4.15 발주 내역 — 목록 조회

- **목적**: 본인 거래처의 발주 내역이 최신순으로 표시되는지 확인합니다.
- **사전조건**: 발주 이력이 1건 이상 있는 상태.
- **테스트 단계**:
  1. 하단 탭 "내역"을 클릭합니다.
  2. 주문 카드 목록(주문번호·상태 뱃지·품목 요약·배송방법·금액·일시)을 확인합니다.
  3. 기간 라벨(예: "OOOO년 O월 · O건")의 건수가 카드 수와 일치하는지 확인합니다.
- **기대 결과**: 본인 거래처 주문만 최신순으로 표시되고, 각 카드에 주문번호·상태·금액·일시가 보입니다. 내역이 없으면 "해당 기간에 발주 내역이 없습니다" 안내가 나옵니다.
- **Pass 기준**: 본인 주문이 최신순으로 정확히 표시되면 합격.

### 4.16 발주 내역 — 기간 필터

- **목적**: 기간 프리셋(이번달·1개월·3개월·전체·직접설정)으로 목록이 필터링되는지 확인합니다.
- **사전조건**: 서로 다른 기간에 분포한 발주 이력이 여러 건 있는 상태가 이상적.
- **테스트 단계**:
  1. "내역" 화면 상단의 기간 칩에서 "이번달"(기본 선택)을 확인합니다.
  2. "1개월", "3개월", "전체" 칩을 차례로 클릭하며 목록과 기간 라벨·건수가 바뀌는지 확인합니다.
  3. "직접설정" 칩을 클릭하면 시작일·종료일 날짜 입력칸이 나타나는지 확인합니다.
  4. 시작일·종료일을 지정해 해당 범위만 표시되는지 확인합니다.
- **기대 결과**: 각 프리셋에 맞게 목록이 좁혀지고, 기간 라벨이 "최근 1개월 · O건", "전체 기간 · O건", "시작일 ~ 종료일 · O건" 등으로 갱신됩니다. 직접설정 시 입력한 날짜 범위 내 주문만 표시됩니다.
- **Pass 기준**: 선택한 기간 조건에 맞는 주문만 표시되고 건수 라벨이 일치하면 합격.

### 4.17 발주 내역 — 수정/마감 표시 (접수 상태)

- **목적**: 접수 상태 주문에 마감 시간 전후로 "수정"/"마감" 표시가 올바르게 나타나는지 확인합니다.
- **사전조건**: "접수" 상태 주문이 1건 이상.
- **테스트 단계**:
  1. "내역"에서 "접수" 상태 주문 카드를 확인합니다.
  2. 카드 우측 하단에 "수정"(마감 전) 또는 "마감"(마감 시간 경과) 표시가 있는지 확인합니다.
  3. "수정" 링크를 클릭해 봅니다.
- **기대 결과**: 마감 전이면 "수정" 링크, 마감 후면 취소선이 그어진 "마감" 표시가 나옵니다. "수정"을 누르면 "수정 기능은 데모에서 지원하지 않습니다" 안내가 나옵니다(현재 데모 한정).
- **Pass 기준**: 마감 시간 상태에 따라 수정/마감 표시가 적절히 나타나면 합격. (참고: 라벨이 데모 한정 동작이라는 점을 표시함.)

### 4.18 출고 상태(주문 상세) — 진행 단계 확인

- **목적**: 주문 카드를 눌러 출고 진행 상태(4단계)와 주문 상세를 볼 수 있는지 확인합니다.
- **사전조건**: 발주 이력 1건 이상.
- **테스트 단계**:
  1. "내역" 또는 "홈" 최근 발주에서 주문 카드를 클릭합니다.
  2. 상단에 주문번호·거래처·발주일시·배송방법·상태가 표시되는지 확인합니다.
  3. "출고 진행 상태"에서 4단계(접수 → 확인중 → 준비중 → 출고완료) 표시와 현재 단계 강조를 확인합니다.
  4. 송장번호(있는 경우)와 포장 정보(박스 수·무게, 있는 경우)를 확인합니다.
  5. "주문 상세" 표에서 제품·사이즈·단가·수량·금액·합계를 확인합니다.
  6. "발주 내역으로" 버튼으로 목록에 돌아갈 수 있는지 확인합니다.
- **기대 결과**: 주문 상세 정보와 4단계 진행 표시가 보이고, 현재 단계가 강조됩니다. 송장번호/포장 정보는 해당 데이터가 있을 때만 표시됩니다.
- **Pass 기준**: 주문 상세 정보·진행 단계가 주문 데이터와 일치하면 합격.

### 4.19 발주 취소 요청

- **목적**: 포장 단계 이전 주문을 거래처가 직접 취소 요청할 수 있는지 확인합니다.
- **사전조건**: 포장 단계 이전(접수/확인중 등) 주문이 1건 이상.
- **테스트 단계**:
  1. 취소 가능 주문의 상세(출고 상태) 화면으로 진입합니다.
  2. 하단의 "발주 취소 요청" 버튼(빨간색)을 확인합니다. ("포장 단계 전까지 취소 가능" 안내 동반)
  3. "발주 취소 요청"을 클릭하고, 취소 사유 입력창에 사유를 입력한 뒤 확인합니다.
  4. (대조) 포장 단계 이후 주문은 취소 버튼 대신 "포장 단계 이후로 발주 취소가 불가능합니다..." 안내가 나오는지 확인합니다.
- **기대 결과**: 사유를 입력하면 취소가 처리되어 "OOO 취소 완료 — 알티바이오 시스템에 반영되었습니다" 안내가 나오고 상태가 "취소"로 바뀝니다. 사유를 비우면 취소가 진행되지 않습니다. 포장 단계 이후 주문은 취소 버튼이 보이지 않습니다.
- **Pass 기준**: 취소 가능 주문은 사유 입력 후 취소되고 상태가 갱신되며, 취소 불가 주문은 차단되면 합격. (도메인 규칙: 취소 시 서버에서 SUBMITTED/CONFIRMED → CANCELLED로 전이되며 예약 재고는 자동 복원됩니다.)

### 4.20 공지사항 — 목록 열람 및 읽음 처리

- **목적**: 본인 거래처에 적용되는 공지를 열람하고 읽음 처리·미읽음 배지가 동작하는지 확인합니다.
- **사전조건**: 알티바이오가 등록한 공지가 1건 이상이고, 그 중 본인 거래처(전체/대리점/병원/특정 거래처 대상)에 해당하는 공지가 있는 상태.
- **테스트 단계**:
  1. 하단 탭 "공지"에 빨간 미읽음 배지(숫자)가 표시되는지 확인합니다.
  2. 하단 탭 "공지"를 클릭해 공지사항 목록으로 이동합니다.
  3. 공지 항목의 제목([고정]/[중요] 표시·NEW 배지), 작성자, 작성일, 만료일(있는 경우), 본문을 확인합니다.
  4. 공지를 본 뒤 하단 "공지" 탭의 미읽음 배지가 사라지거나 줄어드는지 확인합니다.
  5. 홈 화면 상단의 공지 미니 영역(상위 3건)과 "전체보기 ›" 링크도 확인합니다.
- **기대 결과**: 본인 대상 공지만 표시되며, 고정/중요 공지가 상단에 정렬됩니다. 페이지 진입 시 미읽음 공지가 읽음 처리되어 NEW 배지·하단 미읽음 배지가 갱신됩니다. 대상 공지가 없으면 "받은 공지사항이 없습니다." 안내가 나옵니다.
- **Pass 기준**: 본인 대상 공지가 정렬되어 표시되고, 열람 후 미읽음 배지가 정상 갱신되면 합격.

### 4.21 공지사항 — 로그인 직후 팝업 / 다시 보지 않기

- **목적**: 로그인 직후 우선순위가 높은 미읽음 공지가 팝업으로 뜨고 "다시 보지 않기"가 동작하는지 확인합니다.
- **사전조건**: 본인 거래처 대상 미읽음 공지가 1건 이상인 상태에서 새로 로그인.
- **테스트 단계**:
  1. 로그아웃 후 다시 로그인합니다.
  2. 로그인 직후(약간의 지연 후) 공지 팝업이 뜨는지 확인합니다.
  3. 팝업의 제목·작성자·날짜·본문을 확인합니다.
  4. (선택) "다시 보지 않기" 체크박스를 켜고 "확인"을 누릅니다.
- **기대 결과**: 가장 우선순위 높은(고정·중요 우선) 미읽음 공지가 팝업으로 표시됩니다. "다시 보지 않기"를 켜고 확인하면 해당 공지가 읽음 처리되어 다시 팝업되지 않습니다.
- **Pass 기준**: 미읽음 공지가 있을 때 팝업이 뜨고, "다시 보지 않기" 처리가 반영되면 합격. (참고: 미읽음 대상 공지가 없으면 팝업이 뜨지 않는 것이 정상입니다.)

### 4.22 내 계정 정보 확인

- **목적**: 내 계정 화면에 거래처 정보가 정확히 표시되는지 확인합니다.
- **사전조건**: 로그인 완료.
- **테스트 단계**:
  1. 하단 탭 "계정"을 클릭합니다.
  2. 거래처명·담당자·유형·이메일·전화·결제조건·마감기간·주소를 확인합니다.
- **기대 결과**: 본인 거래처 정보가 정확히 표시됩니다(미입력 항목은 "-"로 표시될 수 있음).
- **Pass 기준**: 표시된 거래처 정보가 실제 등록 정보와 일치하면 합격.

### 4.23 권한 / 접근 제어 (참고 확인)

- **목적**: CLIENT 권한 사용자가 본인 거래처 데이터만 보는지, 비정상 접근이 차단되는지 확인합니다.
- **사전조건**: 거래처 계정으로 로그인.
- **테스트 단계**:
  1. 발주 내역·홈에서 다른 거래처의 주문이 보이지 않는지 확인합니다.
  2. 공지사항에서 본인 대상이 아닌 공지가 보이지 않는지 확인합니다.
  3. (가능하면) 로그아웃 상태에서 `http://localhost:3000/client` 직접 접근 시 로그인 페이지로 이동하는지 확인합니다.
- **기대 결과**: 본인 거래처 데이터만 노출되며, 미인증 접근은 로그인 화면으로 유도됩니다. CLIENT가 아닌 사용자는 각자의 포털로 자동 이동됩니다.
- **Pass 기준**: 타 거래처 데이터 미노출 및 미인증 접근 차단이 확인되면 합격.

### 4.24 스모크 체크리스트 (핵심 기능 빠른 점검)

- [ ] `/client` 접속 후 거래처 계정으로 로그인되어 홈 화면이 표시된다.
- [ ] 홈에 이번 달 발주 건수·금액, 최근 발주, 공지 미니 영역이 표시된다.
- [ ] "발주" 화면 상단에 택배 마감 시간 안내 바가 보인다.
- [ ] 카테고리 칩(무릎·상지·하지·스프린트) 클릭 시 해당 제품 섹션으로 이동한다.
- [ ] 제품 수량 입력 시 행 소계와 주문 요약(소계·배송비·합계)이 즉시 갱신된다.
- [ ] 거래처별 단가(고정가/할인)가 단가 칸에 반영되어 표시된다.
- [ ] 무료배송 기준 수량 도달 시 배너가 "달성"으로 바뀌고 배송비가 무료가 된다.
- [ ] 빈 주문으로 "발주하기" 시 차단되고, 정상 주문은 발주 확인 화면으로 넘어간다.
- [ ] 발주 확인 화면의 제품·단가·수량·금액·합계가 입력값과 일치한다.
- [ ] 배송지 선택(기본 우선)·신규 배송지 추가·임시 주소 입력이 동작한다.
- [ ] 배송 방법 선택과 비고 입력이 가능하다.
- [ ] "발주 제출" 시 실제 주문번호가 발급되고 완료 팝업이 뜬다.
- [ ] 제출 후 발주 내역 최상단에 방금 발주가 나타난다.
- [ ] 발주 내역 기간 필터(이번달·1개월·3개월·전체·직접설정)가 동작한다.
- [ ] 주문 카드 클릭 시 출고 진행 4단계와 주문 상세가 표시된다.
- [ ] 포장 단계 이전 주문에서 "발주 취소 요청"이 사유 입력 후 처리된다.
- [ ] 공지사항 목록·읽음 처리·미읽음 배지·로그인 팝업이 동작한다.
- [ ] 내 계정 화면에 거래처 정보가 정확히 표시된다.
- [ ] 로그아웃 시 로그인 페이지로 이동하고 세션이 종료된다.
