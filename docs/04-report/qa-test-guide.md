## 0. 시작하기 전에 (공통)

본 문서는 RTBIO ERP의 **사용자 인수 테스트(UAT)** 가이드입니다. 실제 업무 담당자가 직접 화면을 보며 따라 하실 수 있도록 작성되었습니다. 화면이 설명과 다르거나 동작하지 않으면 **버그**입니다. 아래 "버그 리포트 양식"에 맞춰 기록해 주세요.

---

### 0.1 이 문서의 목적과 범위

- **목적**: 실서비스 전환 전에, 5개 포털의 핵심 업무 기능이 실제로 동작하는지 사용자가 직접 검증합니다.
- **범위**: 5개 포털 전체
  - **경영지원 포털** (`/admin`) — 주문·재고·거래명세서·수금·원장·보고서 등 백오피스 전반
  - **영업 포털** (`/exec`) — 담당 거래처/주문 조회, 학회 방명록, 담당자 매출
  - **품질관리(QC) 포털** (`/qc`) — 출고 칸반, 출고 내역, 샘플 출고, 재고·유통기한 관리
  - **거래처 포털** (`/client`) — 대리점이 직접 발주·명세서·수금·계약서를 확인
  - **대표(CEO) 포털** (`/ceo`) — 위젯 기반 경영 대시보드
- **이 섹션(0번)** 은 공통 안내 + **포털을 넘나드는 통합 시나리오** 를 다룹니다. 각 포털별 상세 테스트는 1~5번 섹션을 참고하세요.

### 0.5 ⚠️ 데이터 주의 — 자유롭게 테스트하세요

현재 데이터베이스에는 두 종류의 데이터가 섞여 있습니다.

| 구분 | 내용 | 테스트 시 주의 |
| --- | --- | --- |
| **실제 이관 데이터** | 거래처·제품·과거 거래원장 등 **약 4만 건** | 조회는 자유. **삭제/대량 수정은 지양** (데모용으로 보존) |
| **가공(샘플) 데이터** | 테스트용 주문·거래명세서·수금 | 마음껏 생성/수정/삭제해도 됨 |

> **핵심**: 실서비스 전환 시 가공 데이터는 **클린스타트(초기화)** 됩니다. 즉, **이번 테스트로 만든 주문·거래명세서·수금은 전환 시점에 함께 정리** 되므로, **부담 없이 자유롭게 발주·확정·출고·발급·수금 전 과정을 테스트** 하셔도 됩니다. "데이터가 더러워질까 봐" 걱정하실 필요 없습니다.

### 0.6 공통 규칙 — Pass / Fail 표기 & 버그 리포트

#### 결과 표기법

각 테스트 단계마다 결과를 한 가지로 표기합니다.

| 표기 | 의미 |
| --- | --- |
| **Pass** ✅ | 기대 결과대로 정상 동작함 |
| **Fail** ❌ | 기대 결과와 다름 (→ 버그 리포트 작성) |
| **N/A** ➖ | 해당 환경에서 테스트 불가 / 대상 아님 |
| **보류** ⏸️ | 판단 애매 — 비고에 사유 기록 후 담당자 문의 |

#### 버그 리포트 양식 (Fail 시 반드시 기록)

| 번호 | 항목 | 작성 예시 |
| --- | --- | --- |
| ① | **위치** (포털 / 메뉴 / URL) | 경영지원 포털 / 거래명세서 / `/admin/invoices/xxx` |
| ② | **재현 단계** | 1) admin 로그인 → 2) 주문 상세에서 "거래명세서 발급" 클릭 → 3) … |
| ③ | **기대 결과** | DRAFT 거래명세서가 생성되고 합계에 VAT 10%가 더해져야 함 |
| ④ | **실제 결과** | 합계가 공급가액과 동일하게 표시됨 (VAT 누락) |
| ⑤ | **스크린샷 / 계정** | (화면 캡처 첨부) / `admin@rtbio.com` 사용 |

> 가능하면 ⑤ 스크린샷을 꼭 첨부해 주세요. 사용한 계정·발생 시각도 함께 적으면 재현이 빨라집니다.

## 0.7 크로스-포털 통합 E2E 시나리오 (가장 중요)

하나의 발주가 **거래처 → 품질관리 → 경영지원 → 대표/영업** 포털을 거치며 어떻게 흐르는지를 끝까지 검증합니다. **포털을 옮길 때마다 로그아웃 후 해당 계정으로 다시 로그인** 하세요. (또는 시크릿 창/다른 브라우저를 병행하면 편합니다.)

### 시나리오 개요 (업무 흐름)

```
[거래처]          [품질관리]                         [경영지원]                    [대표/영업]
발주 등록   →   발주 확정 → 칸반 출고단계 이동 → 출고완료   →   거래명세서 발급 → 수금 등록 → 원장   →   대시보드 매출 반영
(OPEN)        (CONFIRMED)   (SHIPPING)        (COMPLETED)      (ISSUED)        (수금)     (잔액)        (집계 확인)
```

> **테스트 전 준비**: 발주에 쓸 거래처/제품/사이즈를 미리 1개 정해 두세요. 본 시나리오는 거래처 **서울메디칼 대리점(C-AGEN-001)**, 제품 **인공무릎 보조재 A(P-KNEE-01)**, 사이즈 **M**, 수량 **2** 를 예시로 사용합니다. 출고 단계에서 **재고 차감**을 확인하기 위해, 시작 전에 경영지원 포털 `재고 현황`에서 해당 제품·사이즈의 **현재 가용재고·실물재고 수치를 메모**해 두세요.

### STEP 1 — 거래처 포털: 발주 등록

| 항목 | 내용 |
| --- | --- |
| **포털** | 거래처 포털 (`/client`) |
| **로그인 계정** | `c-agen-001@client.local` / `rtbio1234!` (서울메디칼 대리점) |
| **절차** | 1) 거래처 포털 로그인 → 2) 좌측 `발주` 메뉴 → 3) "신규 발주" → 4) 제품 **P-KNEE-01**, 사이즈 **M**, 수량 **2** 추가 → 5) 배송지 선택(기본 배송지) → 6) 발주 제출(또는 저장 후 제출) |
| **기대 결과** | • 발주가 목록에 나타나고 상태가 **진행중(OPEN/SUBMITTED)** 으로 표시됨<br>• 라인 단가가 거래처 할인(대리점 15%) 또는 고정가 규칙에 따라 계산되어 보임<br>• 주문번호(예: `ORD-YYYYMMDD-NNN` 또는 임시 `DRAFT-xxx`)가 부여됨 |
| **확인 포인트** | 발주 직후 **재고는 아직 차감되지 않음**(주문 확정 전). 가용재고는 그대로여야 정상. |

### STEP 2 — 품질관리 포털: 발주 확정 (예약/RESERVE)

| 항목 | 내용 |
| --- | --- |
| **포털** | 품질관리 포털 (`/qc`) — *경영지원 `/admin` 에서도 동일 동작 가능* |
| **로그인 계정** | `qc@rtbio.com` / `rtbio1234!` (또는 `admin@rtbio.com`) |
| **절차** | 1) 로그인 → 2) 주문 목록에서 STEP 1 발주 찾기 → 3) 상세 진입 → 4) "확정(CONFIRM)" 클릭 |
| **기대 결과** | • 주문 상태가 **확정(CONFIRMED)** 으로 전이됨<br>• **가용재고(availableStock) 가 주문 수량(2)만큼 차감** 됨<br>• 실물재고(physicalStock) 는 아직 그대로 (예약 단계이므로) |
| **확인 포인트** | STEP 0에서 메모한 가용재고와 비교 → **가용재고 −2** 인지 확인. 재고 부족 시 확정이 거부되면(롤백) 그 또한 정상 동작. |

### STEP 3 — 품질관리 포털: 칸반 출고 단계 이동

| 항목 | 내용 |
| --- | --- |
| **포털** | 품질관리 포털 (`/qc/shipments`) |
| **로그인 계정** | `qc@rtbio.com` / `rtbio1234!` |
| **절차** | 1) `출고 칸반` 메뉴 → 2) 확정된 주문 카드를 첫 단계(**접수대기**)에서 시작 → 3) 카드를 단계별로 이동: **접수대기 → 피킹 → 검수 → 포장 → 출고대기** |
| **기대 결과** | • 카드가 이동한 단계 컬럼으로 옮겨지고, 각 이동이 단계 이력으로 기록됨<br>• 보류(Hold)/재개(Resume) 버튼이 동작함<br>• 아직 **출고완료(COMPLETED) 전이므로 실물재고는 변동 없음** |
| **확인 포인트** | 단계 이동 중에는 재고가 줄지 않음(가용재고 −2 상태 유지). |

| --- | --- |
| **포털** | 품질관리 포털 (`/qc/shipments`) |
| **로그인 계정** | `qc@rtbio.com` / `rtbio1234!` |

### 1.1 로그인 / 로그아웃 / 비밀번호 변경

- **목적**: 올바른 계정으로 포털에 접속하고, 본인 비밀번호를 변경하고, 안전하게 로그아웃할 수 있는지 확인합니다.
- **사전조건**: 관리자가 안내한 `admin@rtbio.com` 계정과 비밀번호.
- **테스트 단계**:
  1. 브라우저에서 `http://localhost:3000/admin` 으로 접속합니다.
  2. 로그인 화면에서 이메일 `admin@rtbio.com`, 비밀번호(관리자 안내값)를 입력하고 로그인합니다.
  3. 로그인되면 왼쪽 사이드바 상단에 사용자명("김소영", 직책 "경영지원팀 팀장")이 표시되는지 확인합니다.
  4. 사이드바 상단 사용자 영역 아래의 "🔑 비밀번호 변경" 링크를 클릭합니다.
  5. 비밀번호 변경 창에서 안내에 따라 새 비밀번호를 입력하고 저장합니다.
  6. 다시 사이드바의 "↩ 로그아웃" 링크를 클릭합니다.
- **기대 결과**:
  - 잘못된 비밀번호면 로그인되지 않고 오류 안내가 표시됩니다.
  - 올바른 계정이면 대시보드가 열립니다.
  - 비밀번호 변경 시 성공 메시지가 표시됩니다.
  - 로그아웃하면 로그인 화면(또는 시작 화면)으로 돌아갑니다.
- **Pass 기준**: 로그인/로그아웃이 정상 동작하고, 비밀번호 변경 후 새 비밀번호로 다시 로그인이 됩니다.

### 1.2 대시보드 — 위젯 조회

- **목적**: 첫 화면의 경영 지표(위젯)들이 정상적으로 표시되는지 확인합니다.
- **사전조건**: 로그인 완료.
- **테스트 단계**:
  1. 사이드바 "대시보드"를 클릭합니다.
  2. 화면에 "경영지원 대시보드" 제목과 여러 개의 카드(위젯)가 표시되는지 확인합니다.
  3. 빈 화면이거나 "불러오는 중…"에서 멈추지 않는지 확인합니다.
- **기대 결과**: 매출/미수금/주문 등 위젯들이 숫자나 표/그래프로 채워집니다.
- **Pass 기준**: 위젯이 깨지지 않고 데이터가 표시됩니다(0이어도 무방, "오류"만 아니면 됨).

### 1.3 대시보드 — 기간 필터(전역 기간)

- **목적**: 상단 기간 선택이 위젯 데이터에 반영되는지 확인합니다.
- **사전조건**: 대시보드 화면.
- **테스트 단계**:
  1. 대시보드 우측 상단 "기간" 옆 드롭다운에서 "오늘 / 최근 7일 / 최근 30일 / 이번 달" 을 차례로 바꿔봅니다.
  2. "커스텀"을 선택하면 시작일/종료일 날짜 입력칸이 나타나는지 확인하고, 날짜를 직접 지정해 봅니다.
- **기대 결과**: 기간을 바꾸면 위젯 숫자가 다시 계산되어 갱신됩니다. "커스텀" 선택 시 날짜 입력칸이 나타납니다.
- **Pass 기준**: 기간 변경 시 화면이 갱신되고 오류가 없습니다.

### 1.4 대시보드 — 위젯 추가(갤러리/직접 만들기)

- **목적**: 새 위젯을 대시보드에 추가할 수 있는지 확인합니다.
- **사전조건**: 대시보드 화면.
- **테스트 단계**:
  1. 우측 상단 "➕ 위젯 추가" 버튼을 클릭합니다.
  2. "위젯 추가" 창이 열리고 "갤러리에서 선택" 영역에 위젯 목록이 표시되는지 확인합니다.
  3. 갤러리에서 위젯 하나를 선택해 추가합니다.
  4. (선택) "+ 직접 만들기" 버튼을 눌러 위젯 빌더 창을 엽니다. 제목 입력, 데이터 소스/차트 종류/측정값을 골라 미리보기가 나오는지 확인 후 "저장"합니다.
  5. (선택) "🤖 AI에게 말하기" 영역에 원하는 위젯을 자연어로 입력해 봅니다.
- **기대 결과**: 추가한 위젯이 대시보드에 새 카드로 나타납니다. 빌더에서는 소스/측정값 선택 시 미리보기가 표시됩니다.
- **Pass 기준**: 위젯 추가 후 대시보드에 카드가 추가되어 보입니다.

### 1.5 대시보드 — 위젯 수정/복제/새로고침/삭제

- **목적**: 개별 위젯의 우클릭(또는 메뉴) 동작을 확인합니다.
- **사전조건**: 위젯이 1개 이상 있는 대시보드.
- **테스트 단계**:
  1. 위젯 카드의 메뉴(점 3개 ⋮ 또는 우클릭)를 엽니다.
  2. "✏ 위젯 수정" → 제목 변경, 기간(override) 변경 후 "저장"합니다.
  3. 다시 메뉴에서 "복제"를 눌러 같은 위젯이 하나 더 생기는지 확인합니다.
  4. "새로고침"으로 데이터가 다시 불러와지는지 확인합니다.
  5. "삭제"로 위젯을 제거합니다.
- **기대 결과**: 수정 시 제목/기간이 바뀌고, 복제 시 카드가 늘어나며, 삭제 시 카드가 사라집니다.
- **Pass 기준**: 4개 동작(수정·복제·새로고침·삭제)이 각각 화면에 반영됩니다.

### 1.6 대시보드 — 레이아웃 내보내기/가져오기/초기화

### 1.7 매입매출장 — 매출장/매입장 조회

- **목적**: 기간·거래처·담당자별 매출/매입 내역을 표로 조회할 수 있는지 확인합니다.
- **사전조건**: 로그인 완료. 조회할 거래 데이터가 있을 것(시드 데이터 존재).
- **테스트 단계**:
  1. 사이드바 "매입매출장"을 클릭합니다.
  2. 상단 탭에서 "매출장"이 기본 선택되어 있는지 확인하고, "매입장" 탭으로도 전환해 봅니다.
  3. "시작일", "종료일"을 지정합니다.
  4. "거래처", "담당자" 드롭다운에서 특정 값을 선택합니다.
  5. "검색" 입력칸에 적요/품목 일부를 입력합니다.
  6. "조회" 버튼을 클릭합니다.
- **기대 결과**: 조건에 맞는 거래 목록이 표에 나타나고, 하단에 합계가 표시됩니다. 목록이 길면 페이지 이동(페이지네이션)이 보입니다.
- **Pass 기준**: 필터 조건대로 표가 갱신되고, 매출/매입 탭 전환이 정상 동작합니다.

### 1.8 매입매출장 — 엑셀 다운로드

- **테스트 단계**:
  1. 매입매출장 화면 하단의 "엑셀 다운로드" 버튼을 클릭합니다.
- **기대 결과**: "엑셀 파일이 다운로드되었습니다" 안내 메시지가 표시됩니다.
- **Pass 기준**: 버튼 클릭 시 다운로드 동작 또는 안내 메시지가 정상적으로 뜹니다.

### 1.9 거래처원장 — 거래처별 상세 원장 조회

- **목적**: 특정 거래처의 일자별 매출(차변)/수금(대변)/잔액 흐름을 조회합니다.
- **사전조건**: 등록된 거래처가 있을 것.
- **테스트 단계**:
  1. 사이드바 "거래처원장"을 클릭합니다.
  2. "거래처 검색" 칸에 거래처명 또는 코드 일부를 입력하면 드롭다운 목록이 뜹니다. 원하는 거래처를 선택합니다.
  3. "시작일", "종료일"을 지정합니다.
  4. "담당자" 드롭다운에서 필요 시 값을 선택합니다.
- **기대 결과**: 상단에 요약 카드(전월이월/매출/수금/잔액 등)가, 아래 표에 일자·구분·적요·차변(매출)·대변(수금)·잔액이 누적 형태로 표시됩니다.
- **Pass 기준**: 거래처 선택 시 해당 거래처 원장이 정확히 표시되고 잔액이 누적으로 계산됩니다.

### 1.10 거래처원장 — 인쇄 / 엑셀 다운로드

  1. 화면 하단 "인쇄" 버튼을 클릭합니다.

### 1.11 마감원장 — 기간별 정산 원장 조회 + 정렬

- **목적**: 모든 거래처의 전월이월/당월매출/수금/미수잔액을 한 표로 조회·정렬합니다.
- **사전조건**: 거래/수금 데이터 존재.
- **테스트 단계**:
  1. 사이드바 "마감원장"을 클릭합니다.
  2. "시작일", "종료일"을 지정하고 "조회"를 클릭합니다.
  3. "정렬" 드롭다운에서 "거래처명순 / 미수금 높은순 / 미수금 낮은순 / 매출 높은순"을 바꿔봅니다.
  4. 표 머리글의 "거래처 ↕", "미수잔액 ↕"를 클릭해 정렬이 토글되는지 확인합니다.
- **기대 결과**: 요약 카드와 표가 표시되고, 정렬 변경 시 행 순서가 바뀝니다. 거래처가 많으면 하단 페이지 이동이 나타납니다.
- **Pass 기준**: 조회·정렬이 정상 동작하고 합계가 맞게 표시됩니다.

### 1.12 마감원장 — 마감 확정

- **기대 결과**: 마감 처리 안내 메시지가 표시됩니다.
- **Pass 기준**: 버튼 클릭 후 마감 관련 안내/처리가 정상적으로 동작합니다.

- **Pass 기준**: 두 버튼 모두 안내 메시지가 정상적으로 뜹니다.

### 1.14 매출 보고서 — 기간별 매출 분석 조회

- **목적**: 기간/거래처 유형/담당자별 매출을 공문 형태 보고서로 조회합니다.
- **사전조건**: 매출 데이터 존재.
- **테스트 단계**:
  1. 사이드바 "매출 보고서"를 클릭합니다.
  2. "시작일", "종료일"을 지정합니다.
  3. "거래처 유형"(전체/대리점/병원/기타), "담당자"를 선택합니다.
  4. "조회" 버튼을 클릭합니다.
  5. 화면의 "빠른 월 선택(1~12월)" 영역에서 특정 월 버튼을 눌러 봅니다.
- **기대 결과**: 화면 중앙에 공문 형태의 보고서 영역이 채워지고, 월 빠른 선택을 누르면 해당 월 기준으로 보고서가 갱신됩니다.
- **Pass 기준**: 필터·월 선택에 따라 보고서가 갱신되고 매출 합계가 표시됩니다.

### 1.15 거래명세서 — 명세서 생성(미리보기)

- **목적**: 특정 거래처/기간으로 거래명세서를 생성하고 미리보기를 확인합니다.
- **사전조건**: 거래 내역이 있는 거래처 존재.
- **테스트 단계**:
  1. 사이드바 "거래명세서"를 클릭합니다.
  2. "거래처 검색/선택" 칸에 거래처명을 입력하면 드롭다운이 뜹니다. 거래처를 선택합니다.
  3. "시작일", "종료일"을 지정합니다.
  4. 필요 시 "담당자 검색" 드롭다운에서 담당자를 선택합니다.
  5. "명세서 생성" 버튼을 클릭합니다.
- **기대 결과**: 아래에 "거 래 명 세 서" 양식이 표시됩니다. 공급자(주식회사 알티바이오)/공급받는자 정보, 미수 요약(전월이월·당월매출·수금액·미수잔액), 품목 표, 공급가액·부가세(10%)·합계가 나타납니다.
- **Pass 기준**: 선택한 거래처의 품목과 합계(부가세 포함)가 정확히 계산되어 표시됩니다.

### 1.17 거래명세서 — 발행 이력 조회 / 상태 필터

- **목적**: 발행한 명세서들의 이력(미발송/발송완료/취소)을 조회·필터합니다.
- **사전조건**: 발행 이력이 있을 것.
- **테스트 단계**:
  1. "거래명세서" 화면 아래쪽 "발행 이력" 카드로 이동합니다.
  2. 상태 필터 드롭다운에서 "전체 상태 / 미발송 / 발송완료 / 취소"를 바꿔봅니다.
  3. 발행 이력 패널의 기간/검색 입력으로도 걸러봅니다.
  4. 발행 횟수가 2회 이상인 행의 "이력 보기" 링크를 클릭해 발행/발송 이력 팝업을 확인합니다.
- **기대 결과**: 상태에 맞는 명세서 목록이 표시되고, 명세서 번호/발행일/거래처/합계/상태/발송/횟수/액션 열이 보입니다.
- **Pass 기준**: 필터 변경 시 목록이 정확히 갱신되고, 이력 팝업이 열립니다.

---

### 1.18 거래명세서 — 재발행

- **목적**: 기존 명세서를 다시 발행하고 발행 횟수가 증가하는지 확인합니다. (실제 발행 처리)
- **사전조건**: 발행 이력에 취소가 아닌 명세서가 1건 이상 있을 것.
- **테스트 단계**:
  1. "발행 이력" 목록에서 한 행의 "재발행" 버튼을 클릭합니다.
- **기대 결과**: "거래명세서 ... 재발행 완료 (N회차)" 성공 메시지가 표시되고, 해당 행의 발행 횟수가 1 증가합니다. 발행일이 갱신됩니다.
- **Pass 기준**: 성공 메시지가 뜨고 발행 횟수가 올라갑니다(실패 시 빨간 오류 메시지).

---

### 1.19 거래명세서 — 발송 / 재발송 (발송완료 표시)

- **목적**: 명세서를 발송완료로 표시하고 발송일이 기록되는지 확인합니다. (실제 처리)
- **사전조건**: 상태가 "미발송(ISSUED)" 또는 "발송완료(SENT)"인 명세서.
- **테스트 단계**:
  1. "발행 이력"에서 대상 행의 "발송"(이미 발송된 건은 "재발송") 버튼을 클릭합니다.
  2. 확인 창이 뜨면 확인합니다.
- **기대 결과**: 상태가 "✓ 발송완료"로 바뀌고 발송일시가 표시됩니다. 발송 횟수가 증가합니다.
- **Pass 기준**: 발송 처리 후 상태/발송일/횟수가 정확히 갱신됩니다.

### 1.20 수금관리 — 현황 요약 확인

- **목적**: 수금 관련 요약 통계가 표시되는지 확인합니다.

  2. 상단의 요약 카드(통계)와 "수금 규칙" 안내(월 1~25일 발주 → 월말 일괄 입금)를 확인합니다.
- **기대 결과**: 요약 카드들이 숫자로 채워지고 안내 박스가 표시됩니다.
- **Pass 기준**: 통계 카드가 오류 없이 표시됩니다.

### 1.21 수금관리 — 은행 입금 수기 등록

- **목적**: 은행 입금 내역을 직접 입력해 등록할 수 있는지 확인합니다.

  2. 추가된 행에 거래처, 은행, 입금자명, 입금액, 일자, 참조/적요를 입력합니다.
  3. "입금 등록" 버튼을 클릭합니다.
- **기대 결과**: 입금이 등록되고, 아래 "최근 등록 내역(최대 10건)" 표에 새 항목이 나타납니다. 해당 거래처 미수금이 갱신됩니다.
- **Pass 기준**: 입력한 입금이 최근 등록 내역에 반영되고 성공 메시지가 표시됩니다.

### 1.22 수금관리 — 대리점별 미수금 / 미수금 Aging / 엑셀

- **목적**: 미수금 분포(대리점별, 연체 경과별)를 조회하고 엑셀로 받을 수 있는지 확인합니다.

  2. 각 패널의 기간/검색/페이지 이동 기능을 사용해 봅니다.
  3. 하단 "엑셀 다운로드" 버튼을 클릭합니다.
- **기대 결과**: 대리점별·경과기간별 미수 현황이 표시되고, 엑셀 다운로드 안내 메시지가 뜹니다.

### 1.23 미수금관리 — 전체 미수 KPI + 거래처 검색

- **목적**: 미수가 있는 거래처를 전용 화면에서 검색·조회합니다.

- **테스트 단계**:
  1. 사이드바 "미수금관리"를 클릭합니다.
  2. 상단 전체 미수 KPI 카드들을 확인합니다.
  3. "거래처명/코드" 검색칸에 거래처 일부를 입력해 드롭다운에서 선택합니다.
  4. "유형 필터"(전체/대리점/병원), "정렬"(미수금 높은순/연체 오래된순/거래처명순)을 변경합니다.
- **기대 결과**: 미수 거래처 목록이 조건대로 표시됩니다.
- **Pass 기준**: KPI/목록이 표시되고 필터·정렬이 정상 동작합니다.

---

### 1.24 미수금관리 — 거래처별 월별 분석

- **목적**: 검색한 거래처의 연도별·월별 매출/입금/미수와 상세 거래내역을 봅니다.
- **사전조건**: 1.23에서 거래처를 선택한 상태.
- **테스트 단계**:

- **Pass 기준**: 검색/필터에 따라 카드 목록이 정확히 줄어들거나 늘어납니다.

### 1.26 거래처관리 — 신규 거래처 등록

- **Pass 기준**: 등록 후 목록에 새 거래처가 보입니다(필수값 누락 시 오류 안내).

### 1.27 거래처관리 — 거래처 상세 보기

- **사전조건**: 거래처 1건 이상 존재.
- **테스트 단계**:

  3. (해당 거래처에 할인율 매트릭스가 있으면) 매트릭스 표가 보입니다.
- **기대 결과**: 거래처 상세 정보가 정확히 표시됩니다.
- **Pass 기준**: 상세 항목이 빠짐없이 표시됩니다.

### 1.28 거래처관리 — 거래처 수정 / 배송지 / 할인·고정가 / 메모

- **목적**: 거래처 정보, 복수 배송지, 할인율 매트릭스, 특이사항 메모를 수정할 수 있는지 확인합니다. (실제 저장)
- **사전조건**: 거래처 1건 이상 존재.
- **테스트 단계**:
  1. 거래처 상세에서 수정 진입(편집) 후 정보를 변경하고 저장합니다.
  2. 배송지 추가/삭제 기능으로 배송지를 1건 추가했다가 삭제해 봅니다.
  3. 할인율 매트릭스의 "편집" 버튼으로 할인율을 바꾼 뒤 저장합니다.
  4. 특이사항/경쟁업체 메모가 있으면 입력 후 저장합니다.
- **기대 결과**: 각 변경이 성공 메시지와 함께 반영됩니다. 배송지는 기본 배송지가 거래처당 1개만 유지됩니다.
- **Pass 기준**: 수정·배송지·할인·메모 변경이 저장되고 다시 열었을 때 유지됩니다.

### 1.29 발주관리 — 발주 목록 조회 / 상태 탭 / 필터

- **사전조건**: 발주 데이터 존재.
- **테스트 단계**:

  2. 상단 상태 탭 "전체 / 접수 / 확정 / 출고중 / 완료"를 차례로 클릭합니다.
  3. "전체 거래처" 드롭다운에서 특정 거래처를 선택합니다.
- **기대 결과**: 탭/필터에 맞는 발주가 표(발주번호·거래처·발주일·배송·금액·상태·담당자)로 표시되고 요약이 갱신됩니다.
- **Pass 기준**: 상태 탭·거래처 필터에 따라 목록이 정확히 바뀝니다.

### 1.30 발주관리 — 발주 상세 보기

- **목적**: 개별 발주의 품목·금액·배송 정보를 확인하고, 명세서/발주서로 연결합니다.
- **사전조건**: 발주 1건 이상.
- **테스트 단계**:
  1. 발주 목록에서 한 건을 클릭합니다.
  2. 상세 창에서 발주번호·상태·거래처·발주일·배송·품목 표·합계를 확인합니다.

- **기대 결과**: 발주 상세가 정확히 표시되고, "거래명세서 출력" 클릭 시 해당 거래처가 선택된 명세서 화면으로 이동합니다.
- **Pass 기준**: 상세 정보가 맞고, 두 연결 버튼이 정상 동작합니다.

### 1.31 품질 보고서 작성 — ISO 13485 양식 + AI 자동 채움

- **목적**: 품질 양식을 선택하고 AI 어시스턴트로 자동 채운 뒤 출력합니다.
- **사전조건**: 로그인 완료.
- **테스트 단계**:
  1. 사이드바 "품질 보고서 작성"을 클릭합니다.

  3. 우측 "AI 어시스턴트" 입력칸에 예시 문장(예: "메디팜 의료기에서 리코탭플러스 50개 주문 들어왔어")을 입력하고 전송(↑)합니다.
  4. 좌측 양식이 자동으로 채워지는지 확인합니다.
  5. "임시저장", "양식대로 인쇄", "PDF 출력" 버튼을 각각 눌러봅니다.
- **기대 결과**: 양식 미리보기가 표시되고, AI 응답으로 양식 일부가 채워지며, 저장/인쇄/PDF 버튼이 동작합니다.
- **Pass 기준**: 양식 선택·AI 채움·출력 버튼이 오류 없이 동작합니다.

### 1.33 데이터 탐색기 — 행 추가 / 편집 / 삭제

- **목적**: 데이터 행을 직접 추가·수정·삭제할 수 있는지 확인합니다. (편집 가능 테이블 한정)
- **사전조건**: 편집 권한이 있는 테이블 선택.
- **테스트 단계**:
  1. 편집 가능한 테이블을 선택합니다.
  2. "＋ 새 행" 버튼으로 우측 패널을 열고 값을 입력 후 "추가"합니다.

  4. 행 패널에서 "삭제" 버튼으로 행을 삭제합니다(확인 창 확인).
- **기대 결과**: 추가/저장/삭제 시 각각 성공 메시지가 표시됩니다. 읽기 전용 테이블이면 "읽기 전용입니다" 안내가 표시됩니다.
- **Pass 기준**: 추가·편집·삭제가 정상 동작하고, 읽기 전용 테이블은 보호됩니다.

## 2. 영업 포털 (EXEC)

> **대상 독자**: 알티바이오 영업팀 담당자 (UAT 수행자)
> **접속 URL**: `http://localhost:3000/exec`
> **로그인 계정**: `sales1@rtbio.com` / 비밀번호는 **관리자가 안내한 비밀번호**를 입력하세요.
> **권한(Role)**: EXEC (영업팀)

### 공통 안내
- 로그인 후 좌측 **사이드바**에 다음 메뉴가 보이면 정상입니다.
  - **영업** 그룹: `영업 대시보드`, `영업 현황`, `거래처 관리`, `담당자 관리`, `학회 관리`, `영업사원 계정`
  - **기타** 그룹: `📢 공지사항`, `사용량 입력`, `보고서 작성`, `데이터 탐색기`
  - (`직원 관리` 메뉴는 **팀 관리자/메타 관리자 권한이 있는 계정에서만** 보입니다. 일반 영업 계정은 보이지 않는 것이 정상입니다.)
- 사이드바 상단에 로그인 사용자명(`김영업` 등)과 역할(`영업팀`)이 표시됩니다.
- 사이드바 하단에 **모드 전환 드롭다운**(관리자/박진우/신현호/이경동)이 있습니다. 본 가이드의 일부 테스트는 이 모드 값에 따라 화면이 달라집니다.
- 화면 우하단(또는 상단)에 나타나는 작은 알림 메시지를 **토스트(toast)** 라고 부릅니다. 저장·삭제 등 동작 직후 토스트가 떠야 정상입니다.
- 본 포털은 프로토타입 단계로, 일부 데이터는 **새로고침 시 초기화**되는 Mock 데이터입니다(각 테스트에 별도 표시).

---

### 2.1 로그인 및 진입

- **목적**: 영업 포털에 정상 로그인하고 첫 화면(영업 대시보드)이 뜨는지 확인한다.
- **사전조건**: 관리자에게서 `sales1@rtbio.com` 계정의 비밀번호를 안내받았다.
- **테스트 단계**:
  1. 브라우저 주소창에 `http://localhost:3000/exec` 를 입력하고 이동한다.
  2. 로그인 화면이 나오면 아이디 `sales1@rtbio.com`, 비밀번호(관리자 안내값)를 입력하고 로그인한다.
  3. 좌측 사이드바와 우측 본문이 표시되는지 확인한다.
- **기대 결과**: 좌측 사이드바에 "영업/기타" 메뉴가 표시되고, 우측에 **영업 대시보드** 화면(상단 KPI 카드 4장 + 위젯 영역)이 나타난다.
- **Pass 기준**: 영업 대시보드가 로딩되고, 사이드바 사용자명/역할이 표시되면 합격. 로그인 실패하거나 빈 화면이면 불합격.

---

### 2.2 영업 대시보드 — 상단 KPI 카드

- **목적**: 월 매출/수금/미수금/담당 거래처 KPI가 실제 데이터로 표시되는지 확인한다.
- **사전조건**: 영업 포털 로그인 완료.
- **테스트 단계**:
  1. 좌측 메뉴 **`영업 대시보드`** 를 클릭한다.
  2. 상단 KPI 카드 4장을 확인한다: **월 매출 현황 / 월 수금액 현황 / 월 미수금 현황 / 담당 거래처**.
- **기대 결과**:
  - `월 매출 현황`: 최신 정산월의 누적 매출 금액(원)과 "YYYY-MM 누적" 부제가 표시된다.
  - `월 수금액 현황`: 해당 월 수금액과 부제가 표시된다.
  - `월 미수금 현황`: 미수금 합계와 "거래처 N개" 부제가 표시된다.
  - `담당 거래처`: 활성 거래처 수와 "대리점 N / 병원 N" 부제가 표시된다.
- **Pass 기준**: 4장 카드 모두 숫자(₩ 금액 또는 개수)가 채워지고 "—"나 깨진 값이 없으면 합격.

---

### 2.3 영업 대시보드 — 위젯 추가

- **목적**: 대시보드에 위젯을 새로 추가할 수 있는지 확인한다.
- **사전조건**: 영업 대시보드 화면.
- **테스트 단계**:
  1. 우측 상단 **`➕ 위젯 추가`** 버튼을 클릭한다.
  2. "위젯 추가" 모달이 열리면 **`갤러리에서 선택`** 영역에서 위젯 카드 하나를 클릭한다.
  3. 모달이 닫히면 대시보드 그리드에 새 위젯이 추가됐는지 확인한다.
- **기대 결과**: 선택한 위젯이 대시보드 그리드 영역에 카드 형태로 표시된다.
- **Pass 기준**: 위젯이 그리드에 나타나면 합격. 모달이 안 열리거나 추가가 반영되지 않으면 불합격.

---

### 2.4 영업 대시보드 — 위젯 직접 만들기 (커스텀 위젯)

- **목적**: 데이터 소스/집계/필터를 직접 지정해 커스텀 위젯을 만들 수 있는지 확인한다.
- **사전조건**: "위젯 추가" 모달이 열린 상태.
- **테스트 단계**:
  1. `➕ 위젯 추가` → "위젯 추가" 모달에서 **`+ 직접 만들기`** 버튼을 클릭한다.
  2. "위젯 직접 만들기" 모달에서 **위젯 제목**(예: 이번 달 매출)을 입력한다.
  3. **데이터 소스**와 **차트 종류**를 선택한다.
  4. **측정값(집계)**, 필요 시 **분류(그룹)**, **필터 조건**(`+ 조건` 버튼), **정렬/최대 행**을 설정한다.
  5. 우측 **미리보기** 영역에 실데이터 미리보기가 뜨는지 확인한다.
  6. 하단 **`저장`** 버튼을 클릭한다.
- **기대 결과**: 소스·측정값을 선택하면 우측 미리보기가 채워지고, 저장 시 대시보드에 새 위젯이 추가된다.
- **Pass 기준**: 미리보기가 표시되고 저장 후 위젯이 대시보드에 나타나면 합격.

---

### 2.5 영업 대시보드 — 위젯 수정/복제/새로고침/삭제 (위젯 메뉴)

- **목적**: 위젯별 컨텍스트 메뉴 동작을 확인한다.
- **사전조건**: 대시보드에 위젯이 1개 이상 있다.
- **테스트 단계**:
  1. 대시보드의 위젯에서 메뉴 버튼(⋮ / 우클릭 등)으로 **위젯 컨텍스트 메뉴**를 연다.
  2. 메뉴 항목을 차례로 확인: **`✏ 위젯 수정`**, **`복제`**, **`새로고침`**, **`삭제`**.
  3. **`✏ 위젯 수정`** 클릭 → "위젯 수정" 모달에서 제목을 바꾸고 **`저장`** 한다.
  4. **`복제`** 클릭 → 같은 위젯이 하나 더 생기는지 확인한다.
  5. **`새로고침`** 클릭 → 위젯 데이터가 다시 그려지는지 확인한다.
  6. **`삭제`** 클릭 → 위젯이 제거되는지 확인한다.
- **기대 결과**: 각 메뉴 동작(제목 변경/복제/새로고침/삭제)이 즉시 화면에 반영된다.
- **Pass 기준**: 4개 동작이 모두 의도대로 반영되면 합격. (복제/삭제는 위젯 수 변화로 판정)

### 2.6 영업 대시보드 — 레이아웃 내보내기/가져오기/초기화 (⋮ 더보기 메뉴)

- **목적**: 대시보드 레이아웃 백업·복원·초기화 기능을 확인한다.
- **사전조건**: 영업 대시보드 화면.
- **테스트 단계**:
  1. 제목 "영업 대시보드" 옆 **`⋮`(더보기)** 버튼을 클릭한다.
  2. 메뉴에서 **`대시보드 내보내기`** 클릭 → JSON 파일이 내려받아지는지 확인한다.
  3. 다시 `⋮` → **`대시보드 가져오기`** 클릭 → 방금 받은 JSON 파일을 선택해 불러온다.
  4. 다시 `⋮` → **`기본 레이아웃으로 초기화`** 클릭 → 레이아웃이 기본값으로 돌아가는지 확인한다.
- **기대 결과**: 내보내기 시 `.json` 파일 다운로드, 가져오기 시 레이아웃 복원, 초기화 시 기본 레이아웃 복귀.
- **Pass 기준**: 세 동작이 각각 정상 수행되면 합격.

### 2.7 영업 현황 — 기간/유형 필터 조회

- **목적**: 기간과 거래처 유형 기준으로 매출 현황을 조회할 수 있는지 확인한다.
- **사전조건**: 영업 포털 로그인 완료.
- **테스트 단계**:
  1. 좌측 메뉴 **`영업 현황`** 을 클릭한다.
  2. 상단 필터에서 **시작일**, **종료일**을 지정한다.
  3. **유형** 드롭다운에서 `전체 / 병원만 / 대리점만 / 기타만` 중 하나를 선택한다.
  4. **`조회`** 버튼을 클릭한다.
- **기대 결과**: 부제(소제목)에 선택한 기간과 유형이 반영되고(`YYYY-MM-DD ~ YYYY-MM-DD · 유형`), 아래 KPI 카드/차트/표가 갱신된다.
- **Pass 기준**: 조회 후 소제목이 선택값으로 바뀌고 화면이 갱신되면 합격.

### 2.8 영업 현황 — 빠른 기간 버튼 (오늘/이번 주/이번 달)

- **목적**: 빠른 기간 버튼으로 날짜가 자동 설정되는지 확인한다.
- **사전조건**: `영업 현황` 화면.
- **테스트 단계**:
  1. 필터 우측의 **`오늘`** 버튼을 클릭한다 → 시작일·종료일이 오늘 날짜로 설정된다.
  2. **`이번 주`** 버튼을 클릭한다 → 시작일이 7일 전, 종료일이 오늘로 설정된다.
  3. **`이번 달`** 버튼을 클릭한다 → 시작일이 해당 월 1일, 종료일이 오늘로 설정된다.
- **기대 결과**: 각 버튼 클릭 시 시작/종료일이 규칙대로 채워지고 즉시 조회가 다시 실행된다.
- **Pass 기준**: 3개 버튼이 모두 날짜를 올바르게 채우고 화면을 갱신하면 합격.

### 2.9 영업 현황 — KPI 및 병원/대리점 매출 차트

- **목적**: 매출 KPI 4장과 병원/대리점 매출 막대그래프가 표시되는지 확인한다.
- **사전조건**: `영업 현황` 화면, 조회 완료.
- **테스트 단계**:
  1. `영업 현황` 화면에서 KPI 카드 4장(**총 매출 / 병원 매출 / 대리점 매출 / 신규 거래처**)을 확인한다.
  2. 그 아래 **병원 매출** 차트와 **대리점 매출** 차트(막대그래프)가 그려지는지 확인한다.
- **기대 결과**: KPI 4장 모두 금액/개수가 표시되고, 두 막대그래프가 정상 렌더링된다.
- **Pass 기준**: KPI와 차트가 모두 표시되면 합격. 차트가 빈 캔버스이거나 KPI가 비면 불합격.

### 2.10 영업 현황 — 담당자별 매출 표 + 드릴다운 필터

- **목적**: 담당자별 매출 표에서 담당자를 클릭해 품목 드릴다운을 필터링하는 동작을 확인한다.
- **사전조건**: `영업 현황` 화면.
- **테스트 단계**:
  1. **담당자별 매출** 표에서 컬럼(담당자/역할/담당 거래처/발주 건수/매출액/**반품건수**/신규)을 확인한다.
  2. 담당자 행(예: `박진우`)을 클릭한다.
  3. 화면 아래 **품목/부위/사이즈 드릴다운** 영역에 해당 담당자 기준 안내 배너(담당 비중 %)가 뜨는지 확인한다.
  4. 배너의 **`✕ 필터 해제`** 또는 같은 담당자 행을 다시 클릭해 필터를 해제한다.
- **기대 결과**: 담당자 클릭 시 해당 행이 선택 표시(테두리/배경)되고 드릴다운 수치가 비중만큼 축소 반영, 토스트로 "OOO 담당자 기준 드릴다운" 안내가 뜬다. 해제 시 전체 기준으로 복귀.
- **Pass 기준**: 선택/해제 토글이 동작하고 드릴다운 수치가 바뀌면 합격.

### 2.11 영업 현황 — 품목/부위/사이즈 드릴다운 펼치기

- **목적**: 브랜드별 매출을 부위/사이즈 단위로 펼쳐볼 수 있는지 확인한다.
- **사전조건**: `영업 현황` 화면 하단 드릴다운 영역.
- **테스트 단계**:
  1. 드릴다운에서 브랜드 행(예: `리코트 플러스`)의 `▶` 토글을 클릭한다.
  2. 부위(상지/하지 등) 및 사이즈별 수량/금액 표가 펼쳐지는지 확인한다.
  3. 다시 클릭해 접는다.
- **기대 결과**: 클릭 시 `▶`→`▼` 로 바뀌며 하위 표가 펼쳐지고, 재클릭 시 접힌다.
- **Pass 기준**: 펼침/접힘이 정상 토글되면 합격.

### 2.12 영업 현황 — 모드 전환에 따른 표시 변화

- **목적**: 사이드바 모드(관리자/개별 사원)에 따라 담당자별 표시가 달라지는지 확인한다.
- **사전조건**: `영업 현황` 화면.
- **테스트 단계**:
  1. 사이드바 하단 **모드 드롭다운**에서 `이경동 (사원)` 등 개별 사원을 선택한다.
  2. "모드 전환: OOO" 토스트가 뜨는지 확인한다.
  3. 담당자별 매출 표에서 본인 외 담당자 행이 흐리게(opacity 낮게) 표시되는지 확인한다.
  4. 다시 `관리자 (팀장)` 로 되돌린다.
- **기대 결과**: 개별 사원 모드에서는 해당 사원 행만 진하게, 나머지는 흐리게 표시된다. 관리자 모드에서는 전부 진하게 표시된다.
- **Pass 기준**: 모드 변경 시 토스트 + 표시 변화가 반영되면 합격.

### 2.13 거래처 관리 — 목록/검색

- **목적**: 거래처 목록 조회와 검색을 확인한다. (경영지원팀과 동일한 통합 폼 사용)
- **사전조건**: 영업 포털 로그인 완료, 거래처 데이터가 시드되어 있다.
- **테스트 단계**:

  2. 거래처 목록(표 또는 카드)이 표시되는지 확인한다.
  3. 상단 검색창에 거래처명 일부(예: `한빛`)를 입력한다.
- **기대 결과**: 거래처 목록이 로드되고, 검색어에 맞는 거래처만 필터링되어 표시된다.
- **Pass 기준**: 목록이 표시되고 검색이 동작하면 합격.

### 2.14 거래처 관리 — 거래처 상세 보기

- **목적**: 거래처 상세 정보(담당 영업사원/미수금/누적 매출/취급 품목/품목별 판매수량 등)를 확인한다.
- **사전조건**: 거래처 목록이 표시된 상태.
- **테스트 단계**:
  1. `거래처 관리` 목록에서 거래처 한 곳을 선택해 상세를 연다.
  2. 상세 모달/화면에서 거래처명·유형·담당 영업사원·결제 조건·미수금·누적 매출·의사 수·지역/주소·담당자 정보를 확인한다.
  3. **취급 품목** 배지, **품목별 판매수량(이번 달)** 표를 확인한다.
- **기대 결과**: 거래처 기본 정보와 함께 이번 달 품목별 판매수량/판매액 표(합계 행 포함)가 표시된다. 발주 내역이 없으면 "이번 달 발주 내역이 없습니다" 안내가 뜬다.
- **Pass 기준**: 상세 정보와 품목별 표가 정상 표시되면 합격.

### 2.15 거래처 관리 — 경쟁업체/특이사항 비고 저장

- **목적**: 거래처 상세의 경쟁업체 메모(비고)를 저장하는 동작을 확인한다.
- **사전조건**: 거래처 상세 모달이 열린 상태.
- **테스트 단계**:
  1. 거래처 상세 하단 **경쟁업체 품목(비고)** 텍스트 영역에 메모를 입력한다(예: "경쟁사 X 실리콘 밴드 20%").
  2. 모달 하단의 **`비고 저장`** 버튼을 클릭하거나, 텍스트 영역 밖을 클릭(blur)한다.
- **기대 결과**: "비고 저장 완료" 토스트가 뜨고, 다시 상세를 열어도 입력한 메모가 유지된다. (서버 저장 — Client.note)
- **Pass 기준**: 저장 토스트가 뜨고 재진입 시 메모가 보존되면 합격. 저장 실패 토스트가 뜨면 불합격.

### 2.16 거래처 관리 — 발주서 출력

- **목적**: 거래처 발주서를 인쇄용 창으로 출력하는 동작을 확인한다.
- **사전조건**: 거래처 상세 모달이 열린 상태. **브라우저 팝업 차단 해제 권장.**
- **테스트 단계**:
  1. 거래처 상세 하단 **`발주서 출력`** 버튼을 클릭한다.
  2. 새 창(인쇄용 발주서)이 열리는지 확인한다.
- **기대 결과**: "발 주 서" 제목, 공급자(알티바이오)/공급받는자(거래처) 정보, 품목·수량·단가·금액·발주일 표가 포함된 새 창이 뜬다. 하단에 `인쇄`/`닫기` 버튼이 있다.
- **Pass 기준**: 발주서 창이 열리고 거래처 정보·품목 표가 채워지면 합격. 팝업이 막히면 "팝업을 허용해주세요" 토스트가 뜨므로 차단을 해제하고 재시도.

### 2.17 거래처 관리 — 거래명세표 출력

- **목적**: 거래처 거래명세표(공급가액/부가세/총합계 포함)를 출력하는 동작을 확인한다.
- **사전조건**: 거래처 상세 모달이 열린 상태. 팝업 차단 해제 권장.
- **테스트 단계**:
  1. 거래처 상세 하단 **`거래명세표 출력`** 버튼을 클릭한다.
  2. 새 창(거래명세표)이 열리는지 확인한다.
- **기대 결과**: "거래명세표" 제목, 공급자/공급받는자 정보, 거래일·품목·수량·단가·공급가액 표, **공급가액 합계 / 부가세(10%) / 총 합계** 행이 표시된다.
- **Pass 기준**: 거래명세표 창이 열리고 합계 3행이 계산되어 표시되면 합격.

### 2.18 담당자 관리 — 담당자 카드 조회

### 2.19 담당자 관리 — 거래처 배정

- **목적**: 특정 담당자에게 거래처를 배정하는 동작을 확인한다.
- **사전조건**: `담당자 관리` 화면, 모드 `관리자 (팀장)`.

  1. 우측 **AI 어시스턴트** 입력창에 예시 문구(예: "한빛정형외과로 발주서 작성")를 입력한다.
  2. **`↑`(전송)** 버튼을 클릭한다.
  3. (제공되는 경우) 추천 문구(suggestions)를 클릭해도 본다.
- **기대 결과**: AI 메시지 영역에 응답이 표시되고, 관련 양식 항목이 자동으로 채워진다.
- **Pass 기준**: 메시지 전송 후 응답/채움 동작이 일어나면 합격. (백엔드 미연동 시 응답이 제한될 수 있으므로, 입력/전송 UI 동작 여부로 1차 판정)

### 2.37 보고서 작성 — 임시저장 / 인쇄 / PDF 출력

- **목적**: 보고서 양식의 저장·인쇄·PDF 출력 버튼을 확인한다.
- **사전조건**: `보고서 작성` 화면, 양식이 채워진 상태.
- **테스트 단계**:
  1. 하단 **`임시저장`** 버튼을 클릭한다.
  2. **`양식대로 인쇄`** 버튼을 클릭한다 → 인쇄 미리보기/대화상자가 뜨는지 확인한다.
  3. **`PDF 출력`** 버튼을 클릭한다 → PDF가 생성/다운로드되는지 확인한다.
- **기대 결과**: 임시저장은 저장 처리, 인쇄는 인쇄 대화상자, PDF는 파일 출력이 각각 동작한다.
- **Pass 기준**: 세 버튼이 각각 의도된 동작(저장/인쇄/PDF)을 수행하면 합격.

### 2.38 데이터 탐색기 — 테이블 선택/조회

- **목적**: 원본 DB 테이블을 선택해 데이터를 조회하는 동작을 확인한다.
- **사전조건**: 영업 포털 로그인 완료.
- **테스트 단계**:
  1. 좌측 메뉴 **`데이터 탐색기`** 를 클릭한다.
  2. 상단 **테이블 선택** 드롭다운에서 테이블을 하나 선택한다.
  3. 데이터 그리드(표)가 표시되는지 확인한다.
- **기대 결과**: 선택한 테이블의 데이터가 그리드에 표시되고, 하단에 페이지네이션이 보인다.
- **Pass 기준**: 테이블 선택 시 데이터 그리드가 표시되면 합격.

### 2.39 데이터 탐색기 — 필터링/페이지 이동

- **목적**: 데이터 탐색기의 필터와 페이지 이동을 확인한다.

- **테스트 단계**:
  1. 상단 **필터 바**에서 조건을 지정해 데이터를 좁힌다.
  2. 하단 **페이지네이션**에서 다음 페이지로 이동한다.
- **기대 결과**: 필터 적용 시 그리드가 좁혀지고, 페이지 이동 시 다른 행 묶음이 표시된다. 요약 바(건수 등)가 갱신된다.
- **Pass 기준**: 필터/페이지 이동이 그리드에 반영되면 합격.

- **Pass 기준**: 파일이 내려받아지면 합격. (라벨/버튼 위치는 툴바 영역에 있으며, 환경에 따라 다를 수 있음 — 출력 동작 여부로 판정)

### 2.43 비밀번호 변경 (본인 계정)

- **목적**: 로그인한 본인 계정의 비밀번호를 변경하는 동작을 확인한다.
- **사전조건**: 영업 포털 로그인 완료, 현재 비밀번호를 안다.
- **테스트 단계**:
  1. 사이드바 상단의 **`🔑 비밀번호 변경`** 링크를 클릭한다.

- **기대 결과**: 변경 성공 메시지가 뜨고 모달이 닫힌다. 현재 비밀번호 불일치/짧은 새 비밀번호 등은 오류 메시지로 안내된다.
- **Pass 기준**: 올바른 입력 시 변경이 처리되고, 잘못된 입력 시 오류가 안내되면 합격.

- **Pass 기준**: 모든 메뉴가 올바른 페이지를 표시하고 모바일 메뉴 토글이 동작하면 합격.

> 로그인 계정: `qc@rtbio.com` / 비밀번호는 **관리자가 안내한 비밀번호**를 사용하세요.
> 권한(role): **QC** (일부 메뉴는 팀관리자 권한이 있어야 보입니다.)

### 사용 전 안내
- 이 포털은 발주 접수 → 확정 → 출고 → 출고완료로 이어지는 **머니플로우의 중간(출고) 단계**를 담당합니다.

- 화면 우측 하단에 잠깐 떴다 사라지는 **토스트 메시지**(예: "발주가 확정되었습니다")로 작업 성공/실패를 안내합니다. 이 메시지를 꼭 확인하세요.

### 3.1 로그인 및 포털 진입

- **목적**: QC 계정으로 품질관리 포털에 정상 진입하는지 확인.
- **사전조건**: 관리자에게 `qc@rtbio.com` 계정의 비밀번호를 안내받았을 것.
- **테스트 단계**:
  1. 브라우저에서 `http://localhost:3000/qc` 로 접속합니다.
  2. 로그인 화면이 나오면 아이디 `qc@rtbio.com`, 비밀번호(관리자 안내값)를 입력하고 로그인합니다.
  3. 좌측 사이드바 상단에 사용자 정보(예: "이미영 / 품질관리팀")가 표시되는지 확인합니다.
  4. 첫 화면이 "품질관리 대시보드"로 열리는지 확인합니다.
- **기대 결과**: 로그인 후 `/qc` 대시보드가 열리고, 좌측 메뉴에 업무/관리/입출고/보고서/거래처/시스템 그룹이 보입니다.
- **Pass 기준**: 로그인 실패·권한 오류(403) 없이 대시보드가 표시되면 합격. 다른 포털로 튕기거나 메뉴가 비면 불합격.

### 3.2 대시보드 — 위젯 보기

- **목적**: 대시보드 위젯이 정상적으로 로드되고 핵심 지표가 보이는지 확인.
- **사전조건**: 로그인 완료.
- **테스트 단계**:
  1. 좌측 메뉴 **'대시보드'** 클릭.
  2. 화면에 위젯 카드들이 격자(그리드)로 표시되는지 확인합니다.
  3. 상단 제목이 "품질관리 대시보드"인지 확인합니다.
- **기대 결과**: 위젯들이 로딩되어 숫자/차트가 표시됩니다(빈 경우 "위젯 추가" 안내).
- **Pass 기준**: 위젯 영역이 깨지거나 영구적으로 "불러오는 중"에 멈추지 않으면 합격.

### 3.3 대시보드 — 위젯 추가 / 편집 / 초기화

- **목적**: 위젯 추가·내보내기·가져오기·기본 레이아웃 초기화 동작 확인.
- **사전조건**: 대시보드 화면.
- **테스트 단계**:
  1. 우측 상단 **'➕ 위젯 추가'** 클릭 → "위젯 추가" 모달이 열립니다.
  2. 모달의 **'갤러리에서 선택'** 영역에서 위젯 카드를 하나 클릭해 추가합니다.
  3. 추가된 위젯이 대시보드 격자에 나타나는지 확인합니다.
  4. (선택) 모달 상단 **'🤖 AI에게 말하기'** 입력창에 원하는 위젯을 자연어로 요청해 봅니다.
  5. (선택) **'+ 직접 만들기'** 버튼으로 위젯 빌더를 열어 제목/데이터 소스/차트 종류를 설정합니다.
  6. 제목 옆 더보기 메뉴(**⋮**)를 눌러 **'대시보드 내보내기'**, **'대시보드 가져오기'**, **'기본 레이아웃으로 초기화'**를 확인합니다.
  7. **'기본 레이아웃으로 초기화'**를 눌러 기본 배열로 돌아오는지 확인합니다.
- **기대 결과**: 위젯 추가 후 즉시 격자에 반영되고, 초기화 시 기본 위젯 구성으로 복원됩니다.
- **Pass 기준**: 위젯 추가/초기화가 화면에 반영되면 합격. 모달이 안 열리거나 추가가 무반응이면 불합격.

### 3.4 발주확정 — 페이지 진입 및 접수 대기 목록

- **목적**: 접수된(SUBMITTED) 발주가 "접수 대기" 목록에 보이는지 확인.
- **사전조건**: 거래처 포털 또는 영업팀이 발주를 접수(SUBMITTED 상태)한 주문이 최소 1건 있을 것.
- **테스트 단계**:
  1. 좌측 메뉴 **'발주확정'** 클릭.
  2. 상단 통계 카드(접수 대기 건수 등)와 "접수 대기 N" 섹션을 확인합니다.
  3. 발주 카드를 클릭해 펼치면(▼) 품목 목록·재고 표시·합계가 나오는지 확인합니다.
- **기대 결과**: 접수 대기 카드에 거래처명, 발주번호, 시간, 품목 수, 합계가 표시되고, 펼치면 각 품목의 재고 배지(재고 N개)와 "출고 후 재고" 경고가 보입니다.
- **Pass 기준**: 접수된 발주가 목록에 보이고 펼침/접힘이 동작하면 합격.

### 3.5 발주 확정 (SUBMITTED → CONFIRMED, 재고 예약 + 출고 자동시작)

- **목적**: 발주를 확정하면 상태가 CONFIRMED로 바뀌고 출고(칸반)가 자동 시작되는지 확인. 이때 가용재고가 예약 차감됩니다.
- **사전조건**: 접수 대기 발주 1건. 가용재고가 충분할 것(부족 시 확정이 거부될 수 있음).
- **테스트 단계**:
  1. **'발주확정'** 페이지에서 대상 카드 하단 **'✔ 확정'** 클릭.
  2. "발주 확정" 모달에서 발주번호·거래처·합계를 확인합니다.
  3. **'담당자 배정'** 드롭다운에서 담당자를 선택합니다(필수).
  4. 모달 확인 버튼을 누릅니다.
- **기대 결과**: "발주가 확정되었습니다" 토스트가 뜨고, 카드가 접수 대기 목록에서 사라집니다. 출고관리(칸반) 보드의 첫 단계 컬럼에 해당 주문 카드가 생성됩니다(출고 자동 시작).
- **Pass 기준**: 담당자 미선택 시 "담당자를 배정해주세요" 안내로 막히고, 담당자 선택 후 확정하면 칸반에 카드가 생기면 합격. 재고 부족 등 오류는 토스트로 사유가 표시되어야 합격(조용히 실패하면 불합격).

### 3.6 발주 보류 (HELD) 및 보류 건 복원

- **목적**: 발주를 보류 처리하고, 보류 섹션에서 다시 접수 복원 또는 바로 확정할 수 있는지 확인.
- **사전조건**: 접수 대기 발주 1건.
- **테스트 단계**:
  1. 대상 카드 하단 **'⏸ 보류'** 클릭.
  2. "⏸ 발주 보류" 모달에서 **보류 사유**를 입력(필수)하고 확인합니다.
  3. 카드가 "접수 대기"에서 사라지고, 하단 **'⏸ 보류 건'** 섹션 카운트가 증가하는지 확인합니다.
  4. 보류 섹션 헤더를 클릭해 펼친 뒤, 보류된 카드의 **'접수 복원'** 또는 **'바로 확정'** 버튼을 확인합니다.
  5. **'접수 복원'**을 눌러 다시 접수 대기로 돌아오는지 확인합니다.
- **기대 결과**: 보류 시 "보류" 토스트, 보류 섹션에 카드 이동. 복원 시 접수 대기 목록으로 되돌아옵니다.
- **Pass 기준**: 사유 미입력 시 막히고, 보류/복원이 목록에 반영되면 합격.

### 3.7 발주 반려 (REJECTED)

- **목적**: 발주를 반려 처리하면 목록에서 사라지고 반려 사유가 기록되는지 확인.
- **사전조건**: 접수 대기 발주 1건.
- **테스트 단계**:
  1. 대상 카드 하단 **'✖ 반려'** 클릭.
  2. "발주 반려" 모달에서 **반려 사유**를 입력(필수)합니다(예: 재고 부족, 단가 확인 필요).
  3. 빨간색 **'반려'** 버튼을 누릅니다.
- **기대 결과**: "발주가 반려되었습니다" 토스트가 뜨고 카드가 사라집니다.
- **Pass 기준**: 사유 미입력 시 막히고, 반려 후 목록에서 제거되면 합격.

### 3.8 출고관리(칸반) — 보드 보기 및 날짜 필터

- **목적**: 확정된 주문이 칸반 보드 단계별 컬럼에 표시되고 날짜 필터가 동작하는지 확인.
- **사전조건**: 확정된(출고 시작된) 주문 1건 이상.
- **테스트 단계**:
  1. 좌측 메뉴 **'출고관리'** 클릭.
  2. 상단 날짜 필터에서 **'어제' / '오늘' / '이번주' / '기간선택'** 버튼을 차례로 눌러봅니다.
  3. **'기간선택'**을 누르면 시작/종료 날짜 입력칸이 나타나며, 날짜를 바꾸면 보드가 갱신되는지 확인합니다.
  4. 우측의 **일일 진행률** 표시와 단계별 컬럼(접수 대기 등)에 카드가 있는지 확인합니다.
- **기대 결과**: 필터에 따라 보드의 카드가 달라지고, 컬럼 헤더에 단계 아이콘/이름이 보입니다.
- **Pass 기준**: 날짜 필터 전환과 기간 입력이 보드에 반영되면 합격.

### 3.9 칸반 — 카드 다음 단계 / 이전 단계 이동 (출고 완료 = 실재고 차감)

- **목적**: 카드를 다음/이전 단계로 이동하고, 마지막(완료) 단계에 도달하면 주문이 출고완료(COMPLETED) 처리되며 실재고가 차감되는지 확인.
- **사전조건**: 칸반에 출고 시작된 카드 1건.
- **테스트 단계**:
  1. 칸반 카드의 **'상세 ▼'**를 눌러 상세를 펼치거나 카드를 클릭해 상세 모달을 엽니다.
  2. 상세 모달 하단의 **'➡ (다음 단계명)'** 버튼을 클릭합니다.
  3. "(다음 단계)으로 이동" 모달에서 해당 단계 담당자를 선택하고 **'이동하기'**를 누릅니다.
  4. 카드가 다음 컬럼으로 이동하는지 확인합니다.
  5. 마지막 단계까지 이동시켜 주문이 "완료"로 바뀌는지 확인합니다.
  6. (되돌리기) 상세 모달의 **'⬅ (이전 단계명)'** 버튼으로 이전 단계로 되돌려 봅니다.
- **기대 결과**: 단계 이동 시 토스트로 "(단계) 이동 — 담당: (이름)" 안내. 마지막 단계 도달 시 주문이 출고완료(COMPLETED)되고 실재고가 줄어듭니다.
- **Pass 기준**: 다음/이전 단계 이동이 보드에 반영되고, 마지막 단계에서 완료 처리되면 합격. "출고가 시작되지 않은 주문" 안내가 뜨면 발주확정에서 출고 시작이 안 된 것이므로 3.5를 먼저 수행.

### 3.10 칸반 — 카드 상세 보기 및 거래명세서 출력

- **목적**: 카드 상세에서 품목(사이즈별 그리드)·담당자·진행 단계·거래명세서를 확인.
- **사전조건**: 칸반 카드 1건.
- **테스트 단계**:
  1. 칸반 카드를 클릭해 상세 모달을 엽니다.
  2. 발주번호·거래처·담당자·배송·합계·진행 단계, 그리고 카테고리별 사이즈 수량 그리드가 보이는지 확인합니다.
  3. **'거래명세서 출력'** 버튼을 눌러 인쇄/PDF 미리보기가 뜨는지 확인합니다.
- **기대 결과**: 상세 정보가 모두 표시되고, 거래명세서 출력 시 인쇄 가능한 양식이 새 창/미리보기로 나타납니다.
- **Pass 기준**: 상세 모달 정보 표시와 거래명세서 출력이 동작하면 합격.

### 3.11 칸반 — 담당자 배정/제거 (현재 단계, 최대 3명)

- **목적**: 카드 상세에서 현재 단계 담당자를 추가/제거할 수 있는지 확인.
- **사전조건**: 칸반 카드 1건.
- **테스트 단계**:
  1. 카드 상세 모달의 **'담당자 (현재 단계)'** 영역을 확인합니다.
  2. **'담당자 추가...'** 드롭다운에서 담당자 선택 후 **'+ 추가'** 클릭.
  3. 담당자 행 우측 **'✕'**로 제거해 봅니다.
  4. 3명까지 추가한 뒤 "최대 3명까지 배정 가능합니다" 안내가 나오는지 확인합니다.
- **기대 결과**: 담당자가 즉시 목록에 추가/제거되고, 카드 아바타에 반영됩니다.
- **Pass 기준**: 추가/제거가 반영되고 3명 제한 안내가 보이면 합격.

### 3.12 칸반 — 배지 달기 / 메모 추가

- **목적**: 카드에 색상 배지와 메모를 부착·삭제할 수 있는지 확인.
- **사전조건**: 칸반 카드 1건. (우클릭 또는 카드 컨텍스트 메뉴 사용)
- **테스트 단계**:
  1. 카드 컨텍스트 메뉴(우클릭 등)에서 **'배지 달기'** 선택.
  2. 색상 선택 후 텍스트(예: 긴급, VIP)를 입력하고 **'배지 추가'** 클릭.
  3. 카드에 배지가 표시되는지 확인하고, 배지의 **'✕'**로 삭제해 봅니다.
  4. 같은 메뉴에서 **'메모 추가'** 선택 → 메모 입력 후 **'저장'** → 카드에 메모 아이콘이 표시되는지 확인.
  5. 메모를 다시 열어 **'메모 삭제'**로 제거합니다.
- **기대 결과**: 배지/메모가 카드에 표시되고, 삭제 시 사라집니다.
- **Pass 기준**: 배지·메모 추가/삭제가 카드에 반영되면 합격. (참고: 배지는 화면 전용 표시 기능입니다.)

### 3.13 칸반 — 보드 단계(컬럼) 수정 / 추가 / 삭제

- **목적**: 칸반 단계의 아이콘·이름·색상 수정과 단계 추가/삭제가 동작하는지 확인.
- **사전조건**: 출고관리 페이지.
- **테스트 단계**:
  1. 칸반 툴바 우측 **'보드 수정'** 클릭 → "보드 수정" 모달이 열립니다.
  2. 한 단계의 아이콘/이름/색상을 변경한 뒤 **'저장'**을 누릅니다.
  3. 모달에서 **'+ 열 추가'**를 눌러 새 단계가 추가되는지 확인합니다(최대 개수 제한 확인).
  4. 단계 행의 **'✕'**로 삭제를 시도합니다(최소 2개 유지, 사용 중인 단계는 삭제가 거부될 수 있음).
- **기대 결과**: 수정 시 보드 헤더에 즉시 반영, 추가/삭제 시 컬럼 수가 변합니다. 최소 2개 미만으로는 삭제 불가, 카드가 있는 단계 삭제 시 안내가 뜹니다.
- **Pass 기준**: 단계 수정/추가/삭제와 제한 안내가 정상 동작하면 합격.

### 3.14 출고내역 — 조회 및 필터

- **목적**: 출고완료된 주문 내역을 기간·거래처로 조회할 수 있는지 확인.
- **사전조건**: 출고완료(COMPLETED) 주문이 1건 이상.
- **테스트 단계**:
  1. 좌측 메뉴 **'출고내역'** 클릭.
  2. 날짜 필터 **'오늘' / '이번주' / '이번달' / '기간선택'**을 전환합니다.
  3. **'기간선택'** 시 시작/종료 날짜를 입력합니다.
  4. **'거래처 검색...'** 입력칸에 거래처명을 입력해 필터링합니다.
  5. 요약 통계 칩(건수 / 합계금액 / 명세서 발급)을 확인합니다.
- **기대 결과**: 필터에 맞는 출고완료 주문이 최신순으로 표에 표시되고, 통계 칩이 갱신됩니다. 데이터가 없으면 "해당 기간에 출고 완료된 주문이 없습니다" 안내.
- **Pass 기준**: 필터·검색·통계가 표에 반영되면 합격.

### 3.15 출고내역 — 상세 보기

- **목적**: 출고내역 행을 눌러 상세 정보를 볼 수 있는지 확인.
- **사전조건**: 출고내역 목록에 1건 이상.
- **테스트 단계**:
  1. 출고내역 표의 한 행(또는 행의 **'상세 →'**)을 클릭합니다.
  2. 상세 모달에 거래처·금액·완료 시간 등이 표시되는지 확인합니다.
- **기대 결과**: 상세 모달이 열려 해당 주문 정보가 보입니다.
- **Pass 기준**: 상세 모달이 정상 표시되면 합격.

### 3.16 재고현황 — 목록 / 카테고리 탭 / 검색

- **목적**: 제품 재고 목록과 카테고리 탭·검색·상태 표시가 정상인지 확인.
- **사전조건**: 제품 데이터가 존재.
- **테스트 단계**:
  1. 좌측 메뉴 **'재고현황'** 클릭.
  2. 상단 통계 카드(총 제품 수 / 재고 부족 제품 / 총 재고 수량)를 확인합니다.
  3. **카테고리 탭**(전체/무릎/상지 등)을 눌러 목록이 필터링되는지 확인합니다.
  4. **'제품명으로 검색...'** 입력칸에 제품명 또는 SKU 일부를 입력해 검색합니다.
  5. 표의 실재고/가용재고/예약/상태(정상·주의·부족) 컬럼을 확인합니다.
- **기대 결과**: 탭·검색에 따라 목록이 갱신되고, 가용재고 기준 상태 배지와 재고 막대가 표시됩니다.
- **Pass 기준**: 탭/검색이 목록에 반영되고 상태 표시가 보이면 합격.

### 3.17 재고현황 — 신규 제품 등록

- **목적**: 신규 제품 등록 폼으로 제품을 추가할 수 있는지 확인.
- **사전조건**: 재고현황 페이지.
- **테스트 단계**:
  1. 우측 상단 **'+ 신규 제품 등록'** 클릭 → 제품 등록 모달이 열립니다.
  2. 제품명·SKU·카테고리·기본단가·사이즈(체크박스) 등 필드를 입력합니다.
  3. 저장(폼 액션) 버튼을 누릅니다.
- **기대 결과**: 등록 성공 토스트 후 목록에 새 제품이 나타납니다.
- **Pass 기준**: 필수값 누락 시 안내가 뜨고, 정상 입력 시 목록에 추가되면 합격.

### 3.18 재고현황 — 제품 수정

- **목적**: 기존 제품의 정보를 수정할 수 있는지 확인.
- **사전조건**: 제품 1건 이상.
- **테스트 단계**:
  1. 표의 대상 제품 행 우측 **'수정'** 클릭 → 수정 모달이 열립니다.
  2. 단가 등 일부 값을 변경하고 저장합니다.
- **기대 결과**: 수정 성공 토스트 후 목록의 값이 갱신됩니다.
- **Pass 기준**: 변경값이 목록에 반영되면 합격.

### 3.19 재고현황 — 제품 삭제

- **목적**: 제품 삭제(소프트 삭제) 확인 절차가 동작하는지 확인.
- **사전조건**: 제품 1건 이상.
- **테스트 단계**:
  1. 대상 제품 행 우측 빨간 **'삭제'** 클릭 → "제품 삭제 확인" 모달이 열립니다.
  2. 경고 문구("되돌릴 수 없으며 재고 이력도 사라집니다")를 확인하고 삭제를 확정합니다.
- **기대 결과**: 삭제 완료 토스트 후 목록에서 제품이 사라집니다.
- **Pass 기준**: 확인 모달을 거쳐 목록에서 제거되면 합격. (확인 없이 즉시 삭제되면 불합격)

### 3.20 샘플출고 — 단일 추가 (카트 방식)

- **목적**: 샘플/데모 제품을 카트에 담아 출고 등록(재고만 차감, 매출 미반영)하는지 확인.
- **사전조건**: 제품·거래처 데이터 존재. 화면 상단 "단일 추가" 탭 선택.
- **테스트 단계**:
  1. 좌측 메뉴 **'샘플출고'** 클릭 → 상단 안내("샘플 출고는 매출에 반영되지 않으며, 재고에서만 차감됩니다") 확인.
  2. 제품·사이즈·수량을 고른 뒤 **'+ 추가'**로 카트에 담습니다(여러 품목 가능).
  3. 거래처를 선택(검색)하고, **사유**를 입력합니다. 사유 입력 도움말 링크(샘플요청/데모용/교환·반품)도 사용 가능.
  4. 하단 **'(샘플 출고 등록)'** 버튼을 누릅니다.
- **기대 결과**: "N개 품목 M개 샘플 출고 등록 완료" 토스트가 뜨고, 아래 샘플 이력에 추가됩니다. 해당 사이즈 재고가 차감됩니다.
- **Pass 기준**: 거래처/제품/사유 누락 시 각각 안내로 막히고, 정상 입력 시 등록되면 합격.

### 3.21 샘플출고 — 그리드 입력 (일괄)

- **목적**: 카테고리 × 사이즈 그리드로 여러 품목을 한 번에 샘플 출고하는지 확인.
- **사전조건**: 샘플출고 페이지.
- **테스트 단계**:
  1. 화면 상단 **'그리드 입력'** 탭 클릭 → 그리드 화면으로 전환.
  2. 거래처와 사유를 선택/입력하고, 그리드의 사이즈별 칸에 수량을 입력합니다.
  3. **'그리드 일괄 출고'** 클릭 → 확인 창(거래처/품목수/총수량/사유)에서 확인합니다.
  4. (선택) **'초기화'**로 입력값을 비울 수 있습니다.
- **기대 결과**: 수량이 입력된 라인만 일괄 출고되고 이력에 반영됩니다.
- **Pass 기준**: 수량 0건/거래처·사유 누락 시 막히고, 정상 입력 시 일괄 출고되면 합격.

### 3.22 입고등록

- **목적**: 공급처로부터 들어온 제품을 입고 등록해 실재고를 늘리는지 확인.
- **사전조건**: 입고등록 페이지. 제품 데이터 존재.
- **테스트 단계**:
  1. 좌측 메뉴 **'입고등록'** 클릭.
  2. 상단에서 입고일자·공급처·비고를 입력합니다.
  3. 입고 품목 표에서 제품·규격을 선택하고 **입고수량**을 입력합니다. "입고 후 재고"가 자동 계산되는지 확인합니다.
  4. 필요 시 **'+ 행 추가'**로 품목을 더 추가합니다.
  5. (선택) **'임시저장'**을 눌러봅니다.
  6. **'입고 확정'**을 클릭합니다.
- **기대 결과**: "입고 확정 — N개 품목 / 총 M개" 토스트가 뜨고, 입력 행이 초기화되며 입고 이력에 기록됩니다. 해당 사이즈 실재고가 증가합니다.
- **Pass 기준**: 제품/사이즈/수량이 없는 행은 무시되고, 유효 품목 입고 시 토스트와 이력이 반영되면 합격.

### 3.23 재고조정 (실사 보정 / 반품 / 폐기)

- **목적**: 시스템 재고와 실사 재고 차이를 사유와 함께 조정 반영하는지 확인.
- **사전조건**: 재고조정 페이지.
- **테스트 단계**:
  1. 좌측 메뉴 **'재고조정'** 클릭.
  2. 조정 입력 표에서 한 행의 **실사 재고**를 시스템 재고와 다르게 입력합니다(차이 발생).
  3. **사유**(파손/반품/분실/실사조정/기타)와 **비고**를 입력합니다.
  4. **'조정 확정'**을 클릭합니다.
- **기대 결과**: "재고 조정 확정 — N건 적용" 토스트가 뜨고, 조정 이력에 기록되며 시스템 재고가 실사 값으로 맞춰집니다. (반품은 +수량, 파손/분실은 -수량으로 처리)
- **Pass 기준**: 차이가 0인 행은 무시되고, 차이 있는 행만 사유에 따라 조정되면 합격. 변경이 전혀 없으면 "조정할 변경이 없습니다" 안내가 떠야 합격.

### 3.24 보고서 작성 (ISO 13485 품질양식 + AI 어시스턴트)

- **목적**: 품질양식을 선택해 작성하고 AI 어시스턴트로 자동 채움·인쇄·PDF 출력이 되는지 확인.
- **사전조건**: 보고서 작성 페이지.
- **테스트 단계**:
  1. 좌측 메뉴 **'보고서 작성'** 클릭.
  2. 상단 양식 선택 드롭다운에서 양식을 하나 고르면 미리보기가 표시되는지 확인합니다.
  3. 우측 **'AI 어시스턴트'** 입력창에 예시처럼 자연어(예: "리코탭플러스 무릎 숏 50개 출고 검사 합격")를 입력하고 전송(↑)합니다.
  4. AI 응답이 양식 필드를 채우는지 확인합니다.
  5. 하단 **'임시저장' / '양식대로 인쇄' / 'PDF 출력'** 버튼을 각각 눌러봅니다.
- **기대 결과**: 양식 미리보기 표시, AI 자동 채움 시도, 인쇄/PDF 미리보기가 동작합니다.
- **Pass 기준**: 양식 선택과 인쇄/PDF 출력이 동작하면 합격. (AI 자동 채움은 연동 환경에 따라 응답이 다를 수 있음 — 입력/전송 UI가 동작하면 합격)

### 3.25 데이터 탐색기 (원본 DB 조회/편집/내보내기)

- **목적**: 원본 데이터 테이블을 선택해 조회·필터·CSV/Excel 내보내기가 되는지 확인.
- **사전조건**: 데이터 탐색기 페이지.
- **테스트 단계**:

  3. 필터 바에서 조건을 추가해 행이 걸러지는지 확인합니다.
  4. 툴바의 내보내기(CSV/Excel) 동작과 페이지네이션을 확인합니다.
- **기대 결과**: 선택한 테이블의 행이 표로 표시되고, 필터/페이지 이동/내보내기가 동작합니다.
- **Pass 기준**: 테이블 조회와 내보내기가 동작하면 합격.

### 3.28 설정 (테넌트 설정)

- **목적**: 테넌트 설정 값을 조회·변경·저장할 수 있는지 확인(경영지원팀과 동기).

  3. 값을 하나 변경하고 저장합니다.

### 3.29 직원 관리 (팀관리자 전용)

- **목적**: 팀관리자 권한 계정에서만 직원 관리 메뉴가 보이고 동작하는지 확인.
- **사전조건**: 팀관리자(isTeamAdmin) 권한이 부여된 QC 계정. (일반 QC 계정에는 메뉴가 보이지 않음 — 정상)
- **테스트 단계**:
  1. 좌측 메뉴 하단 **'직원 관리'** 항목이 보이는지 확인합니다.
  2. 보이면 클릭해 직원 목록/생성/수정 화면이 뜨는지 확인합니다.
- **기대 결과**: 권한이 있으면 메뉴 노출 및 직원 관리 화면 표시, 권한이 없으면 메뉴 자체가 숨김 처리됩니다.
- **Pass 기준**: 권한에 따라 메뉴 노출/숨김이 올바르면 합격.

### 3.30 비밀번호 변경 / 로그아웃

- **목적**: 본인 비밀번호 변경과 로그아웃이 동작하는지 확인.
- **사전조건**: 로그인 상태.
- **테스트 단계**:
  1. 사이드바 사용자 영역의 **'🔑 비밀번호 변경'** 클릭 → 비밀번호 변경 모달이 열립니다.
  2. 현재/새 비밀번호를 입력하고 변경합니다.
  3. **'↩ 로그아웃'**을 눌러 로그인 화면으로 빠져나오는지 확인합니다.
  4. (확인) 변경한 새 비밀번호로 다시 로그인되는지 확인합니다.
- **기대 결과**: 비밀번호 변경 성공 안내 후, 새 비밀번호로 재로그인됩니다. 로그아웃 시 로그인 화면으로 이동합니다.
- **Pass 기준**: 비밀번호 변경 및 재로그인, 로그아웃이 정상 동작하면 합격.

### 3.31 모바일 사이드바 토글

- **목적**: 모바일 화면에서 햄버거 메뉴로 사이드바를 열고 닫을 수 있는지 확인.
- **사전조건**: 좁은 화면(모바일) 또는 브라우저 창을 좁게 조절.
- **테스트 단계**:
  1. 화면을 좁히면 상단에 햄버거(≡) 버튼과 "품질관리팀" 타이틀이 보입니다.
  2. 햄버거 버튼을 눌러 사이드바를 엽니다.
  3. 메뉴 밖 어두운 영역(오버레이)을 눌러 사이드바를 닫습니다.
- **기대 결과**: 사이드바가 열리고 닫힙니다.
- **Pass 기준**: 열기/닫기가 동작하면 합격.

## 4. 거래처 포털 (CLIENT)

거래처(대리점·병원) 담당자가 직접 사용하는 발주 전용 포털입니다. 제품을 엑셀 형태의 표에서 수량으로 발주하고, 발주 내역과 출고 상태를 확인하며, 알티바이오가 보낸 공지사항을 받아봅니다. 이 가이드는 개발 지식이 없는 실제 업무 사용자가 그대로 따라 하며 합격/불합격을 판정할 수 있도록 작성되었습니다.

### 접속 및 로그인 정보

| 항목 | 값 |
| --- | --- |
| 접속 주소(URL) | `http://localhost:3000/client` |
| 권한(역할) | CLIENT (거래처 담당자) |
| 로그인 계정 | `{거래처코드}@sample.local` (예: 거래처코드가 `medipharm`이면 `medipharm@sample.local`) |
| 비밀번호 | 관리자가 안내한 비밀번호 |

> 참고: 이 포털은 로그인 후 단일 화면(SPA) 안에서 여러 페이지가 전환되는 구조입니다. 페이지 이동은 화면 하단의 5개 탭(홈·발주·내역·공지·계정) 또는 화면 안의 버튼으로 이루어집니다. PC 화면에서는 상단에 가로 메뉴(홈·발주하기·발주내역·공지사항)가 함께 보일 수 있습니다.

### 발견된 메뉴 / 페이지 전체 목록

| 화면 ID | 이름 | 접근 방법 |
| --- | --- | --- |
| `page-loading` | 로딩 화면 | 접속 직후 자동 표시("거래처 정보를 불러오는 중...") |
| `page-login` | 로그인 화면 | 세션 없을 때 표시(정상 접근 시 미들웨어가 대신 처리) |
| `page-dashboard` | 홈(대시보드) | 하단 탭 "홈" / 상단 "홈" |
| `page-order` | 발주하기(발주폼) | 하단 탭 "발주" / 상단 "발주하기" / 홈의 "발주하기" 버튼 |
| `page-confirm` | 발주 확인 | 발주폼에서 "발주하기" 버튼 클릭 시 진입 |
| `page-history` | 발주 내역 | 하단 탭 "내역" / 상단 "발주내역" / 홈의 "발주내역" 버튼 |
| `page-status` | 출고 상태(주문 상세) | 발주 내역 또는 홈 최근발주의 주문 카드 클릭 |
| `page-notice` | 공지사항 | 하단 탭 "공지" / 상단 "공지사항" / 홈 공지 미니영역 |
| `page-account` | 내 계정 | 하단 탭 "계정" / 우측 상단 사용자 버튼(이름 첫 글자) |

주요 사용자 액션: 로그인 / 로그아웃, 카테고리 빠른 이동, 수량 입력(엑셀 표), 무료배송 안내 확인, 발주 확인 진입, 배송지 선택·신규 배송지 추가·임시 배송지 입력, 배송 방법 선택, 비고 입력, 발주 제출, 발주 내역 기간 필터(이번달·1개월·3개월·전체·직접설정), 주문 상세 보기, 발주 취소 요청, 공지사항 열람(읽음 처리·미읽음 배지·팝업·다시 보지 않기).

### 4.1 로그인

- **목적**: 거래처 계정으로 포털에 정상 진입할 수 있는지 확인합니다.
- **사전조건**: 관리자가 발급한 거래처 계정 1개(`{거래처코드}@sample.local` + 비밀번호)가 준비되어 있어야 합니다.
- **테스트 단계**:
  1. 브라우저에서 `http://localhost:3000/client` 에 접속합니다.
  2. 접속 직후 "거래처 정보를 불러오는 중..." 로딩 화면이 잠깐 보이는지 확인합니다.
  3. (로그인 화면이 표시되면) "아이디" 칸에 거래처 아이디, "비밀번호" 칸에 관리자가 안내한 비밀번호를 입력합니다.
  4. "로그인" 버튼을 클릭합니다. (또는 비밀번호 칸에서 Enter 키)
- **기대 결과**: 정상 계정이면 로딩 후 자동으로 "홈(대시보드)" 화면이 표시됩니다. 상단에 거래처명/담당자 인사말("안녕하세요, OOO님")과 회사명이 보이고, 하단에 5개 탭(홈·발주·내역·공지·계정)이 나타납니다.
- **Pass 기준**: 올바른 계정으로 홈 화면 진입에 성공하고 거래처명·담당자명이 본인 정보와 일치하면 합격. 아이디/비밀번호가 틀린 경우 "아이디 또는 비밀번호가 올바르지 않습니다." 안내가 나타나면 합격(잘못된 로그인 차단 확인).

### 4.2 로그아웃

- **목적**: 로그아웃이 정상 동작하고 세션이 종료되는지 확인합니다.
- **사전조건**: 4.1 로그인 완료 상태.
- **테스트 단계**:
  1. 하단 탭 "계정"을 클릭해 "내 계정" 화면으로 이동합니다. (또는 우측 상단의 이름 첫 글자 버튼을 눌러 "계정 정보" 팝업 → "로그아웃")
  2. 화면 하단의 빨간색 테두리 "로그아웃" 버튼을 클릭합니다.
- **기대 결과**: 세션이 종료되고 로그인 화면(`/login`)으로 이동합니다.
- **Pass 기준**: 로그아웃 후 로그인 페이지로 이동하고, 뒤로가기 등으로 다시 포털에 들어가도 자동 로그인되지 않으면 합격.

### 4.3 홈(대시보드) — 발주 요약 / 바로가기

- **목적**: 홈 화면에 이번 달 발주 건수·금액, 최근 발주, 공지 요약이 올바르게 표시되는지 확인합니다.
- **사전조건**: 로그인 완료. 가능하면 해당 거래처로 발주 이력이 1건 이상 있는 상태.
- **테스트 단계**:
  1. 하단 탭 "홈"을 클릭합니다.
  2. 상단 인사말("안녕하세요, OOO님")과 회사명을 확인합니다.
  3. "OOOO년 O월 발주" / "OOOO년 O월 금액" 두 개의 통계 카드를 확인합니다.
  4. "공지사항" 미니 영역과 "전체보기 ›" 링크를 확인합니다.
  5. "최근 발주" 영역에서 최근 주문 카드(최대 3건)를 확인합니다.
  6. 상단 박스 안의 "발주하기" / "발주내역" 바로가기 버튼을 각각 클릭해 해당 화면으로 이동하는지 확인합니다.
- **기대 결과**: 발주 건수/금액 카드에 실제 데이터가 표시됩니다(발주 이력이 없으면 "0건", "₩0"). 최근 발주 영역에는 주문번호·상태 뱃지·품목 요약·금액·일시가 표시되며, 발주 이력이 없으면 "발주 내역이 없습니다" 안내가 나옵니다. 바로가기 버튼은 각각 발주폼/발주내역으로 이동합니다.
- **Pass 기준**: 통계 카드 숫자가 실제 발주 데이터와 일치하고, 바로가기 버튼이 의도한 화면으로 이동하면 합격.

### 4.4 발주폼 — 마감 시간 안내 바

- **목적**: 발주폼 상단의 택배 마감 시간 안내가 표시되는지 확인합니다.
- **사전조건**: 로그인 완료.
- **테스트 단계**:
  1. 하단 탭 "발주"를 클릭합니다.
  2. 화면 상단의 시계 아이콘이 있는 시간 바를 확인합니다.
- **기대 결과**: "현재 OO:OO | 택배 마감 OO:OO (O시간 OO분 남음)" 형식으로 현재 시각·마감 시각·남은 시간이 표시됩니다. 마감이 임박/지났는지에 따라 바 색상이 달라질 수 있습니다.
- **Pass 기준**: 현재 시각과 마감 시각, 남은 시간이 표시되면 합격.

### 4.5 발주폼 — 카테고리 빠른 이동

- **목적**: 카테고리 칩(무릎·상지·하지·스프린트)을 눌러 해당 제품 표로 스크롤 이동하는지 확인합니다.
- **사전조건**: 발주폼("발주") 화면 진입.
- **테스트 단계**:
  1. 발주폼 상단의 카테고리 칩 줄에서 "상지" 칩을 클릭합니다.
  2. 화면이 상지 제품 섹션으로 부드럽게 스크롤되는지 확인합니다.
  3. 다른 카테고리(예: "스프린트")도 동일하게 확인합니다.
- **기대 결과**: 클릭한 카테고리의 제품 표가 화면 상단으로 스크롤되어 나타납니다.
- **Pass 기준**: 각 칩 클릭 시 해당 카테고리 섹션으로 이동하면 합격.

### 4.6 발주폼 — 수량 입력(엑셀 표)과 가격/소계 자동 계산

- **목적**: 사이즈별 수량 입력 시 행 소계와 주문 요약(소계·배송비·합계)이 즉시 갱신되는지 확인합니다.
- **사전조건**: 발주폼("발주") 화면 진입.
- **테스트 단계**:
  1. 원하는 제품 행에서 사이즈 칸(S/M/L/XL, 또는 Free 제품은 단일 칸)에 수량(예: 2)을 입력합니다.
  2. 입력한 칸의 색상이 강조되고, 해당 행의 "소계" 칸에 금액이 표시되는지 확인합니다.
  3. 단가 칸에 할인율(예: `-10%`)이 함께 표시되는 제품이 있는지 확인합니다(거래처별 고정가/카테고리 할인 적용 제품).
  4. PC 화면이면 우측 "주문 요약" 사이드바, 모바일 화면이면 표 아래 "주문 요약"과 하단 바에서 선택 품목 수·소계·배송비·합계가 갱신되는지 확인합니다.
  5. 수량을 0으로 지우면 해당 행 소계와 요약에서 빠지는지 확인합니다.
- **기대 결과**: 수량 입력 즉시 행 소계 = 단가 × 수량으로 계산되고, 주문 요약의 소계/합계가 모든 입력 품목을 더한 값으로 갱신됩니다. 단가는 거래처에 맞는 가격(고정가 > 카테고리 할인 > 기본가 순)으로 표시됩니다.
- **Pass 기준**: 행 소계와 요약 합계가 손계산과 일치하고, 수량 변경에 실시간 반영되면 합격. (도메인 규칙: 단가는 주문 시점 스냅샷으로, 제출 시 서버에서 다시 계산됩니다.)

### 4.7 발주폼 — 무료배송 안내 배너

- **목적**: 카테고리별 기준 수량에 따라 무료배송 달성/미달성 안내가 바뀌는지 확인합니다.
- **사전조건**: 발주폼 진입.
- **테스트 단계**:
  1. 수량을 입력하기 전 배너 문구(카테고리별 무료배송 기준 수량 안내)를 확인합니다.
  2. 특정 카테고리 제품 수량을 조금씩 늘립니다.
  3. 기준 수량 미만일 때 "OOO 제품 O개 더 추가하면 무료배송!" 문구가 나오는지 확인합니다.
  4. 기준 수량 이상으로 늘려 "OOO 무료배송 달성! (현재/기준 개)" 문구로 바뀌는지 확인합니다.
  5. 무료배송 달성 시 주문 요약의 "배송비"가 "무료"로 바뀌는지 확인합니다.
- **기대 결과**: 수량에 따라 배너 문구와 색상이 미달성↔달성으로 전환되고, 달성 시 배송비가 0(무료)으로 계산됩니다.
- **Pass 기준**: 기준 수량 경계에서 배너 문구·배송비가 올바르게 바뀌면 합격.

### 4.8 발주 확인 화면 진입

- **목적**: 발주폼에서 "발주하기" 클릭 시 발주 확인 화면으로 넘어가는지, 빈 주문은 막히는지 확인합니다.
- **사전조건**: 발주폼 진입.
- **테스트 단계**:
  1. (먼저) 아무 수량도 입력하지 않은 상태에서 "발주하기" 버튼을 클릭합니다.
  2. 다음으로 제품 1개 이상 수량을 입력한 뒤 "발주하기" 버튼을 클릭합니다.
- **기대 결과**: 수량이 없으면 "주문할 제품을 선택해주세요." 안내가 나오고 이동하지 않습니다. 수량이 있으면 "발주 확인" 화면으로 이동하며 주문 제품 표·합계가 표시됩니다.
- **Pass 기준**: 빈 주문은 차단되고, 정상 주문은 확인 화면으로 넘어가면 합격.

### 4.9 발주 확인 — 주문 제품 / 합계 검증

- **목적**: 발주 확인 화면의 제품 목록·단가·수량·금액·합계가 발주폼 입력과 일치하는지 확인합니다.
- **사전조건**: 4.8을 통해 발주 확인 화면 진입.
- **테스트 단계**:
  1. "주문 제품" 표에서 제품명·사이즈·단가·수량·금액 행을 발주폼 입력과 비교합니다.
  2. 표 하단 "합계 금액"을 확인합니다.
  3. 화면 하단 "최종 결제 금액"과 배송비 포함/무료 안내를 확인합니다.
- **기대 결과**: 표의 각 행 금액 = 단가 × 수량이고, 합계 금액 = 모든 행 금액의 합, 최종 결제 금액 = 합계 + 배송비(무료면 합계와 동일)입니다.
- **Pass 기준**: 표시된 모든 금액이 손계산과 일치하면 합격.

### 4.10 발주 확인 — 배송지 선택

- **목적**: 등록된 배송지 중 하나를 선택할 수 있고 기본 배송지가 기본 선택되는지 확인합니다.
- **사전조건**: 발주 확인 화면 진입. 거래처에 배송지가 1개 이상 등록되어 있으면 더 좋습니다.
- **테스트 단계**:
  1. "배송지 선택" 영역에서 등록된 배송지 목록(별칭·주소·수령인·연락처·메모)을 확인합니다.
  2. "기본" 뱃지가 붙은 배송지가 처음에 선택(라디오 체크)되어 있는지 확인합니다.
  3. 다른 배송지를 클릭해 선택을 변경합니다.
- **기대 결과**: 배송지 목록이 표시되고, 기본 배송지가 우선 선택되어 있으며, 클릭으로 선택을 바꿀 수 있습니다. 등록된 배송지가 없으면 "등록된 배송지가 없습니다..." 안내가 나오고 임시 주소 입력이 활성화됩니다.
- **Pass 기준**: 기본 배송지 우선 선택 및 선택 변경이 정상 동작하면 합격.

### 4.11 발주 확인 — 신규 배송지 추가

- **목적**: "+ 신규 배송지 추가"로 배송지를 추가하고 목록에 반영·자동 선택되는지 확인합니다.
- **사전조건**: 발주 확인 화면 진입.
- **테스트 단계**:
  1. "배송지 선택" 영역 우측 상단 "+ 신규 배송지 추가" 버튼을 클릭합니다.
  2. 팝업에서 "배송지 별칭 *"(예: 강남 지점), "배송 주소 *"는 필수로 입력합니다. 수령인·연락처·메모는 선택 입력합니다.
  3. (선택) "기본 배송지로 설정" 체크박스를 켭니다.
  4. 팝업의 확인 버튼을 클릭합니다.
  5. 일부러 별칭이나 주소를 비운 채 저장을 시도해 봅니다.
- **기대 결과**: 별칭·주소가 모두 입력되면 새 배송지가 목록에 추가되고 자동 선택되며 "배송지 'OOO'을(를) 추가했습니다." 안내가 나옵니다. "기본 배송지로 설정"을 켜면 다른 배송지의 기본 표시가 해제됩니다. 별칭/주소가 비면 "별칭과 주소를 입력해주세요." 안내가 나오고 저장되지 않습니다.
- **Pass 기준**: 정상 입력 시 추가·자동선택되고, 필수값 누락은 차단되면 합격. (참고: 이 화면에서 추가한 배송지는 이번 주문 화면 기준으로 즉시 반영됩니다.)

### 4.12 발주 확인 — 이번 주문만 다른 주소(임시 배송지)

- **목적**: 저장하지 않고 이번 주문에만 사용할 임시 주소를 입력할 수 있는지 확인합니다.
- **사전조건**: 발주 확인 화면 진입.
- **테스트 단계**:
  1. "이번 주문만 다른 주소로 배송 (저장 안 함)" 라디오를 선택합니다.
  2. 아래에 나타나는 입력란(받는 분·연락처·배송 주소·배송 메모)에 값을 입력합니다.
- **기대 결과**: 라디오 선택 시 임시 주소 입력란이 펼쳐지고, 다른 배송지를 선택하면 다시 접힙니다. 입력한 주소는 이 주문에만 사용됩니다.
- **Pass 기준**: 임시 주소 입력란이 토글되고 입력이 가능하면 합격. (도메인 규칙: 임시 주소는 배송지로 저장되지 않고 주문 스냅샷으로만 보관됩니다.)

### 4.13 발주 확인 — 배송 방법 / 비고 입력

- **목적**: 배송 방법 선택과 비고 입력이 가능한지 확인합니다.
- **사전조건**: 발주 확인 화면 진입.
- **테스트 단계**:
  1. "배송 방법"에서 택배(기본)·방문수령·퀵(추가비용 별도) 중 하나를 선택합니다.
  2. "비고 / 메모" 칸에 배송 요청사항을 입력합니다.
- **기대 결과**: 라디오로 배송 방법을 변경할 수 있고, 비고 칸에 자유롭게 텍스트를 입력할 수 있습니다.
- **Pass 기준**: 배송 방법 선택과 비고 입력이 정상 동작하면 합격.

### 4.14 발주 제출

- **목적**: 발주가 서버에 정상 등록되고 발주번호가 발급되는지 확인합니다.
- **사전조건**: 4.9~4.13 완료(제품·배송지·배송 방법 선택). 배송지는 반드시 주소가 채워져 있어야 합니다.
- **테스트 단계**:
  1. (먼저) 배송지를 선택하지 않거나 임시 주소를 비운 채 "발주 제출"을 눌러 봅니다.
  2. 정상적으로 배송지를 지정한 뒤 화면 하단 "발주 제출" 버튼을 클릭합니다.
  3. 버튼이 "발주 처리 중..."으로 바뀌며 중복 클릭이 막히는지 확인합니다.
  4. 완료 팝업의 내용을 확인합니다.
  5. 팝업 확인을 누른 뒤 이동한 화면을 확인합니다.
- **기대 결과**: 배송지가 비면 "배송지를 선택하거나 입력해주세요." 안내가 나오고 제출되지 않습니다. 정상 제출 시 "발주가 성공적으로 접수되었습니다" 완료 팝업이 뜨고 품목 수·배송방법·합계, 실제 "주문번호"(예: `ORD-...`), 배송지 정보, "택배 마감 15:30 이전 접수분은 당일 출고됩니다." 안내가 표시됩니다. 확인 후 발주 내역 화면으로 이동하며 방금 발주가 목록 최상단에 나타납니다.
- **Pass 기준**: 실제 주문번호가 발급되고 완료 팝업·발주 내역 반영이 확인되면 합격. 제출 실패 시 오류 메시지가 표시되고 버튼이 다시 활성화되면 합격(오류 처리 확인). (도메인 규칙: 제출 시 라인 단가는 서버에서 재계산되며 주문은 SUBMITTED 상태로 접수됩니다.)

### 4.15 발주 내역 — 목록 조회

- **목적**: 본인 거래처의 발주 내역이 최신순으로 표시되는지 확인합니다.
- **사전조건**: 발주 이력이 1건 이상 있는 상태.
- **테스트 단계**:
  1. 하단 탭 "내역"을 클릭합니다.
  2. 주문 카드 목록(주문번호·상태 뱃지·품목 요약·배송방법·금액·일시)을 확인합니다.
  3. 기간 라벨(예: "OOOO년 O월 · O건")의 건수가 카드 수와 일치하는지 확인합니다.
- **기대 결과**: 본인 거래처 주문만 최신순으로 표시되고, 각 카드에 주문번호·상태·금액·일시가 보입니다. 내역이 없으면 "해당 기간에 발주 내역이 없습니다" 안내가 나옵니다.
- **Pass 기준**: 본인 주문이 최신순으로 정확히 표시되면 합격.

### 4.16 발주 내역 — 기간 필터

- **목적**: 기간 프리셋(이번달·1개월·3개월·전체·직접설정)으로 목록이 필터링되는지 확인합니다.
- **사전조건**: 서로 다른 기간에 분포한 발주 이력이 여러 건 있는 상태가 이상적.
- **테스트 단계**:
  1. "내역" 화면 상단의 기간 칩에서 "이번달"(기본 선택)을 확인합니다.
  2. "1개월", "3개월", "전체" 칩을 차례로 클릭하며 목록과 기간 라벨·건수가 바뀌는지 확인합니다.
  3. "직접설정" 칩을 클릭하면 시작일·종료일 날짜 입력칸이 나타나는지 확인합니다.
  4. 시작일·종료일을 지정해 해당 범위만 표시되는지 확인합니다.
- **기대 결과**: 각 프리셋에 맞게 목록이 좁혀지고, 기간 라벨이 "최근 1개월 · O건", "전체 기간 · O건", "시작일 ~ 종료일 · O건" 등으로 갱신됩니다. 직접설정 시 입력한 날짜 범위 내 주문만 표시됩니다.
- **Pass 기준**: 선택한 기간 조건에 맞는 주문만 표시되고 건수 라벨이 일치하면 합격.

### 4.17 발주 내역 — 수정/마감 표시 (접수 상태)

- **목적**: 접수 상태 주문에 마감 시간 전후로 "수정"/"마감" 표시가 올바르게 나타나는지 확인합니다.
- **사전조건**: "접수" 상태 주문이 1건 이상.
- **테스트 단계**:
  1. "내역"에서 "접수" 상태 주문 카드를 확인합니다.
  2. 카드 우측 하단에 "수정"(마감 전) 또는 "마감"(마감 시간 경과) 표시가 있는지 확인합니다.
  3. "수정" 링크를 클릭해 봅니다.
- **기대 결과**: 마감 전이면 "수정" 링크, 마감 후면 취소선이 그어진 "마감" 표시가 나옵니다. "수정"을 누르면 "수정 기능은 데모에서 지원하지 않습니다" 안내가 나옵니다(현재 데모 한정).
- **Pass 기준**: 마감 시간 상태에 따라 수정/마감 표시가 적절히 나타나면 합격. (참고: 라벨이 데모 한정 동작이라는 점을 표시함.)

### 4.18 출고 상태(주문 상세) — 진행 단계 확인

- **목적**: 주문 카드를 눌러 출고 진행 상태(4단계)와 주문 상세를 볼 수 있는지 확인합니다.
- **사전조건**: 발주 이력 1건 이상.
- **테스트 단계**:
  1. "내역" 또는 "홈" 최근 발주에서 주문 카드를 클릭합니다.
  2. 상단에 주문번호·거래처·발주일시·배송방법·상태가 표시되는지 확인합니다.
  3. "출고 진행 상태"에서 4단계(접수 → 확인중 → 준비중 → 출고완료) 표시와 현재 단계 강조를 확인합니다.
  4. 송장번호(있는 경우)와 포장 정보(박스 수·무게, 있는 경우)를 확인합니다.
  5. "주문 상세" 표에서 제품·사이즈·단가·수량·금액·합계를 확인합니다.
  6. "발주 내역으로" 버튼으로 목록에 돌아갈 수 있는지 확인합니다.
- **기대 결과**: 주문 상세 정보와 4단계 진행 표시가 보이고, 현재 단계가 강조됩니다. 송장번호/포장 정보는 해당 데이터가 있을 때만 표시됩니다.
- **Pass 기준**: 주문 상세 정보·진행 단계가 주문 데이터와 일치하면 합격.

### 4.19 발주 취소 요청

- **목적**: 포장 단계 이전 주문을 거래처가 직접 취소 요청할 수 있는지 확인합니다.
- **사전조건**: 포장 단계 이전(접수/확인중 등) 주문이 1건 이상.
- **테스트 단계**:
  1. 취소 가능 주문의 상세(출고 상태) 화면으로 진입합니다.
  2. 하단의 "발주 취소 요청" 버튼(빨간색)을 확인합니다. ("포장 단계 전까지 취소 가능" 안내 동반)
  3. "발주 취소 요청"을 클릭하고, 취소 사유 입력창에 사유를 입력한 뒤 확인합니다.
  4. (대조) 포장 단계 이후 주문은 취소 버튼 대신 "포장 단계 이후로 발주 취소가 불가능합니다..." 안내가 나오는지 확인합니다.
- **기대 결과**: 사유를 입력하면 취소가 처리되어 "OOO 취소 완료 — 알티바이오 시스템에 반영되었습니다" 안내가 나오고 상태가 "취소"로 바뀝니다. 사유를 비우면 취소가 진행되지 않습니다. 포장 단계 이후 주문은 취소 버튼이 보이지 않습니다.
- **Pass 기준**: 취소 가능 주문은 사유 입력 후 취소되고 상태가 갱신되며, 취소 불가 주문은 차단되면 합격. (도메인 규칙: 취소 시 서버에서 SUBMITTED/CONFIRMED → CANCELLED로 전이되며 예약 재고는 자동 복원됩니다.)

### 4.20 공지사항 — 목록 열람 및 읽음 처리

- **목적**: 본인 거래처에 적용되는 공지를 열람하고 읽음 처리·미읽음 배지가 동작하는지 확인합니다.
- **사전조건**: 알티바이오가 등록한 공지가 1건 이상이고, 그 중 본인 거래처(전체/대리점/병원/특정 거래처 대상)에 해당하는 공지가 있는 상태.
- **테스트 단계**:
  1. 하단 탭 "공지"에 빨간 미읽음 배지(숫자)가 표시되는지 확인합니다.
  2. 하단 탭 "공지"를 클릭해 공지사항 목록으로 이동합니다.
  3. 공지 항목의 제목([고정]/[중요] 표시·NEW 배지), 작성자, 작성일, 만료일(있는 경우), 본문을 확인합니다.
  4. 공지를 본 뒤 하단 "공지" 탭의 미읽음 배지가 사라지거나 줄어드는지 확인합니다.
  5. 홈 화면 상단의 공지 미니 영역(상위 3건)과 "전체보기 ›" 링크도 확인합니다.
- **기대 결과**: 본인 대상 공지만 표시되며, 고정/중요 공지가 상단에 정렬됩니다. 페이지 진입 시 미읽음 공지가 읽음 처리되어 NEW 배지·하단 미읽음 배지가 갱신됩니다. 대상 공지가 없으면 "받은 공지사항이 없습니다." 안내가 나옵니다.
- **Pass 기준**: 본인 대상 공지가 정렬되어 표시되고, 열람 후 미읽음 배지가 정상 갱신되면 합격.

### 4.21 공지사항 — 로그인 직후 팝업 / 다시 보지 않기

- **목적**: 로그인 직후 우선순위가 높은 미읽음 공지가 팝업으로 뜨고 "다시 보지 않기"가 동작하는지 확인합니다.
- **사전조건**: 본인 거래처 대상 미읽음 공지가 1건 이상인 상태에서 새로 로그인.
- **테스트 단계**:
  1. 로그아웃 후 다시 로그인합니다.
  2. 로그인 직후(약간의 지연 후) 공지 팝업이 뜨는지 확인합니다.
  3. 팝업의 제목·작성자·날짜·본문을 확인합니다.
  4. (선택) "다시 보지 않기" 체크박스를 켜고 "확인"을 누릅니다.
- **기대 결과**: 가장 우선순위 높은(고정·중요 우선) 미읽음 공지가 팝업으로 표시됩니다. "다시 보지 않기"를 켜고 확인하면 해당 공지가 읽음 처리되어 다시 팝업되지 않습니다.
- **Pass 기준**: 미읽음 공지가 있을 때 팝업이 뜨고, "다시 보지 않기" 처리가 반영되면 합격. (참고: 미읽음 대상 공지가 없으면 팝업이 뜨지 않는 것이 정상입니다.)

### 4.22 내 계정 정보 확인

- **목적**: 내 계정 화면에 거래처 정보가 정확히 표시되는지 확인합니다.
- **사전조건**: 로그인 완료.
- **테스트 단계**:
  1. 하단 탭 "계정"을 클릭합니다.
  2. 거래처명·담당자·유형·이메일·전화·결제조건·마감기간·주소를 확인합니다.
- **기대 결과**: 본인 거래처 정보가 정확히 표시됩니다(미입력 항목은 "-"로 표시될 수 있음).
- **Pass 기준**: 표시된 거래처 정보가 실제 등록 정보와 일치하면 합격.

### 4.23 권한 / 접근 제어 (참고 확인)

- **목적**: CLIENT 권한 사용자가 본인 거래처 데이터만 보는지, 비정상 접근이 차단되는지 확인합니다.
- **사전조건**: 거래처 계정으로 로그인.
- **테스트 단계**:
  1. 발주 내역·홈에서 다른 거래처의 주문이 보이지 않는지 확인합니다.
  2. 공지사항에서 본인 대상이 아닌 공지가 보이지 않는지 확인합니다.
  3. (가능하면) 로그아웃 상태에서 `http://localhost:3000/client` 직접 접근 시 로그인 페이지로 이동하는지 확인합니다.
- **기대 결과**: 본인 거래처 데이터만 노출되며, 미인증 접근은 로그인 화면으로 유도됩니다. CLIENT가 아닌 사용자는 각자의 포털로 자동 이동됩니다.
- **Pass 기준**: 타 거래처 데이터 미노출 및 미인증 접근 차단이 확인되면 합격.

### 5.1 로그인 및 포털 진입

- **목적**: 대표 계정으로 정상 로그인하여 대표 포털에 진입하는지 확인.
- **사전조건**: 로그인 계정 `owner@rtbio.com`과 관리자가 안내한 비밀번호.
- **테스트 단계**:
  1. 브라우저 주소창에 `http://localhost:3000/ceo` 입력 후 이동.
  2. 로그인 화면이 나오면 이메일에 `owner@rtbio.com`, 비밀번호에 안내받은 비밀번호 입력.
  3. 로그인 버튼 클릭.
- **기대 결과**: 대표 포털 화면이 열리고, 왼쪽 사이드바 사용자 영역에 **정대표 / 대표이사**가 표시되며, 기본으로 **임원 대시보드** 화면이 보입니다.
- **Pass 기준**: 로그인 성공 후 `/ceo` 임원 대시보드가 보이면 합격. 로그인 실패·다른 포털로 튕김·빈 화면이면 불합격.

### 5.2 임원 대시보드 — 화면 진입 및 위젯 표시

- **목적**: 임원 대시보드가 위젯(카드/차트)들과 함께 정상적으로 그려지는지 확인.
- **사전조건**: 5.1 로그인 완료.
- **테스트 단계**:
  1. 왼쪽 사이드바에서 **임원 대시보드** 클릭.
  2. 화면 상단 제목 **대표 대시보드** 와 우측 **➕ 위젯 추가** 버튼, 좌측 **⋮(더보기)** 버튼이 보이는지 확인.
  3. 그 아래 위젯 영역(격자)에 매출/미수금 등 카드 또는 차트 위젯이 채워지는지 확인.
- **기대 결과**: 기본 위젯 4종(이번 달 매출, 총 미수금, 진행 주문, 활성 거래처 류)이 격자에 표시됩니다.
- **Pass 기준**: 제목·버튼·위젯이 모두 보이면 합격. 위젯 영역이 완전히 비어 있거나 "불러오는 중…"에서 멈추면 불합격.

### 5.3 임원 대시보드 — 위젯 추가 (갤러리에서 선택)

- **목적**: 갤러리에서 미리 만들어진 위젯을 골라 대시보드에 추가하는 기능 확인.
- **사전조건**: 임원 대시보드 화면.
- **테스트 단계**:
  1. 우측 상단 **➕ 위젯 추가** 클릭 → "위젯 추가" 모달이 열립니다.
  2. 모달 중간 **갤러리에서 선택** 영역에서 원하는 위젯 카드를 하나 클릭.
  3. 모달이 닫히고 대시보드로 돌아오는지 확인.
- **기대 결과**: 선택한 위젯이 대시보드 격자 맨 아래/빈 자리에 새로 추가됩니다.
- **Pass 기준**: 새 위젯이 대시보드에 나타나면 합격. 모달이 닫히지 않거나 위젯이 추가되지 않으면 불합격.

---

### 5.4 임원 대시보드 — 위젯 추가 (직접 만들기)

- **목적**: 데이터 소스·집계·필터를 직접 골라 사용자 정의 위젯을 만드는 기능 확인.
- **사전조건**: "위젯 추가" 모달이 열린 상태.
- **테스트 단계**:
  1. **➕ 위젯 추가** 클릭 → 모달의 **갤러리에서 선택** 헤더 우측 **+ 직접 만들기** 버튼 클릭.
  2. "위젯 직접 만들기" 모달이 열리면 **위젯 제목**(예: `이번 달 매출`) 입력.
  3. **데이터 소스**, **차트 종류** 선택 → **측정값(집계)**, 필요 시 **분류(그룹)** 선택.
  4. (선택) **+ 조건** 버튼으로 필터 조건 추가, **정렬 기준·정렬·최대 행** 지정.
  5. 우측 **미리보기** 영역에 실데이터 미리보기가 나타나는지 확인.
  6. 하단 **저장** 버튼 클릭(소스·측정값을 채우면 활성화됨).
- **기대 결과**: 입력한 제목/구성대로 위젯이 대시보드에 추가됩니다.
- **Pass 기준**: 미리보기가 표시되고 저장 후 새 위젯이 대시보드에 나타나면 합격. 미리보기가 안 뜨거나 저장 버튼이 끝까지 비활성(disabled)이면 해당 점 기록 후 불합격 여부 판단.

---

### 5.5 임원 대시보드 — AI로 위젯 만들기

- **목적**: "위젯 추가" 모달의 AI 입력으로 대화하듯 위젯을 생성/추천받는 기능 확인.
- **사전조건**: "위젯 추가" 모달이 열린 상태. (AI 연동은 환경에 따라 동작하지 않을 수 있으므로, 입력창 표시 여부 위주로 점검)
- **테스트 단계**:
  1. **➕ 위젯 추가** 클릭 → 모달 상단 **🤖 AI에게 말하기** 영역 확인.
  2. 입력창에 요청 문구(예: `이번 달 거래처별 매출 보여줘`)를 입력하고 전송.
- **기대 결과**: AI 응답으로 위젯 추천 카드/미리보기가 표시됩니다(연동 환경일 때).
- **Pass 기준**: AI 입력 영역이 정상 표시되면 1차 합격. 추천 결과가 떠서 추가까지 되면 완전 합격. (연동 미설정 시 입력 영역 표시만 확인하고 "환경 의존"으로 기록)

### 5.6 임원 대시보드 — 위젯 수정 / 복제 / 새로고침 / 삭제

- **목적**: 개별 위젯의 컨텍스트 메뉴 동작 확인.
- **사전조건**: 대시보드에 위젯이 1개 이상 있음.
- **테스트 단계**:
  1. 위젯에서 컨텍스트 메뉴(위젯 우측 상단 메뉴 또는 우클릭)를 열어 항목을 확인: **✏ 위젯 수정 / 복제 / 새로고침 / 삭제**.
  2. **✏ 위젯 수정** 클릭 → "위젯 수정" 모달에서 **위젯 제목** 변경 후 **저장**.
  3. 같은 위젯에서 **복제** 클릭 → 동일 위젯이 하나 더 생기는지 확인.
  4. **새로고침** 클릭 → 위젯 데이터가 다시 불러와지는지 확인.
  5. 복제한 위젯에서 **삭제** 클릭 → 위젯이 사라지는지 확인.
- **기대 결과**: 제목 변경 반영, 복제본 추가, 새로고침 동작, 삭제 시 제거가 각각 정상 처리됩니다.
- **Pass 기준**: 4개 동작(수정/복제/새로고침/삭제)이 모두 의도대로 작동하면 합격. 하나라도 무반응이면 해당 항목 기록 후 불합격.

### 5.7 임원 대시보드 — 레이아웃 이동/크기 조절 저장

- **목적**: 위젯을 드래그로 옮기거나 크기를 조절하면 레이아웃이 저장되는지 확인.
- **사전조건**: 대시보드에 위젯이 2개 이상 있음.
- **테스트 단계**:
  1. 위젯 하나를 드래그하여 다른 위치로 이동.
  2. 위젯 모서리(우하단/좌하단 핸들)를 끌어 크기를 변경.
  3. 화면을 새로고침(F5)하여 다시 진입.
- **기대 결과**: 새로고침 후에도 변경한 위치/크기가 그대로 유지됩니다.
- **Pass 기준**: 새로고침 후 레이아웃이 유지되면 합격. 원래대로 되돌아가면 불합격.

### 5.8 임원 대시보드 — 내보내기 / 가져오기 / 기본 레이아웃 초기화

- **목적**: 대시보드 구성을 파일로 저장(내보내기)·복원(가져오기)하고, 기본 레이아웃으로 되돌리는 기능 확인.
- **사전조건**: 대시보드 화면.
- **테스트 단계**:
  1. 상단 제목 옆 **⋮(더보기)** 버튼 클릭 → 메뉴 열림.
  2. **대시보드 내보내기** 클릭 → `.json` 파일이 다운로드되는지 확인.
  3. 다시 **⋮** → **대시보드 가져오기** 클릭 → 방금 받은 `.json` 파일 선택 → 대시보드가 그 구성으로 바뀌는지 확인.
  4. 다시 **⋮** → **기본 레이아웃으로 초기화** 클릭 → 기본 위젯 4종 구성으로 되돌아가는지 확인.
- **기대 결과**: 내보내기 시 파일 다운로드, 가져오기 시 구성 복원, 초기화 시 기본 4위젯 복귀.
- **Pass 기준**: 세 동작이 모두 작동하면 합격. 파일이 안 받아지거나 가져오기/초기화가 반영 안 되면 불합격.

### 5.9 통합 현황 — 전사 KPI / 팀별 요약 / 알림 / 차트

- **목적**: 전사 통합 현황 페이지의 KPI 4종, 팀별 요약, 중요 알림, 팀별 차트가 표시되는지 확인.
- **사전조건**: 5.1 로그인 완료.
- **테스트 단계**:
  1. 왼쪽 사이드바 **통합 현황** 클릭.
  2. 상단 보라색 안내 배너 **임원진 전용 페이지** 표시 확인.
  3. **KPI 4종** 카드 확인: 이번 달 총 매출 / 총 출고 건수 / 총 미수금 / 활성 거래처.
  4. **팀별 상태 요약** 영역에 품질관리팀·경영지원팀·영업팀 카드 3개 확인.
  5. **중요 알림** 영역에 재고 부족·미수금 위험·출고 한도 등 알림 카드 확인.
  6. 하단 **팀별 이번 달 처리 건수** 막대 차트가 그려지는지 확인.
- **기대 결과**: KPI·팀 카드·알림·차트가 모두 채워져 표시됩니다.
- **Pass 기준**: 4개 블록이 모두 표시되면 합격. 차트가 빈 사각형이거나 카드가 비어 있으면 불합격.

### 5.10 통합 현황 — 알림에서 팀 포털로 이동

- **목적**: 중요 알림 카드의 **확인** 버튼으로 해당 팀 포털로 이동하는 동작 확인.
- **사전조건**: 통합 현황 페이지의 중요 알림이 보이는 상태.
- **테스트 단계**:
  1. **재고 부족 경고** 알림의 **확인** 버튼 클릭.
  2. "해당 팀 포탈로 이동할까요?" 확인창에서 **확인** 클릭.
- **기대 결과**: 품질관리팀 포털(`qc-portal.html`)로 이동합니다. (미수금 위험 알림의 확인은 경영지원팀으로 이동)
- **Pass 기준**: 확인 시 올바른 팀 포털로 이동하면 합격. 취소 시 현재 페이지에 머물면 정상. 이동이 안 되면 불합격.

### 5.11 직원별 지표 — 요약·순위표·상세 카드

- **목적**: 직원별 업무 지표 페이지의 요약 통계, 순위표, 상세 카드가 표시되는지 확인.
- **사전조건**: 5.1 로그인 완료.
- **테스트 단계**:
  1. 왼쪽 사이드바 **직원별 지표** 클릭.
  2. 상단 **요약 통계** 4종 확인: 이번 달 총 처리 / 평균 완료율 / 평균 처리시간 / 현재 가동 인원.
  3. **순위표**(순위·직원·처리건수·평균처리시간·완료율·현재작업)가 처리건수 내림차순으로 정렬되는지 확인.
  4. **직원별 상세** 카드들이 직원별로 펼쳐지는지 확인.
- **기대 결과**: 요약 4종·순위표·상세 카드가 모두 표시되며, 1위 직원이 처리건수가 가장 많습니다.
- **Pass 기준**: 세 영역이 모두 채워지고 순위가 내림차순이면 합격. 빈 표/카드면 불합격.

### 5.12 직원별 지표 — 영업사원 실적 및 추이 차트

- **목적**: 영업사원 실적 카드와 최근 3개월 매출 추이 라인 차트 확인.
- **사전조건**: 직원별 지표 페이지.
- **테스트 단계**:
  1. **직원별 지표** 페이지에서 아래로 스크롤하여 **영업사원 실적** 섹션 확인.
  2. 영업사원 카드(매출·발주 건수·신규 거래처·담당 거래처)가 표시되는지 확인.
  3. 그 아래 **영업사원별 최근 3개월 매출 추이** 라인 차트(2월/3월/4월)가 그려지는지 확인.
- **기대 결과**: 영업사원 카드와 라인 차트가 모두 표시됩니다.
- **Pass 기준**: 카드·차트가 표시되면 합격. 차트가 비어 있으면 불합격.

### 5.13 공지사항 — 목록 조회 및 검색/필터

- **목적**: 공지사항 목록이 표시되고 검색·대상·긴급도·작성자 필터가 동작하는지 확인.
- **사전조건**: 5.1 로그인 완료. (목록에 공지가 1건 이상 있으면 좋음)
- **테스트 단계**:
  1. 왼쪽 사이드바 **📢 공지사항** 클릭 → 상단 제목 **공지사항** 확인.
  2. **검색** 입력창에 키워드를 입력하면 목록이 즉시 필터되는지 확인.
  3. **대상** 드롭다운(전체/전체 거래처/대리점만/병원만/특정 업체) 변경 시 목록 변화 확인.
  4. **긴급도**(전체/긴급/일반), **작성자**(경영지원팀/영업팀/품질관리팀/임원진) 필터도 각각 변경해 확인.
- **기대 결과**: 각 필터 조건에 맞는 공지만 목록에 남습니다. 조건에 맞는 공지가 없으면 "조건에 맞는 공지가 없습니다." 안내가 표시됩니다.
- **Pass 기준**: 검색·필터가 목록에 즉시 반영되면 합격. 필터를 바꿔도 변화가 없으면 불합격.

### 5.14 공지사항 — 신규 공지 작성/발송

- **목적**: 대표(임원진)가 거래처로 공지를 작성·발송하는 핵심 흐름 확인.
- **사전조건**: 공지사항 페이지.
- **테스트 단계**:
  1. 우측 상단 **＋ 공지 작성** 클릭 → "공지사항 작성 — 🔴 임원진" 모달이 열림(작성자가 임원진으로 자동 설정됨).
  2. **제목 \***, **본문 \*** 입력(둘 다 필수).
  3. **긴급도**(일반/긴급), **발송 대상**(전체 거래처/대리점만/병원만/특정 업체), **만료일**(선택) 지정.
  4. (선택) **상단 고정 (📌 표시)** 체크.
  5. 하단 발송(확인) 버튼 클릭.
- **기대 결과**: "공지사항이 발송되었습니다" 토스트가 뜨고, 목록 맨 위에 방금 작성한 공지가 작성자 **🔴 임원진** 배지와 함께 나타납니다.
- **Pass 기준**: 제목·본문 입력 후 발송 시 목록에 추가되면 합격. 제목/본문을 비우면 "제목과 본문은 필수입니다" 안내가 떠야 정상.

### 5.15 공지사항 — "특정 업체" 대상 선택

- **목적**: 발송 대상을 "특정 업체"로 선택했을 때 업체 선택 위젯(검색·전체선택·칩)이 동작하는지 확인.
- **사전조건**: 공지 작성 모달이 열린 상태.
- **테스트 단계**:
  1. **＋ 공지 작성** → **발송 대상** 드롭다운에서 **특정 업체** 선택.
  2. 나타나는 **대상 업체 선택** 영역에서 **🔍 업체명/유형 검색** 입력 → 목록이 검색어로 필터되는지 확인.
  3. 업체 체크박스를 선택하면 위쪽 칩 영역에 선택된 업체가 칩으로 표시되는지 확인.
  4. **검색결과 전체선택** / **전체해제** 버튼 동작 확인.
  5. 제목·본문 입력 후 발송 시도.
- **기대 결과**: 선택한 업체가 칩으로 표시되고, 최소 1개 선택 시 발송됩니다.
- **Pass 기준**: 특정 업체를 0개 선택하고 발송하면 "대상 업체를 1개 이상 선택해주세요" 안내가 떠야 합격. 1개 이상 선택 시 정상 발송되면 합격.

### 5.16 공지사항 — 미리보기

- **목적**: 작성된 공지를 발송 화면 형태로 미리 보는 기능 확인.
- **사전조건**: 목록에 공지 1건 이상.
- **테스트 단계**:

  2. 제목·작성자·작성일·본문·대상·만료가 담긴 미리보기 팝업이 열리는지 확인.
- **기대 결과**: 공지 내용이 깔끔한 카드 형태로 표시됩니다.
- **Pass 기준**: 미리보기 팝업이 정상 표시되면 합격. 빈 팝업/무반응이면 불합격.

### 5.17 공지사항 — 편집

- **목적**: 기존 공지의 제목/본문/긴급도/상단고정/만료일을 수정하는 기능 확인. (발송 대상은 수정 불가)
- **사전조건**: 목록에 공지 1건 이상.
- **테스트 단계**:
  1. 공지 항목 우측 **편집** 버튼 클릭 → "공지 편집" 모달이 열림.
  2. **제목** 또는 **본문**을 변경하고, 필요 시 **긴급도/만료일/상단 고정** 변경.
  3. 하단 **저장** 클릭.
- **기대 결과**: "공지사항이 수정되었습니다" 토스트가 뜨고 목록에 변경 내용이 반영됩니다. (모달 내 안내처럼 발송 대상은 변경되지 않음)
- **Pass 기준**: 편집 후 목록에 반영되면 합격. **편집을 눌렀는데 삭제가 되면 불합격**(편집/삭제 분리 동작 확인 포인트).

### 5.18 공지사항 — 삭제

- **목적**: 공지를 삭제하는 기능과 삭제 확인 절차 확인.
- **사전조건**: 목록에 공지 1건 이상(테스트용으로 직접 작성한 공지 권장).

  2. 안내 문구(복구 불가) 확인 후 **삭제** 클릭.
- **기대 결과**: "공지사항이 삭제되었습니다" 토스트가 뜨고 목록에서 사라집니다.
- **Pass 기준**: 삭제 확인 후 목록에서 제거되면 합격. 취소 시 그대로 남아 있어야 정상.

### 5.19 직원 관리 — 목록 조회 및 검색/필터

- **목적**: 대표(TENANT_OWNER)만 보이는 직원 관리 페이지에서 직원 목록·검색·필터가 동작하는지 확인.
- **사전조건**: 5.1 로그인 완료(대표 계정이라 메뉴가 보임).
- **테스트 단계**:
  1. 왼쪽 사이드바 **직원 관리** 클릭(대표/관리자만 노출됨) → 상단 제목 **직원 관리** 확인.
  2. **이름 / 이메일 검색** 입력창에 키워드를 넣으면 목록이 즉시 좁혀지는지 확인.
  3. **직급 필터**(전체 직급/각 직급), **상태 필터**(활성/전체 상태/비활성)를 변경해 목록 변화 확인.
- **기대 결과**: 검색/필터 조건에 맞는 직원만 목록에 표시됩니다.
- **Pass 기준**: 검색·필터가 즉시 반영되면 합격. 직원 관리 메뉴가 아예 안 보이면(대표 계정인데 미노출) 불합격으로 기록.

---

### 5.20 직원 관리 — 신규 직원 등록

- **목적**: 새 직원 계정을 생성하는 기능 확인.
- **사전조건**: 직원 관리 페이지.
- **테스트 단계**:
  1. **+ 신규 직원** 버튼 클릭 → 직원 등록 폼 모달이 열림.
  2. 이름·이메일·직급(권한)·부서/직급 등 필수 항목 입력.
  3. **등록** 버튼 클릭.
- **기대 결과**: 새 직원이 목록에 추가됩니다.
- **Pass 기준**: 등록 후 목록에 새 직원이 나타나면 합격. 필수값 누락 시 안내가 떠야 정상.

---

### 5.21 직원 관리 — 직원 정보 수정 / 비밀번호 재발급

- **목적**: 기존 직원의 정보 수정과 비밀번호 재발급 기능 확인.
- **사전조건**: 직원 관리 목록에 직원 1명 이상.
- **테스트 단계**:
  1. 직원 항목의 수정 진입(편집 버튼) → 폼에서 직급/부서 등 변경 후 **저장**.
  2. 같은 직원에서 비밀번호 재발급 진입 → **재발급** 버튼 클릭.
- **기대 결과**: 수정 내용이 목록에 반영되고, 비밀번호 재발급이 처리됩니다.
- **Pass 기준**: 수정 반영·재발급 처리가 정상이면 합격. 무반응/오류면 불합격.

---

### 5.22 직원 관리 — 부서·직급 관리 (조직 옵션)

- **목적**: 부서/직급 목록을 추가·관리하는 기능 확인.
- **사전조건**: 직원 관리 페이지(메타관리자 전용 버튼 노출 시).
- **테스트 단계**:
  1. 검색 바 우측 **⚙ 부서·직급 관리** 버튼 클릭(노출되는 경우) → 관리 모달.
  2. 부서/직급 항목 입력 후 **추가** 클릭.
- **기대 결과**: 추가한 부서/직급이 목록과 직원 등록 폼의 선택지에 반영됩니다.
- **Pass 기준**: 추가/삭제가 반영되면 합격. 버튼이 보이지 않으면 권한 조건(메타관리자)으로 기록.

---

### 5.23 팀 관리자 지정

- **목적**: 대표/관리자가 특정 직원에게 팀 관리자 권한을 부여/회수하는 기능 확인.
- **사전조건**: 5.1 로그인 완료. 직원 1명 이상 존재.
- **테스트 단계**:
  1. 왼쪽 사이드바 **팀 관리자 지정** 클릭(메타관리자 전용 노출) → 상단 제목 **팀 관리자 지정** 확인.
  2. 직원 목록에서 대상 직원의 팀 관리자 지정/해제 동작 수행.
- **기대 결과**: 지정/해제가 목록에 반영됩니다.
- **Pass 기준**: 지정/해제가 반영되면 합격. 메뉴가 안 보이면 권한 조건으로 기록.

---

### 5.24 본인 비밀번호 변경

- **목적**: 로그인한 대표 본인의 비밀번호를 변경하는 기능 확인.
- **사전조건**: 5.1 로그인 완료.
- **테스트 단계**:
  1. 사이드바 사용자 영역 아래 **🔑 비밀번호 변경** 링크 클릭 → 비밀번호 변경 모달.
  2. 현재 비밀번호·새 비밀번호 입력 후 **변경** 클릭.
- **기대 결과**: 비밀번호 변경 성공 안내가 표시됩니다.
- **Pass 기준**: 변경 처리가 정상이면 합격. (실제로 변경했다면 다음 로그인 시 새 비밀번호로 로그인되는지까지 확인하면 더 좋음)

---

### 5.25 팀 포털 이동

- **목적**: 사이드바 "팀 포탈 이동" 링크로 다른 팀 포털에 진입하는지 확인.
- **사전조건**: 5.1 로그인 완료.
- **테스트 단계**:
  1. 사이드바 하단 **팀 포탈 이동** 묶음에서 **품질관리팀** 클릭 → QC 포털로 이동 확인.
  2. 뒤로가기 후 **경영지원팀**, **영업팀** 링크도 각각 이동되는지 확인.
- **기대 결과**: 각각 QC/경영지원/영업 포털 화면으로 이동합니다.
- **Pass 기준**: 세 링크가 모두 올바른 포털로 이동하면 합격.

---

### 5.26 역할 전환 (로그아웃)

- **목적**: 대표 포털에서 로그아웃하여 로그인 화면으로 돌아가는지 확인.
- **사전조건**: 5.1 로그인 완료.
- **테스트 단계**:
  1. 사이드바 맨 아래 **← 역할 전환 (로그아웃)** 클릭.
- **기대 결과**: 로그아웃 처리 후 로그인 화면(`/login`)으로 이동합니다.
- **Pass 기준**: 로그인 화면으로 이동하면 합격. 로그아웃이 안 되거나 대시보드에 머물면 불합격.

---

### 5.27 (참고) 모바일/반응형 사이드바

- **목적**: 좁은 화면에서 햄버거 메뉴로 사이드바를 여닫는 동작 확인.
- **사전조건**: 브라우저 창 폭을 좁히거나 모바일 기기로 접속.
- **테스트 단계**:
  1. 화면 폭을 좁히면 상단에 햄버거(☰) 버튼과 **대표** 제목이 보입니다.
  2. 햄버거 버튼을 눌러 사이드바를 펼치고, 메뉴 클릭 후 자동으로 닫히는지 확인.
- **기대 결과**: 햄버거로 사이드바가 열리고, 메뉴 선택 시 닫힙니다.
- **Pass 기준**: 열기/닫기와 메뉴 이동이 정상이면 합격.
