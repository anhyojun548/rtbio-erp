## 0. 시작하기 (공통)

이 장은 RTBIO ERP를 처음 사용하시는 모든 분을 위한 공통 안내입니다. 어느 팀에 속해 계시든 먼저 이 장을 읽어 주시면, 로그인부터 화면 조작, 그리고 우리 회사 업무가 시스템에서 어떻게 흘러가는지 한눈에 이해하실 수 있습니다.

---

### 0-1. 로그인 방법

#### 접속 주소

웹 브라우저(크롬 권장)를 열고 주소창에 아래 주소를 입력해 주세요.

```
localhost:3000
```

> 사내망 또는 정식 서비스 주소로 운영될 때는 담당자가 안내드리는 주소(예: `altibio.rtbio-erp.com`)를 사용하시면 됩니다.

#### 로그인 절차

1. 접속하면 **로그인 화면**이 나타납니다.
2. 발급받으신 **이메일**과 **비밀번호**를 입력합니다.
3. **로그인** 버튼을 누릅니다.
4. 로그인에 성공하면, **본인 권한에 맞는 포털 화면으로 자동 이동**합니다. (예: 영업 담당자는 영업 포털, 품질팀은 QC 포털로 바로 들어갑니다.)

> 비밀번호를 5회 이상 잘못 입력하면 잠시 후 다시 시도하셔야 할 수 있습니다. 비밀번호를 잊으셨다면 경영지원팀(시스템 관리자)에 문의해 초기화를 요청해 주세요.

#### 로그인 계정표 (테스트/초기 계정)

아래는 알티바이오 초기 샘플 계정입니다. **공통 비밀번호는 `rtbio1234!`** 이며, 첫 로그인 후 반드시 비밀번호를 변경해 주세요.

| 이메일 | 이름 | 역할 | 이동 포털 | 비밀번호 |
| --- | --- | --- | --- | --- |
| `owner@rtbio.com` | 이대표 | 대표(TENANT_OWNER) | 대표 포털(CEO) | `rtbio1234!` |
| `admin@rtbio.com` | 김경영 | 경영지원(ADMIN) | 경영지원 포털(Admin) | `rtbio1234!` |
| `qc@rtbio.com` | 박품질 | 품질관리(QC) | 품질관리 포털(QC) | `rtbio1234!` |
| `sales1@rtbio.com` | 최영업 | 영업(EXEC) | 영업 포털(Exec) | `rtbio1234!` |
| `sales2@rtbio.com` | 정영업 | 영업(EXEC) | 영업 포털(Exec) | `rtbio1234!` |

#### 거래처(대리점) 계정

거래처 대리점은 **거래처 포털**로 로그인합니다. 거래처 계정 이메일은 **거래처 코드를 소문자로 바꾼 뒤 `@client.local`을 붙인 형식**입니다.

| 거래처 | 거래처 코드 | 로그인 이메일 | 비밀번호 |
| --- | --- | --- | --- |
| 서울메디칼 대리점 | C-AGEN-001 | `c-agen-001@client.local` | `rtbio1234!` |
| 부산의료기 대리점 | C-AGEN-002 | `c-agen-002@client.local` | `rtbio1234!` |
| 대전메드 대리점 | C-AGEN-003 | `c-agen-003@client.local` | `rtbio1234!` |

> 즉, 거래처 계정 이메일 = **`{거래처코드 소문자}@client.local`** 입니다. 새 대리점이 등록되면 같은 규칙으로 계정이 만들어집니다.

### 0-2. 화면 공통 요소 사용법

모든 포털은 **상단바 + 좌측 사이드바 + 본문**의 동일한 구조를 가집니다. 한 번 익혀 두시면 어느 포털에서나 똑같이 사용하실 수 있습니다.

#### 상단바 (화면 맨 위)

| 요소 | 위치 | 하는 일 |
| --- | --- | --- |
| 사용자명 | 우측 | 현재 로그인한 본인 이름과 역할이 표시됩니다. |
| 알림 종(🔔) | 우측 | 새 알림이 있으면 빨간 숫자 배지가 뜹니다. 클릭하면 알림 목록이 열립니다. |
| 비밀번호 변경 | 우측 메뉴 | 사용자명을 클릭하면 나오는 메뉴에서 비밀번호를 바꿀 수 있습니다. |
| 로그아웃 | 우측 메뉴 | 사용을 마치면 반드시 로그아웃해 주세요. (공용 PC라면 특히 중요합니다.) |

> **비밀번호 변경 방법**: 상단바 우측의 본인 이름을 클릭 → **비밀번호 변경** 선택 → 현재 비밀번호와 새 비밀번호를 입력 → 저장.

#### 좌측 사이드바 (메뉴 이동)

- 화면 왼쪽 세로 영역이 **메뉴판**입니다. 클릭하면 해당 업무 화면으로 이동합니다.
- 현재 보고 있는 메뉴는 **색이 진하게 강조**되어 어디에 있는지 쉽게 알 수 있습니다.
- 보이는 메뉴는 **본인 권한(역할)에 따라 다릅니다.** (아래 0-5 참고)

#### 공지 팝업 닫는 법

- 로그인 직후 또는 새 공지가 등록되면 화면 가운데에 **공지 팝업**이 뜰 수 있습니다.
- 내용을 확인하신 뒤 팝업의 **닫기(X)** 버튼 또는 **확인** 버튼을 누르면 사라집니다.
- "오늘 하루 보지 않기"가 있으면 체크 후 닫으면 같은 공지가 그날은 다시 뜨지 않습니다.

#### 토스트 알림 (자동으로 떴다 사라지는 메시지)

- 저장·삭제·발급 같은 작업을 하면 화면 모서리에 작은 알림 띠(토스트)가 잠깐 나타납니다.
  - **초록색**: 정상 처리됨 (예: "저장되었습니다")
  - **빨간색**: 오류 발생 (예: "재고가 부족합니다") — 메시지를 읽고 다시 시도해 주세요.
- 토스트는 몇 초 뒤 자동으로 사라지므로 따로 닫지 않으셔도 됩니다.

#### 표(목록)의 공통 조작

대부분의 목록 화면(주문, 거래처, 재고, 명세서 등)은 동일한 방식으로 다룹니다.

| 기능 | 사용법 |
| --- | --- |
| 검색 | 표 위의 **검색창**에 이름·코드·번호 일부를 입력하면 일치하는 행만 추려집니다. (대소문자 구분 없음) |
| 필터 | **상태·기간·유형** 등의 드롭다운/날짜 선택으로 원하는 조건만 골라 봅니다. (예: 주문 상태 "확정", 기간 "이번 달") |
| 정렬 | 표의 **열 제목을 클릭**하면 그 기준으로 오름차순↔내림차순 정렬됩니다. (예: 금액 큰 순) |
| 페이지네이션 | 행이 많으면 아래쪽 **페이지 번호(◀ 1 2 3 ▶)** 로 나뉘어 표시됩니다. 번호를 눌러 이동합니다. |

> 검색·필터를 적용한 상태는 보통 주소(URL)에 반영되므로, 그 상태 그대로 새로고침하거나 동료에게 링크로 공유할 수 있습니다.

### 0-3. 핵심 용어 사전

업무에서 자주 쓰는 용어를 쉬운 말로 정리했습니다. 시스템 화면에서 이 단어들을 만나면 아래 뜻으로 이해하시면 됩니다.

| 용어 | 쉬운 설명 |
| --- | --- |
| **발주(주문)** | 거래처(대리점/병원)가 "이 제품을 이만큼 보내 주세요"라고 넣는 주문입니다. 처음에는 임시저장(작성 중) 상태로 만들어집니다. |
| **발주 확정** | 임시 주문을 "이대로 진행"하겠다고 확정하는 단계입니다. 확정되면 정식 주문번호가 매겨지고, 주문 수량만큼 재고가 **예약**됩니다. |
| **예약** | 주문이 확정되면 그 수량만큼 재고를 "찜"해 둡니다. 실제 창고 수량은 그대로지만, **다른 주문에 팔 수 없는 상태**가 됩니다. |
| **출고 시작** | 확정된 주문을 실제로 보내기 위해 창고 작업(피킹·포장 등)을 시작하는 것입니다. 칸반 보드의 첫 단계로 들어갑니다. |
| **출고 완료** | 물품이 최종적으로 출하된 상태입니다. 이때 **실물재고가 실제로 차감**되고 주문은 "완료"로 바뀝니다. |
| **칸반 단계** | 출고 진행 상황을 보여 주는 단계별 보드입니다. 카드(주문)를 단계 칸으로 옮기며 진행합니다. 기본 단계: 접수대기 → 피킹 → 검수 → 포장 → 출고대기 → 출고완료. |
| **거래명세서** | 출고가 끝난 주문에 대해 발행하는 정식 거래 증빙(세금계산서 성격)입니다. 공급가액 + 부가세(VAT 10%)가 계산됩니다. |
| **수금** | 거래처가 보낸 입금을 시스템에 기록하는 것입니다. 명세서 금액과 실제 입금을 맞춰 관리합니다. |
| **거래처원장** | 거래처별로 "전월 잔액 + 이달 매출 − 이달 수금 = 이달 잔액(미수금)"을 월 단위로 정리한 장부입니다. |
| **월 마감** | 한 달의 거래처원장을 확정(잠금)하는 작업입니다. 마감 후에는 그 달 숫자가 임의로 바뀌지 않습니다. (필요 시 사유를 남기고 재오픈 가능) |
| **재고 — 실물재고** | 창고에 **실제로 놓여 있는** 수량입니다. 출고 완료 시점에 줄어듭니다. |
| **재고 — 가용재고** | 예약분을 뺀 **지금 더 팔 수 있는** 수량입니다. 주문이 확정되면 가용재고가 먼저 줄어듭니다. |
| **스냅샷 단가** | 주문(또는 배송지)을 확정하는 **그 시점의 가격·배송지 정보를 사진 찍듯 저장**해 두는 것입니다. 이후 제품 가격이나 거래처 주소가 바뀌어도 **과거 주문은 그대로 보존**됩니다. |
| **유통기한 로트(Lot)** | 같은 제품·사이즈라도 입고 묶음(로트)별로 유통기한이 다릅니다. 임박/만료 로트를 단계로 표시해 관리합니다. |

### 0-4. 전체 업무 흐름 한눈에

RTBIO ERP는 **하나의 주문이 거래처 → 품질팀 → 경영지원 → 대표**의 손을 거치며 완결되도록 설계되어 있습니다. 한 건의 주문이 어떻게 흘러가는지 이야기처럼 따라가 보겠습니다.

#### ① 거래처가 발주합니다 — *거래처 포털 (대리점 담당자)*

서울메디칼 대리점 담당자가 거래처 포털에 로그인해, 필요한 제품과 수량, 받을 배송지를 골라 **발주**를 넣습니다. 이때 거래처별 할인율·고정가가 자동 적용되어 **그 시점 단가가 스냅샷으로 저장**됩니다.

#### ② 영업이 확인하고, 품질팀이 확정·출고합니다 — *영업 포털 / 품질관리(QC) 포털*

- **영업 담당자**(영업 포털)는 본인이 맡은 거래처의 주문을 확인하고 챙깁니다.
- 주문이 **확정**되면 그 수량만큼 **가용재고가 예약**됩니다.
- **품질팀**(QC 포털)은 **출고 칸반 보드**에서 주문 카드를 접수대기 → 피킹 → 검수 → 포장 → 출고대기 → **출고완료** 단계로 옮기며 실제 출하를 진행합니다. 출고완료가 되면 **실물재고가 차감**되고 주문은 "완료"가 됩니다. (품질팀은 재고 현황·유통기한·샘플 출고도 함께 관리합니다.)

#### ③ 경영지원이 명세서를 발급하고 수금합니다 — *경영지원(Admin) 포털*

- 출고가 완료된 주문에 대해 경영지원팀이 **거래명세서**를 발급합니다. (공급가액 + 부가세 10% 자동 계산, PDF 출력 가능)
- 거래처가 대금을 입금하면 **수금**으로 기록하고, 명세서와 입금을 맞춰 둡니다.
- 매월 거래처별 **거래처원장**(전월 잔액 + 매출 − 수금 = 미수금)을 정리하고, 정산이 끝나면 **월 마감**을 합니다.

#### ④ 대표가 대시보드로 전체를 확인합니다 — *대표(CEO) 포털*

대표님은 CEO 포털의 **대시보드 위젯**으로 이번 달 매출, 미수금, 진행 중 주문, 만료 임박 계약 같은 핵심 지표를 한눈에 봅니다. 위젯은 보고 싶은 항목으로 직접 구성할 수 있습니다.

#### 요약: 한 건의 주문이 거치는 길

```
거래처 발주  →  영업 확인 / 확정(재고 예약)  →  품질팀 출고(칸반→출고완료, 실물재고 차감)
   거래처포털          영업포털                         QC포털
        →  경영지원 명세서 발급 + 수금 + 원장/마감  →  대표 대시보드 확인
                     경영지원포털                          CEO포털
```

> 핵심 원칙: **재고는 "예약(가용재고 차감) → 출고(실물재고 차감)"의 2단계로 관리**되고, **가격·배송지는 주문 시점에 스냅샷으로 고정**됩니다. 덕분에 나중에 가격이나 주소가 바뀌어도 과거 거래 기록은 항상 정확하게 유지됩니다.

### 0-5. 권한별로 보이는 메뉴가 다릅니다

RTBIO ERP는 **역할(권한)에 따라 들어가는 포털과 보이는 메뉴가 다릅니다.** 같은 시스템이지만 본인 업무에 필요한 화면만 깔끔하게 보이도록 설계되어 있습니다.

| 역할 | 들어가는 포털 | 주로 하는 일 |
| --- | --- | --- |
| 대표 (TENANT_OWNER) | 대표(CEO) 포털 | 대시보드로 전사 지표 확인, 위젯 구성. (필요 시 모든 포털 접근 가능) |
| 경영지원 (ADMIN) | 경영지원(Admin) 포털 | 거래처·제품·주문·재고·명세서·수금·원장·보고서·설정 등 전반 관리 |
| 품질관리 (QC) | 품질관리(QC) 포털 | 출고 칸반, 출고 내역, 재고 현황·알럼, 유통기한, 샘플 출고 |
| 영업 (EXEC) | 영업(Exec) 포털 | 내 거래처·주문 관리, 담당자별 매출, 학회 방명록 |
| 거래처 (CLIENT) | 거래처 포털 | 본인 회사의 발주·명세서·수금·계약·내 정보 조회 (읽기 위주) |
