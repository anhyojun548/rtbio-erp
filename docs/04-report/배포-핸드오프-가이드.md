# RTBIO ERP — 배포 핸드오프 가이드 (개발팀 전달용)

> 회사 서버에 올리는 개발팀이 **이 문서만 따라 하면 구동**되도록 작성. 2026-06-08 기준.

## 0. 한눈에
- **무엇**: 알티바이오 멀티테넌트 ERP. Next.js 14(App Router) + TypeScript + Prisma + PostgreSQL 16 + NextAuth.
- **코드**: GitHub `anhyojun548/rtbio-erp` (비공개) — `main` 브랜치. **저장소 접근 권한 부여**로 전달.
- **실행 방식**: Node 서버(`pnpm build && pnpm start`, 기본 포트 3000) + PostgreSQL 16.
- **앞단**: 리버스 프록시(nginx 등)로 HTTPS 종단 후 3000 으로 프록시 권장.

## 1. 사전 요구사항
| 항목 | 버전/비고 |
|---|---|
| Node.js | **20 LTS 이상** (`engines.node >=20`) |
| pnpm | **9 이상** (`corepack enable` 후 `corepack prepare pnpm@latest --activate`) |
| PostgreSQL | **16** (멀티스키마 사용: `public` + `tenant_altibio`) |
| (선택) Docker | 로컬/개발은 `docker compose` 로 PG 띄움. 운영 PG는 별도 권장 |

## 2. 코드 받기
```bash
git clone https://github.com/anhyojun548/rtbio-erp.git
cd rtbio-erp
pnpm install            # devDependencies 포함 (빌드에 필요)
```

## 3. 환경변수 (.env.local)
저장소의 `.env.example` 를 복사해 `.env.local` 생성 후 값 채움. **secret 은 운영용으로 새로 발급**.
```bash
cp .env.example .env.local
```
| 키 | 설명 | 운영 값 예시 |
|---|---|---|
| `DATABASE_URL` | Prisma 연결 (pooled) | `postgresql://USER:PW@DB호스트:5432/rtbio_erp?schema=public` |
| `DIRECT_URL` | 마이그레이션용 직결 | 보통 `DATABASE_URL` 과 동일(풀러 안 쓰면) |
| `NEXTAUTH_URL` | **실제 접속 도메인** | `https://erp.회사도메인.com` |
| `NEXTAUTH_SECRET` | 세션 서명 키 — **새로 생성** | `openssl rand -base64 32` 결과 |
| `NODE_ENV` | `production` | `production` |
| `LOG_LEVEL` | 로그 레벨 | `info` |
| `WIDGET_API_TOKEN` | 위젯/AI 외부연동 토큰(선택) | 미사용 시 비워도 됨 |

> ⚠️ `.env.local` / `.env` 는 `.gitignore` 에 있어 저장소에 안 올라갑니다. **값은 안전한 별도 채널로 전달**(메신저/사내 비밀관리). 절대 저장소 커밋 금지.

## 4. 데이터베이스 셋업
PostgreSQL 16 에 **빈 DB**(`rtbio_erp`) 생성 후, **둘 중 하나** 선택:

### (A) 깨끗한 시작 — 시드 데이터
```bash
pnpm prisma generate
pnpm prisma migrate deploy   # 스키마/테이블 생성
pnpm prisma:seed             # 초기 시드 데이터
```

### (B) 현재 운영 데이터 이관 — 별도 제공 덤프 복원 (이번 핸드오프 방식)
제공되는 `rtbio_erp_dump.sql` 에 **스키마 + 데이터 + 마이그레이션 이력이 모두 포함**되어 있습니다. 빈 DB 에 복원만 하면 끝 — **`migrate deploy` 불필요**(이미 들어있음).
```bash
psql "$DATABASE_URL" -f rtbio_erp_dump.sql   # 빈 DB 에 그대로 복원
pnpm prisma generate                          # Prisma Client 만 생성
```
> ⚠️ 덤프는 거래처/대표자/연락처 등 **개인정보 포함** → 안전 채널로 전달, 접근통제. 저장소에 커밋 금지.
> 이후 스키마 변경(새 마이그레이션)이 생기면 그때만 `pnpm prisma migrate deploy` 추가 실행.

## 5. 빌드 & 실행
```bash
pnpm build      # next build (typecheck/번들)
pnpm start      # next start, 0.0.0.0:3000 (운영)
```
- 헬스 체크: `curl http://localhost:3000/api/auth/providers` → 200 이면 정상.
- 프로세스 관리: **pm2** (`pm2 start "pnpm start" --name rtbio`) 또는 **systemd** 서비스로 상시 구동.

## 6. 운영 배포 (권장 구성)
```
[사용자] --HTTPS--> [nginx 리버스 프록시] --http--> [Node(next start) :3000] --> [PostgreSQL 16]
```
- nginx 에서 TLS 종단(Let's Encrypt), `proxy_pass http://127.0.0.1:3000;`
- `proxy_set_header Host $host;` / `X-Forwarded-Host` / `X-Forwarded-Proto $scheme;` 전달 (앱이 외부 호스트 인식에 사용).
- `NEXTAUTH_URL` = 외부 HTTPS 도메인과 **반드시 일치**.

## 9. 보안·주의 (개발팀 필독)
- `NEXTAUTH_SECRET` 는 **운영용으로 새로 발급** (개발 값 재사용 금지).
- `.env*` 절대 커밋 금지(이미 gitignore).
- `scripts/add-real-client-accounts.ts` (실거래처 계정 생성 스크립트)는 **민감정보 포함 → 저장소·핸드오프에 포함 금지**. (현재 미커밋 상태)
- DB 덤프 전달 시 개인정보(거래처/대표자/연락처) 포함 → 안전 채널·접근통제.
- 기본 시드 계정 비밀번호는 운영 전 **반드시 변경**.
